# CI/CD와 배포 전략 — 10년차 이상 개발자 인터뷰 대비

## 학습 미션

이 워크스페이스는 CI/CD와 배포 전략 주제로 진행되는 시니어 개발자 인터뷰를 체계적으로 준비하기 위한 공간이다.

단순한 도구 사용법이나 파이프라인 개념 암기가 아니라, **배포 안정성**, **자동화 품질**, **롤백 판단**, **release ownership**을 평가하는 10년차 이상 수준의 인터뷰 질문에 답할 수 있는 실무 사고력을 기른다.

## 대상 독자

- CI/CD 도구를 이미 사용해본 경험이 있는 개발자
- 시니어 또는 리드 개발자 포지션 인터뷰를 준비 중인 개발자
- 배포 의사결정과 운영 책임을 맡고 있거나 맡을 예정인 개발자

## 전제 지식

- Git 기본 사용법 (commit, push, branch, merge)
- Linux 기본 명령어
- Docker/컨테이너 기본 개념 (이미지, 컨테이너 실행)
- HTTP 요청/응답 기본 이해

이 트랙은 위 지식을 전제하고 시작한다. 기초 CI/CD 도구 튜토리얼을 원한다면 별도 학습을 권장한다.

## 커리큘럼 주제 범위

| 영역 | 다루는 내용 |
|------|------------|
| CI/CD 파이프라인 | GitHub Actions, Jenkins, 빌드 자동화, 파이프라인 설계 |
| 테스트 자동화 | 테스트 전략, CI 품질 게이트, 커버리지 판단 |
| Artifact 관리 | 버전 관리, 이미지 레지스트리, 배포 패키지 |
| 배포 전략 | Blue-Green, Rolling, Canary 배포의 선택 기준과 trade-off |
| 롤백 전략 | 롤백 판단 기준, 자동/수동 롤백, 데이터 일관성 |
| Release Ownership | 배포 책임 구조, 장애 대응, 포스트모텀 |

## 학습 목표

인터뷰에서 아래 질문을 받았을 때 막힘 없이 답변할 수 있어야 한다:

- "배포 전략을 어떤 기준으로 선택하십니까?"
- "롤백 기준은 어떻게 정의하고 있습니까?"
- "CI 파이프라인에서 품질을 어떻게 보장하십니까?"
- "Canary 배포와 Blue-Green 배포의 차이와 선택 기준은?"
- "배포 중 장애가 발생했을 때 어떻게 판단하고 대응하십니까?"
- "artifact를 어떻게 관리하고 버전을 추적하십니까?"
