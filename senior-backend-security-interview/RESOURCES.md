# 백엔드 보안 / Secure Coding 학습 자료

## 핵심 레퍼런스 (공신력 높은 순)

### 표준 / 공식 문서

- [OWASP Top 10](https://owasp.org/www-project-top-ten/) — 웹 애플리케이션 보안 위험 표준 목록
- [OWASP API Security Top 10](https://owasp.org/www-project-api-security/) — BOLA, Mass Assignment, 과도한 데이터 노출 등 API 특유의 취약점 표준 목록
- [OWASP Cheat Sheet Series](https://cheatsheetseries.owasp.org/) — 주제별 실무 체크리스트 (인증, JWT, SQL Injection 등)
- [OWASP ASVS](https://owasp.org/www-project-application-security-verification-standard/) — 애플리케이션 보안 검증 표준
- [NIST SP 800-63](https://pages.nist.gov/800-63-3/) — 디지털 신원 및 인증 가이드라인
- [RFC 7519 — JWT](https://datatracker.ietf.org/doc/html/rfc7519) — JWT 표준 명세
- [RFC 6749 — OAuth 2.0](https://datatracker.ietf.org/doc/html/rfc6749) — OAuth 2.0 표준
- [RFC 6750 — Bearer Token](https://datatracker.ietf.org/doc/html/rfc6750) — OAuth 2.0 Bearer Token 사용법
- [CWE Top 25](https://cwe.mitre.org/top25/) — 가장 위험한 소프트웨어 취약점 목록
- [STRIDE 위협 모델링](https://learn.microsoft.com/en-us/azure/security/develop/threat-modeling-tool-threats) — Microsoft 위협 분류 프레임워크

### 실습 환경

- [OWASP WebGoat](https://owasp.org/www-project-webgoat/) — 취약점 실습 환경 (의도적으로 취약하게 만든 앱)
- [OWASP Juice Shop](https://owasp.org/www-project-juice-shop/) — 현대적 취약점 실습 환경
- [PortSwigger Web Security Academy](https://portswigger.net/web-security) — 무료 보안 실습 과정 (최고 품질)
- [HackTheBox](https://www.hackthebox.com/) — 실전 침투 테스트 훈련

### 도구

- Burp Suite Community Edition — HTTP 인터셉트 프록시
- OWASP ZAP — 오픈소스 보안 스캐너

## 서적

- *Security Engineering* — Ross Anderson (보안 설계 원칙)
