# 백엔드 보안 / Secure Coding — 10년차 이상 개발자 Interview 대비

## 목표

단순 취약점 정의 암기나 OWASP 나열이 아니라, **위협 모델링, 방어 설계, 실무 보안 판단** 능력을 면접에서 명확하게 보여줄 수 있는 수준까지 도달한다.

## 전제 조건

- 백엔드 개발 경험 5년 이상
- HTTP, REST API, 데이터베이스 기본 개념 이해
- 보안 개요 수준의 사전 지식 (완전 처음이라도 각 레슨에서 전제 개념부터 설명)

## 평가 목표

10년차 이상 시니어 개발자 면접에서 보안 관련 질문은 단순히 "XSS가 뭔가요?"가 아니라 다음을 평가한다.

- 위협을 먼저 식별하고 공격 벡터를 분석하는가?
- 방어를 단일 레이어가 아닌 계층적으로 설계하는가?
- 보안과 편의성/성능 사이의 trade-off를 말할 수 있는가?
- 실무 침해 사례에서 원인과 대응을 연결 지을 수 있는가?
- "왜 그렇게 했나요?"에 근거 있는 답변을 하는가?

## 커리큘럼 개요

| Day | 주제 |
|-----|------|
| 1 | 백엔드 보안 사고 프레임워크와 인증/인가 |
| 2 | 세션 관리와 JWT — 설계 판단 |
| 3 | SQL Injection — 탐지, 방어, 실무 패턴 |
| 4 | XSS와 CSRF — 공격 원리와 방어 설계 |
| 5 | 파일 업로드 보안과 SSRF |
| 6 | API 보안 — 인증, 속도 제한, 데이터 노출 |
| 7 | 비밀번호 저장과 크리덴셜 보호 |
| 8 | Secret 관리와 환경 변수 전략 |
| 9 | Secure Coding 원칙과 코드 리뷰 |
| 10 | 종합 보안 설계 케이스 스터디 |

## 기대 결과

- 면접관의 보안 심층 질문에 위협 → 공격 벡터 → 방어 계층 → trade-off 순서로 답변
- "왜 그렇게 했나요?"에 실무 근거로 대응
- 보안과 편의성 사이의 trade-off를 명확하게 설명
- 침해 사례를 분석하고 재발 방지 설계를 제안
