# 프론트엔드 10년차 이상 개발자 Interview 학습 미션

## 목표

10년차 이상 시니어 프론트엔드 개발자로서 기술 인터뷰에서 **제품 품질과 아키텍처 판단력**을 입증한다.

단순 UI 구현 팁이나 라이브러리 사용법 암기가 아니라, 브라우저라는 플랫폼의 제약을 정확히 이해하고, 성능·접근성·보안·유지보수성을 함께 저울질하며 실무 판단을 내려온 경험을 다음과 같이 시연할 수 있어야 한다.

- 사용자 상호작용이 화면에 반영되기까지 **브라우저 내부에서 일어나는 일**을 구조적으로 설명
- 성능·접근성·보안을 **트레이드오프 언어**로 다루고, 수치(Web Vitals, 성능 예산)로 근거를 제시
- React/Next.js 같은 프레임워크의 **동작 원리**를 알고, 프레임워크가 감추는 비용을 설명
- 장애·회귀·성능 저하 등 **실전 문제 대응 경험**을 구체적으로 공유
- 디자이너·PM·백엔드와의 **크로스펑셔널 협업과 기술 리딩** 경험 제시

## 학습 원칙

- 라이브러리 사용법이나 API 나열이 아니라 **브라우저 제약, 트레이드오프, 실전 장애 사례** 중심으로 다룬다.
- 어려운 개념은 전제 개념부터 쉬운 한국어로 풀고, 어떤 실무 상황과 연결되는지, 흔한 오해까지 설명한다.
- 매 레슨은 면접 질문 → 왜 묻는가 → 전제 개념 → 실무 연결 → 답변 사고 순서 → 핵심 답변 예시 → senior follow-up → 좋은/약한 답변 비교 → 자기 점검 rubric 구조를 따른다.

## 성공 기준

- 인터뷰어가 "follow-up"을 계속 던져도 당황하지 않고 브라우저/아키텍처 수준까지 깊이 있게 답변
- 성능·접근성·보안 트레이드오프를 사용자 경험과 비즈니스 임팩트 관점에서 정당화
- 자신의 실패 경험(성능 회귀, 접근성 미비, 장애 등)을 솔직하게 공유하며 배운 점을 구체적으로 설명
- "지금 이 페이지가 느립니다"와 같은 모호한 문제를 구조적으로 좁혀나가는 진단 절차를 즉답 가능
