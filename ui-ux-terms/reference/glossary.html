용어집 (Glossary)

이 워크스페이스에서 쓰는 용어의 공식 정의. 레슨이 쌓일수록 항목이 추가된다.

기초 개념 (Day 1)

UI — User Interface
사용자 인터페이스
사용자가 제품과 접촉하는 시각적·물리적 요소의 총합. 버튼, 색상, 타이포그래피, 아이콘, 레이아웃 등이 포함된다.
혼동 주의: UX와 같은 뜻으로 쓰지 말 것 — UI는 UX의 한 구성요소다.
UX — User Experience
사용자 경험
사용자가 제품을 발견·사용·이탈하기까지 전 과정에서 쌓이는 인식, 감정, 행동의 총합. UI를 포함하는 더 넓은 개념이다.
혼동 주의: UI와 동의어로 쓰지 말 것
Wireframe
와이어프레임
페이지 레이아웃과 요소 배치를 선과 상자로만 표현한 저충실도(low-fidelity) 스케치. 색상·이미지·폰트 없이 구조만 보여 준다. 정보 구조와 화면 흐름의 빠른 검증에 쓴다.
Mockup
목업
와이어프레임에 색상·폰트·이미지를 입힌 고충실도(high-fidelity) 정적 시각물. 실제처럼 보이지만 클릭이 동작하지 않는다. 시각 디자인 검토와 이해관계자 승인에 쓴다.
혼동 주의: 인터랙티브하면 목업이 아니라 프로토타입이다.
Prototype
프로토타입
실제로 클릭·탭할 수 있는 인터랙티브 모형. 충실도는 낮거나 높을 수 있으며 사용성 테스트와 흐름 검증에 쓴다. 코드 없이 Figma 등으로 만들 수도 있고, 실제 코드로 만들 수도 있다.
