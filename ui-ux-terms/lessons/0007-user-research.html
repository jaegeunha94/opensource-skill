2026-06-27  ·  예상 소요 시간 30분

이 레슨에서 배울 것

  • Persona(페르소나)를 만드는 이유와 구성 요소를 설명할 수 있다
  • User Journey Map(사용자 여정 지도)의 6개 행(Row)과 실무 활용법을 구분할 수 있다
  • Empathy Map(공감 지도)의 4분면과 Pain/Gain을 작성할 수 있다
  • Jobs-to-be-done(JTBD) 프레임워크로 사용자 니즈를 문장으로 표현할 수 있다

1. 사용자 조사(User Research)란

사용자 조사(User Research)는 실제 사용자의 행동·태도·니즈를 이해하기 위해 데이터를 수집하고 분석하는 과정이다. 디자인 결정이 "우리가 생각하는 사용자"가 아닌 "실제 사용자"를 근거로 이루어지도록 한다.

왜 중요한가: Nielsen Norman Group 연구에 따르면 사용자 테스트에 5명만 참가해도 전체 사용성 문제의 약 85%가 발견된다. 조기 리서치 1달러는 출시 후 수정 비용 100달러를 절약한다.

정성적 vs 정량적 조사

Qualitative Research 정성
정성적 조사
왜(Why)와 어떻게(How)를 탐구한다. 인터뷰·관찰·포커스 그룹 등을 통해 사용자의 동기·감정·맥락을 깊이 이해한다. 표본이 작아도 인사이트가 풍부하다.
Quantitative Research 정량
정량적 조사
얼마나(How many)와 얼마나 자주(How often)를 측정한다. 설문·A/B 테스트· 애널리틱스 등을 통해 통계적으로 유의미한 수치를 얻는다. 대규모 표본이 필요하다.
Mixed Methods 혼합
혼합 조사
정성과 정량을 결합한다. 예: 애널리틱스로 이탈 지점을 찾은 뒤(정량) → 사용자 인터뷰로 이유를 규명(정성). 실무에서 가장 신뢰도 높은 접근법이다.

2. Persona (페르소나)

페르소나(Persona)는 조사 데이터를 바탕으로 만든 가상의 대표 사용자다. 실존 인물이 아니라, 여러 사용자 인터뷰·행동 패턴에서 공통된 특징을 추출해 만든 반(半)허구적 캐릭터다. 팀이 "우리 사용자는 누구인가"라는 질문에 일관된 답을 하도록 돕는다.

주의: 페르소나는 데이터 기반이어야 한다. 팀 내부의 가정만으로 만들면 "Proto-Persona(프로토 페르소나)"라 부르며 검증이 반드시 필요하다.

페르소나 구성 요소

👩‍💼
김지수
32세 · 스타트업 마케터 · 서울
목표 (Goals)
  • 캠페인 성과를 빠르게 시각화하고 싶다
  • 데이터 분석에 드는 시간을 줄이고 싶다
  • 상사에게 인사이트를 명확하게 보고하고 싶다
불편함 (Pain Points)
  • 엑셀 수작업으로 인한 반복 업무가 많다
  • 대시보드가 복잡해 원하는 지표를 찾기 어렵다
  • 모바일에서 보고서를 확인하기 불편하다
행동 패턴 (Behaviors)
  • 하루 3회 이상 대시보드 확인
  • Slack으로 팀과 데이터 공유
  • Chrome · MacBook Pro 주 사용
기술 친숙도 (Tech Savvy)
  • Google Analytics 중급
  • Notion·Figma 기본 수준
  • SQL은 배우는 중

페르소나 구성 요소 정리

Demographics
인구통계
나이·직업·거주지·교육 수준. 캐릭터에 현실감을 부여하되 고정관념에 빠지지 않도록 주의한다.
Goals & Motivations
목표와 동기
이 사용자가 제품/서비스를 통해 궁극적으로 이루고 싶은 것. 디자인 결정의 나침반이 된다.
Pain Points
불편함
현재 경험에서 느끼는 불만·장애물·좌절. 해결해야 할 핵심 문제를 드러낸다.
Behaviors
행동 패턴
사용 빈도·채널·도구·루틴. 언제·어디서·어떻게 제품을 사용하는지 그린다.
Quote
대표 인용
실제 인터뷰에서 가져온 한 문장. 팀이 데이터 뒤에 사람이 있음을 상기시킨다.
Tech Savviness
기술 친숙도
사용하는 기기·앱·기술 숙련도. UI 복잡도와 온보딩 설계 기준이 된다.

3. User Journey Map (사용자 여정 지도)

사용자 여정 지도(User Journey Map)는 특정 페르소나가 목표를 이루기 위해 거치는 단계(Stage)별 경험 전체를 시각화한 다이어그램이다. 접점(Touchpoint)·행동(Action)·감정(Emotion)·생각(Thought)·기회(Opportunity)를 한 화면에 담아 "어디서 무너지는가"를 팀이 함께 파악하도록 한다.

여정 지도 예시 — 마케팅 분석 앱 첫 사용

행(Row) 1. 인식 2. 가입 3. 첫 설정 4. 일상 사용 5. 공유·보고
행동
(Action)		 SNS 광고 클릭 이메일로 가입 데이터 소스 연결 대시보드 조회 PDF로 내보내기
접점
(Touchpoint)		 랜딩 페이지 가입 폼 온보딩 위자드 앱 대시보드 공유 모달
생각
(Thought)		 "써 볼만 한가?" "빨리 써보고 싶다" "이게 뭔지 모르겠다" "내가 원하는 게 있네" "상사에게 보내자"
감정
(Emotion)		 😐 호기심 🙂 기대 😕 혼란 😊 만족 😀 자신감
기회
(Opportunity)		 명확한 가치 제안 소셜 로그인 추가 인터랙티브 튜토리얼 개인화 추천 원클릭 Slack 공유
Stage
단계
여정을 나누는 시간 구간. "인식 → 탐색 → 구매 → 사용 → 이탈/재방문" 같은 흐름으로 설정한다.
Touchpoint
접점
사용자가 제품·서비스와 물리적/디지털로 만나는 지점. 앱 화면·이메일·콜센터 모두 접점이 될 수 있다.
Pain Point (Journey)
여정의 불편 지점
감정 곡선이 아래로 꺾이는 지점. 설계 개선의 우선순위를 결정하는 데 사용된다.
Opportunity
기회
각 단계의 불편함을 해결하거나 경험을 향상할 수 있는 디자인·기능·프로세스 아이디어.
Emotional Curve
감정 곡선
단계별 사용자 감정을 선 그래프로 표현한 것. 최저점(Moment of Truth)이 집중 개선 대상이다.
Moment of Truth
결정적 순간
여정에서 사용자가 서비스를 계속 쓸지 떠날지 결정되는 핵심 순간. 부정적이면 이탈로 이어진다.

4. Empathy Map (공감 지도)

공감 지도(Empathy Map)는 사용자의 내면 세계를 4분면(Think·Feel·Say·Do)으로 시각화하는 도구다. 여정 지도보다 더 좁은 특정 상황에 집중하며, 페르소나를 만들기 전 또는 팀이 사용자를 공감하는 워크숍 도구로 자주 쓰인다.

공감 지도 구조

사용자: 김지수 · 상황: 월간 보고서 작성 중
🧠 Think & Feel — 생각과 느낌
  • "이 데이터가 정말 맞는 건지 불안하다"
  • "마감이 내일인데 시간이 부족하다"
  • "상사가 내 보고서를 어떻게 볼까 걱정된다"
👂 Hear — 듣는 것
  • "다른 팀은 자동화 툴 쓴다더라"
  • 팀장: "이번엔 좀 더 시각적으로 보여줘"
  • 동료 Slack: "이 앱 진짜 편하다"
👁 See — 보는 것
  • 복잡한 스프레드시트 화면
  • 동료가 대시보드 스크린샷 공유하는 것
  • LinkedIn 광고 "5분 만에 보고서 완성"
🗣 Say & Do — 말과 행동
  • "엑셀이 좀 불편해도 익숙하니까요"
  • 매주 수동으로 데이터 복붙
  • 새 툴 무료 체험 신청 후 방치
😟 Pains (두려움·좌절·장벽)
  • 데이터 오류에 대한 불안
  • 새 툴 학습 비용이 두려움
  • 보고서 마감 압박
😊 Gains (바람·욕구·성공 기준)
  • 보고서 작성 시간 50% 단축
  • 상사에게 "한눈에 보인다"는 칭찬
  • 데이터에 자신감이 생기는 것

5. Jobs-to-be-done (JTBD)

Jobs-to-be-done(JTBD)은 Clayton Christensen이 제안한 프레임워크로, 사용자가 제품을 "고용(hire)"하는 이유를 기능적·감성적·사회적 관점에서 분석한다. "어떤 기능이 필요한가"가 아니라 "어떤 일을 완수하려고 이 제품을 선택하는가"를 묻는다.

JTBD 문장 구조

JTBD 문장 템플릿
"When I [상황], I want to [동기·목표], so I can [기대 결과]."
예시 1 — 기능적 Job
"마케팅 보고서를 매주 작성할 때, 여러 채널의 지표를 한 화면에서 보고 싶다, 그러면 데이터를 합치는 수작업 없이 빠르게 인사이트를 얻을 수 있다."
Functional Job — 실용적 과제를 해결
예시 2 — 감성적 Job
"임원에게 보고할 때, 데이터가 정확하다는 확신을 갖고 싶다, 그러면 발표 중 실수를 두려워하지 않아도 된다."
Emotional Job — 감정적 안정감을 원함
예시 3 — 사회적 Job
"팀 미팅에서 인사이트를 공유할 때, 전문가처럼 보이고 싶다, 그러면 팀 내 신뢰와 영향력을 높일 수 있다."
Social Job — 타인의 시선과 지위를 고려
JTBD vs 페르소나: 페르소나는 "누구인가(Who)"를 답하고, JTBD는 "왜 사용하는가(Why)"를 답한다. 두 도구는 함께 사용할 때 가장 강력하다.

6. 사용자 조사 프로세스

1
리서치 계획 수립

목표·질문·방법·대상·일정을 정의한다. "우리가 검증하고 싶은 가정은 무엇인가?"에서 출발한다.

2
데이터 수집 (인터뷰·관찰·설문)

사용자 인터뷰 5~8명, 컨텍스추얼 인콰이어리, 설문 조사, 사용성 테스트 등을 목적에 맞게 선택한다.

3
어피니티 다이어그램 (Affinity Diagram)

수집한 발견을 포스트잇에 적어 주제별로 군집화(Clustering)한다. 패턴과 공통 니즈를 찾는 단계다.

4
페르소나·여정 지도·공감 지도 작성

군집화된 데이터를 시각화 도구로 표현한다. 팀이 동일한 사용자 모델을 공유하게 된다.

5
인사이트 도출 및 기회 정의

여정 지도의 Opportunity를 기반으로 HMW(How Might We) 질문을 만들고 다음 디자인 스프린트로 연결한다.

주요 조사 방법 비교

방법 유형 목적 주요 산출물
심층 인터뷰 (In-depth Interview) 정성 동기·맥락·태도 탐구 페르소나, 공감 지도
컨텍스추얼 인콰이어리 (Contextual Inquiry) 정성 실제 환경에서 행동 관찰 여정 지도, 태스크 플로우
설문 조사 (Survey) 정량 의견·빈도·선호 측정 통계 보고서
사용성 테스트 (Usability Test) 혼합 특정 태스크의 성공률·어려움 측정 이슈 리스트, 우선순위 개선 항목
카드 소팅 (Card Sorting) 혼합 사용자의 멘탈 모델 파악 (IA 설계) 사이트맵, 네비게이션 구조
추천 1차 출처
Personas: Study Guide — Nielsen Norman Group

사용자 여정 지도는 Journey Mapping 101 (NN/g)을, JTBD는 Harvard Business Review — Know Your Customers' Jobs-to-be-Done을 참고하자.

7. 확인 퀴즈

답을 보기 전에 먼저 스스로 생각해 보자.

Q1. 페르소나를 만들 때 "데이터가 아닌 팀 내부 가정만으로 만든 페르소나"를 뭐라 부르는가?

Q2. 사용자 여정 지도에서 감정 곡선이 가장 아래로 꺾이는, 사용자가 서비스 지속 여부를 결정하는 순간을 뭐라 하는가?

Q3. 공감 지도(Empathy Map)의 4분면에 해당하지 않는 항목은?

Q4. JTBD 문장 "임원에게 보고할 때 데이터 정확성에 확신을 갖고 싶다, 발표 중 실수를 두려워하지 않기 위해"는 어떤 유형의 Job인가?

Q5. 수집한 인터뷰 메모를 포스트잇에 적어 주제별로 군집화하는 방법은?
