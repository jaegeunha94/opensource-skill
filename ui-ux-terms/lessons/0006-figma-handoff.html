1. 핸드오프(Handoff)란

핸드오프(Handoff)는 디자인 작업이 완료된 후 개발자가 구현할 수 있도록 설계 정보(치수·색상·폰트·컴포넌트 상태 등)를 전달하는 과정 전체를 가리킨다. 전통적으로 PDF·PNG 레드라인(Redline) 문서를 수작업으로 만들었지만, 현재는 Figma와 Zeplin 같은 툴이 이 과정을 자동화한다.

핵심 질문: "이 버튼의 배경색이 뭔가요?" "아이콘 크기는 몇 px인가요?" — 개발자가 이런 질문을 디자이너에게 매번 묻지 않아도 되도록 해 주는 것이 핸드오프 툴의 역할이다.

2. Figma 핵심 핸드오프 용어

Inspect Panel Figma
인스펙트 패널
Figma 우측 패널의 탭 중 하나. 선택한 레이어의 CSS 스니펫·치수·색상· 타이포그래피·간격 값을 자동으로 보여준다. 개발자가 Figma 뷰어 권한만으로 접근할 수 있다.
Dev Mode Figma
개발자 모드
2023년 이후 Figma에 추가된 개발자 전용 뷰. CSS·Android·iOS 코드 스니펫, 변수(Variable) 값, 주석(Annotation), 레이아웃 그리드를 한 화면에 표시한다. Inspect Panel의 상위 버전으로 이해하면 된다.
Design Spec 공통
디자인 스펙
컴포넌트나 화면의 시각적 속성(크기·색상·간격·폰트·상태)을 체계적으로 명세한 문서. Figma Dev Mode, Zeplin, Storybook Docs 등 다양한 형태로 생성된다.
Redline 공통
레드라인
요소 간 거리·패딩·마진을 빨간선(→←)으로 표시한 치수 주석 방식. Figma Dev Mode 또는 Zeplin에서 요소를 hover하면 자동 생성된다. 과거에는 Sketch + Zeplin 또는 PNG 수동 작업으로 만들었다.
Annotation Figma
어노테이션
디자인 파일 위에 추가하는 메모·설명 레이어. 개발자에게 특정 동작·상태·제약 조건을 별도로 설명할 때 사용한다. Figma의 공식 Annotation 기능(Dev Mode) 또는 커스텀 프레임으로 만든다.
Variable Figma
변수
Figma에서 색상·숫자·문자열 값을 저장하는 단위. 디자인 토큰(Day 3)과 동일 개념이며 Dev Mode에서 CSS 변수 이름으로 연결되어 코드에 바로 적용할 수 있다.

3. Inspect Panel — 개발자가 읽는 정보

Figma에서 프레임·컴포넌트·텍스트 레이어를 선택하면 우측 Inspect 탭에서 아래와 같은 정보를 확인할 수 있다.

Figma Inspect Panel — 버튼 컴포넌트 예시
Width × Height 120 × 40 px
Background #4F46E5 (Indigo/600)
Border Radius 8 px
Padding 10 px 20 px
Font Inter · 14 px · Semibold · #FFFFFF
Shadow 0 1px 3px rgba(0,0,0,0.12)
CSS (Dev Mode) background: var(--indigo-600);
팁: Figma Dev Mode에서 토큰 이름(var(--indigo-600))이 나타나려면 디자이너가 Variable이나 Style에 이름을 붙여 두어야 한다. 원시 hex 값만 보인다면 토큰 정의가 빠진 것이므로 디자이너에게 확인 요청을 해야 한다.

4. Export 설정 — 에셋 전달

아이콘·이미지·일러스트 같은 에셋은 Figma에서 직접 내보낼 수 있다. 실무에서 자주 쓰는 포맷과 배율 기준을 정리하면 다음과 같다.

에셋 유형 권장 포맷 배율 설정 사용 이유
아이콘 (단색·다색) SVG 1× 벡터라 크기 무관하게 선명; 코드에서 색상 변경 가능
사진·복잡한 이미지 WebP / JPG 1× 2× 3× Retina 대응; @1x 기본, @2x 고해상도, @3x 최고 해상도
투명 배경 이미지 PNG 1× 2× 3× 알파 채널 지원; 일러스트·스티커 등에 적합
애니메이션 Lottie (JSON) – After Effects 연동; 벡터 기반 경량 애니메이션
@1x / @2x / @3x
배율 (Scale Factor)
디바이스 픽셀 밀도(DPR)에 따른 이미지 배율. 표준 화면은 1×, Retina/고밀도 디스플레이는 2×·3×를 사용한다. srcset 속성 또는 iOS/Android의 @2x/@3x 파일 명명 규칙으로 제공된다.
Slice
슬라이스
Figma에서 특정 영역을 지정해 그 범위만 내보내는 기능. 겹치는 레이어나 그룹 경계를 무시하고 원하는 직사각형 영역을 정확하게 내보낼 때 유용하다.
Mark for Export
내보내기 지정
Figma에서 레이어에 Export 설정을 미리 정의해 두는 것. 개발자가 Assets 패널 → Export에서 한꺼번에 다운로드할 수 있도록 디자이너가 사전에 설정한다.

5. Zeplin — 전용 핸드오프 플랫폼

Zeplin은 Figma/Sketch/XD 파일을 가져와 개발자 전용 핸드오프 환경을 제공하는 서드파티 툴이다. 2023년 이후 Figma가 Dev Mode를 강화하면서 포지셔닝이 겹쳤지만, 스타일 가이드 자동화·조직 단위 관리·Jira 연동 등 팀 규모가 큰 곳에서 여전히 사용된다.

Styleguide Zeplin
스타일가이드
Zeplin이 Figma 파일에서 색상·텍스트 스타일·컴포넌트를 자동 추출해 생성하는 문서. 팀 전체가 공통 스타일을 한 곳에서 확인하고 코드 스니펫을 복사할 수 있다.
Screen Zeplin
스크린
Zeplin에서 개별 디자인 아트보드를 가리키는 단위. 각 스크린에서 레이어를 클릭하면 치수·색상·코드가 표시된다. Figma의 Frame에 해당한다.
Connected Components Zeplin
연결된 컴포넌트
Zeplin 화면의 컴포넌트와 코드 저장소의 실제 컴포넌트를 연결하는 기능. 개발자가 "이 버튼 = <Button variant="primary">" 라는 매핑을 바로 확인할 수 있다.

Figma Dev Mode vs Zeplin — 포지셔닝 비교

기능 Figma Dev Mode Zeplin
접근성 Figma 파일 내 전환 별도 웹앱·데스크탑 앱
코드 스니펫 CSS / iOS (SwiftUI) / Android (Compose) CSS / Swift / Kotlin / React 등 다중
스타일 가이드 Figma Variables / Styles 자동 추출 + 팀 공유 페이지
연동 Jira·GitHub (플러그인) Jira·Slack·GitHub 네이티브 연동
가격 Figma 유료 플랜 포함 별도 구독

6. 핸드오프 준비 체크리스트

디자이너가 핸드오프 전 확인해야 할 사항, 개발자가 수령 시 확인해야 할 사항.

1
컴포넌트 정리

모든 요소가 컴포넌트로 캡슐화되어 있고, 이름이 명확한가? (예: Button/Primary/Default)

2
스타일 정의

색상과 폰트가 Figma Styles 또는 Variables로 정의되어 코드 토큰 이름이 붙어 있는가?

3
상태(State) 포함

Default·Hover·Active·Disabled·Error 등 모든 상태 디자인이 포함되어 있는가?

4
에셋 내보내기 지정

아이콘·이미지 레이어에 Export 설정(SVG/PNG, 배율)이 미리 지정되어 있는가?

5
어노테이션 추가

애니메이션·제스처·엣지케이스처럼 시각만으로 전달하기 어려운 동작을 주석으로 설명했는가?

6
뷰어 권한 확인

개발자에게 Figma 파일 또는 Zeplin 프로젝트의 Viewer/Developer 권한을 부여했는가?

추천 1차 출처
Figma Dev Mode 공식 가이드 — Figma Help Center

Zeplin 비교는 Zeplin 공식 블로그를, Export 포맷 결정은 web.dev의 Learn Images를 참고하자.

Q2. 요소 간 거리·패딩·마진을 빨간선으로 자동 표시해 주는 핸드오프 기능의 이름은?
