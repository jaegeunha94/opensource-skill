Day 4 — 인터랙션 패턴: 어포던스·마이크로인터랙션·피드백

학습 목표

- Affordance(어포던스)와 Signifier(시그니파이어)의 차이를 구분하고 실제 UI에서 찾아낼 수 있다.

- Microinteraction(마이크로인터랙션)의 4단계 구조를 설명할 수 있다.

- Feedback Loop, Direct Manipulation, Gestural Navigation 등 주요 인터랙션 용어를 맥락에 맞게 사용할 수 있다.

1. Affordance (어포던스)

Affordance는 어떤 객체나 UI 요소가 사용자에게 "이렇게 쓸 수 있어"라고 암묵적으로 알려주는 행동 가능성을 뜻한다. 심리학자 J.J. Gibson이 생태학적 심리학에서 처음 제안한 개념으로, Donald Norman이 디자인 문맥에 도입했다.

핵심 정의

사용자와 객체 사이의 관계에서 발생하는 행동 가능성. 버튼처럼 생긴 요소는 "누를 수 있다"는 어포던스를 갖는다.

종류설명예시
Physical Affordance
(물리적 어포던스)		 실제 물체의 물리적 특성에서 오는 행동 가능성 문 손잡이 — 잡아당기거나 밀 수 있음을 형태로 암시
Perceived Affordance
(지각된 어포던스)		 사용자가 인지하는 행동 가능성. 실제 기능과 다를 수 있다. 파란 밑줄 텍스트 — 링크처럼 보이면 클릭하려 함
False Affordance
(거짓 어포던스)		 실제로 그 동작이 안 되지만 가능한 것처럼 보이는 경우 클릭 불가 요소인데 버튼처럼 생긴 배너
Hidden Affordance
(숨겨진 어포던스)		 기능이 있지만 사용자에게 보이지 않아 발견이 어려운 경우 우클릭 메뉴, 스와이프 삭제 기능
실무 예시
모바일 앱에서 카드 컴포넌트에 그림자(Shadow)를 적용하면 "들어 올릴 수 있는 실체"처럼 보여 탭(클릭) 어포던스를 강화한다.

2. Signifier (시그니파이어)

Donald Norman은 The Design of Everyday Things 개정판(2013)에서 어포던스와 구분되는 개념으로 Signifier를 도입했다.

개념역할예시
Affordance 실제 행동 가능성 (관계) 버튼을 누를 수 있다는 물리적·지각적 사실
Signifier 행동 가능성을 명시적으로 알려주는 신호 버튼 위에 쓰인 "클릭" 레이블, 화살표 아이콘

한 줄 요약: Affordance는 "무엇을 할 수 있는지", Signifier는 "어디서·어떻게 하는지를 알려주는 단서"다.

좋은 UI는 충분한 Signifier를 제공해 사용자가 탐색 없이도 기능을 바로 발견할 수 있게 한다. Signifier가 부족하면 기능이 있어도 사용자가 찾지 못하는 Discoverability(발견 가능성) 문제가 생긴다.

3. Microinteraction (마이크로인터랙션)

Dan Saffer가 2013년 저서 Microinteractions에서 정리한 개념. Microinteraction은 하나의 과업을 위한 단일 목적의 작은 인터랙션이다. 제품의 디테일 품질을 결정한다.

마이크로인터랙션의 4단계 구조

┌─────────────────────────────────────────────────────────┐
│  1. Trigger (트리거)                                      │
│     사용자 행동 또는 시스템 이벤트가 마이크로인터랙션을 시작  │
├─────────────────────────────────────────────────────────┤
│  2. Rules (규칙)                                          │
│     트리거 이후 무슨 일이 일어날지 정의하는 로직            │
├─────────────────────────────────────────────────────────┤
│  3. Feedback (피드백)                                     │
│     사용자에게 현재 상태를 알려주는 시각·청각·햅틱 신호      │
├─────────────────────────────────────────────────────────┤
│  4. Loops & Modes (루프 & 모드)                          │
│     반복·지속 조건과 특별 상황(예외 모드) 처리              │
└─────────────────────────────────────────────────────────┘
단계정의예시 (좋아요 버튼)
Trigger인터랙션을 시작시키는 사건사용자가 하트 버튼을 탭
Rules무엇이 일어날지 결정좋아요 수 +1, 상태 토글
Feedback결과를 사용자에게 알림하트 색상 변경 + 팝 애니메이션
Loops & Modes반복 조건 / 예외 처리취소 시 역방향 애니메이션, 네트워크 오류 시 실패 표시
실무 예시
Pull-to-Refresh: 사용자가 아래로 당기면(Trigger) → 일정 임계값 초과 여부 판단(Rules) → 스피너 + 진행 표시(Feedback) → 완료 후 콘텐츠 갱신(Loop).

4. Feedback Loop (피드백 루프)

Feedback Loop는 사용자 행동의 결과가 다시 사용자에게 전달되어 다음 행동을 유도하는 순환 구조다. 닐슨 휴리스틱의 "시스템 상태 가시성(Visibility of System Status)"과 직결된다.

피드백 유형설명예시
Visual Feedback색상·크기·형태 변화로 상태 전달버튼 호버 시 배경색 변경
Auditory Feedback소리로 상태 전달알림음, 오류 비프음
Haptic Feedback진동으로 상태 전달 (모바일)iPhone 햅틱 터치, 결제 성공 진동
Temporal Feedback시간 경과를 알려주는 피드백프로그레스 바, 스켈레톤 UI

Positive vs Negative Feedback

종류역할예시
Positive Feedback
(긍정적 피드백)		 행동을 강화·증폭 (계속 하도록 유도) 스트릭 뱃지 (매일 사용 시 연속 기록 표시)
Negative Feedback
(부정적 피드백)		 행동을 억제·안정화 (이탈 방지, 오류 수정) 비밀번호 불일치 시 빨간 경고 메시지

5. Direct Manipulation (직접 조작)

Ben Shneiderman이 1983년 제안한 인터랙션 패러다임. 사용자가 명령어 입력 없이 화면 객체를 직접 조작하는 방식이다.

직접 조작의 3가지 원칙

  1. Continuous Representation (지속적 표현): 관심 객체가 항상 화면에 보임
  2. Physical Actions (물리적 행동): 드래그·스와이프·핀치 같은 실제 행동으로 조작
  3. Rapid, Reversible Operations (빠르고 취소 가능한 조작): 결과가 즉시 표시되고, Undo가 가능
예시
- 파일 탐색기에서 아이콘 드래그하여 폴더로 이동
- 지도 앱에서 핀치 투 줌(Pinch-to-Zoom)
- Figma에서 레이어를 드래그하여 순서 변경

6. Gestural Navigation (제스처 네비게이션)

물리 버튼 없이 터치·스와이프·핀치 등 손 제스처로 앱을 탐색하는 방식. iOS 12와 Android 10부터 운영체제 기본 네비게이션으로 채택되었다.

제스처일반적 의미플랫폼 예시
Swipe Back이전 화면으로 돌아가기iOS 엣지 스와이프
Swipe Up홈으로 이동 / 카드 올리기iOS 홈 제스처
Long Press추가 옵션(컨텍스트 메뉴)Android 앱 아이콘 장누르기
Pinch / Spread축소 / 확대지도, 갤러리
Pull to Refresh콘텐츠 새로고침소셜 미디어 피드
Swipe to Dismiss카드·알림 제거알림 센터

주의: 제스처는 Discoverability(발견 가능성)가 낮다. 충분한 Signifier(온보딩 애니메이션, 툴팁)가 없으면 사용자가 기능을 모른 채 지나친다.

7. 추가 핵심 용어 빠른 정리

용어정의
Mapping (매핑) 컨트롤과 결과 간의 관계. 직관적 매핑일수록 학습 없이 조작 가능. 예: 볼륨 슬라이더를 오른쪽으로 밀면 소리가 커짐.
Constraints (제약) 잘못된 조작을 물리적·논리적으로 막는 설계. 예: 날짜 입력 필드에 숫자 외 입력 방지, 비활성 버튼(disabled).
Affordance Mismatch 사용자가 기대하는 동작과 실제 동작이 다른 상태. 버튼처럼 생겼는데 클릭이 안 되는 경우.
Hover State (호버 상태) 포인터가 UI 요소 위에 있을 때의 시각 상태. 클릭 가능 여부를 알리는 Signifier 역할.
Active State (활성 상태) 버튼을 클릭/탭하는 순간의 눌린 시각 상태. 피드백을 즉각 제공.
Loading State (로딩 상태) 시스템이 처리 중임을 알리는 상태. 스피너, 스켈레톤 UI, 프로그레스 바 등으로 표현.
Empty State (빈 상태) 콘텐츠가 없는 화면. 다음 행동을 유도하는 일러스트·CTA를 포함해야 함.
Error State (오류 상태) 잘못된 입력 또는 시스템 오류를 알리는 상태. 원인 + 해결 방법을 함께 제시해야 함.

9. 자가 점검 퀴즈

Q1. 문 손잡이가 "당기는 형태"로 설계되었을 때 이것은 어포던스인가, 시그니파이어인가?

두 가지 모두 해당될 수 있다. 손잡이의 물리적 형태 자체는 "잡아당길 수 있다"는 Perceived Affordance를 만든다. 만약 그 위에 추가로 "PULL ←" 레이블이 붙어 있다면, 그것이 Signifier다. 형태가 명확하게 행동을 알려준다면 형태 자체도 Signifier 역할을 겸한다.

Q2. 마이크로인터랙션의 4단계를 순서대로 말해보자.

Trigger → Rules → Feedback → Loops & Modes. Trigger가 없으면 인터랙션이 시작되지 않고, Feedback이 없으면 사용자는 행동이 완료됐는지 알 수 없다.

Q3. "앱에서 메시지를 왼쪽으로 스와이프하면 삭제된다"는 기능을 사용자가 처음 만났을 때 발견하기 어렵다. 이 현상을 설명하는 용어는?

Hidden Affordance(숨겨진 어포던스) 또는 Discoverability 부족. 스와이프 삭제 기능 자체는 있지만, 이를 알려주는 Signifier가 없어 사용자가 우연히 발견하기 전까지 모른다. 해결 방법: 온보딩 애니메이션, 첫 아이템 힌트 강조.

실습 과제
본인이 자주 쓰는 앱 하나를 열고 다음을 찾아보세요:
1) 잘 설계된 Microinteraction 하나 (어떤 Trigger, Feedback인지 설명)
2) Signifier가 부족해서 처음에 발견하기 어려웠던 기능 하나
3) False Affordance 또는 Affordance Mismatch 사례 하나

10. 핵심 용어 요약

용어한 줄 정의
Affordance객체가 가진 행동 가능성 (물리적·지각적 관계)
Signifier행동 가능성을 명시적으로 알려주는 신호/단서
Microinteraction단일 목적의 작은 인터랙션 (Trigger→Rules→Feedback→Loops)
Feedback Loop행동 결과가 다시 사용자에게 전달되는 순환 구조
Direct Manipulation화면 객체를 명령어 없이 직접 조작하는 패러다임
Gestural Navigation터치 제스처로 앱을 탐색하는 인터페이스 방식
Mapping컨트롤과 결과 간의 직관적 관계
Constraints잘못된 조작을 방지하는 디자인 제약
Discoverability기능을 사용자가 얼마나 쉽게 발견할 수 있는지
