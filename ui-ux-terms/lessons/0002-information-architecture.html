1. 정보 구조(IA)란 무엇인가?

정보 구조(Information Architecture, IA)는 디지털 제품 안의 콘텐츠와 기능을 사용자가 쉽게 찾고 이해할 수 있도록 조직화하고 레이블을 붙이는 설계 학문이다. 쉽게 말해 "무엇을 어디에 두고 어떻게 부를 것인가"에 대한 청사진이다.

비유: IA는 도서관의 책 분류 체계와 같다. 책이 아무리 많아도 십진법 분류(DDC)가 있으면 원하는 책을 찾을 수 있다. 앱도 마찬가지다 — 콘텐츠가 아무리 많아도 IA가 잘 설계되면 사용자는 길을 잃지 않는다.

IA는 크게 네 가지 구성 요소로 이루어진다.

Organization Systems
조직화 체계
콘텐츠를 그룹으로 묶는 방식. 주제별, 작업별, 대상별, 시간순 등 다양한 분류 기준을 사용한다.
Labeling Systems
레이블링 체계
각 항목과 카테고리에 이름을 붙이는 방식. "설정" vs "환경설정" vs "옵션" 중 무엇이 사용자에게 더 직관적인가를 결정한다.
Navigation Systems
네비게이션 체계
사용자가 콘텐츠 간에 이동하는 방법. 글로벌 네비게이션, 로컬 네비게이션, 컨텍스추얼 네비게이션이 있다.
Search Systems
검색 체계
사용자가 원하는 정보를 직접 검색으로 찾을 수 있게 하는 방법. 검색 범위, 필터, 자동완성 등을 포함한다.

2. 사이트맵(Sitemap)

사이트맵(Sitemap)은 웹사이트나 앱의 전체 페이지 구조를 시각적으로 표현한 다이어그램이다. IA를 문서화하는 가장 기본적인 산출물로, 개발 전 팀 전체가 구조에 합의할 때 사용한다.

Flat Sitemap
평면형 사이트맵
모든 페이지가 1~2단계 깊이로 배치된 단순한 구조. 소규모 사이트나 단일 목적 앱에 적합하다.

예: 랜딩 페이지, 포트폴리오 사이트
Deep Sitemap
계층형 사이트맵
여러 단계의 계층 구조를 가진 복잡한 구조. 콘텐츠가 많을수록 필요하지만, 깊이가 깊어질수록 사용자가 길을 잃기 쉽다.

예: 이커머스 사이트, 기업 포털
항목 사이트맵(Sitemap) 와이어프레임(Wireframe)
표현 대상 전체 페이지 구조와 연결 관계 개별 페이지의 레이아웃과 요소 배치
단위 페이지/섹션 수준 화면 수준
목적 구조 합의, 범위 파악 레이아웃 검토, 정보 우선순위 확인
작성 시점 와이어프레임 이전 사이트맵 이후
실무 팁: 사이트맵 없이 와이어프레임부터 만들면 나중에 페이지 수가 폭발적으로 늘어나거나 중복 페이지가 생기는 경우가 많다. 사이트맵을 먼저 그리는 습관을 들이자.

3. 카드 소팅(Card Sorting)

카드 소팅(Card Sorting)은 IA를 설계하거나 검증할 때 사용하는 사용자 리서치 기법이다. 참여자들에게 각 콘텐츠 항목이 적힌 카드를 나눠 주고, 자신이 생각하는 방식대로 그룹을 만들고 이름을 붙이게 한다. 사용자의 멘탈 모델(mental model)을 파악하는 데 가장 효과적인 방법이다.

Open Card Sorting
개방형 카드 소팅
참여자가 직접 그룹 이름도 만든다. 사용자가 어떤 카테고리를 자연스럽게 떠올리는지 파악할 때 사용한다.

쓰임: 초기 IA 설계 단계
Closed Card Sorting
폐쇄형 카드 소팅
그룹 이름은 미리 정해져 있고, 참여자는 카드를 해당 그룹에 배치하기만 한다. 기존 IA가 사용자에게 직관적인지 검증할 때 사용한다.

쓰임: 기존 구조 검증 단계
Mental Model
멘탈 모델
사용자가 시스템이나 제품이 어떻게 작동하는지에 대해 머릿속에 가진 인식 체계. IA는 사용자의 멘탈 모델에 맞춰야 마찰이 줄어든다.

4. 네비게이션 패턴(Navigation Patterns)

네비게이션 패턴은 사용자가 앱이나 사이트 내에서 이동하는 방법을 구조화한 UI 패턴이다. 올바른 패턴 선택이 IA의 가독성을 좌우한다.

Global Navigation
글로벌 네비게이션
모든 페이지에서 항상 접근 가능한 최상위 메뉴. 헤더 바, 하단 탭 바 등이 해당한다.

예: 인스타그램 하단 탭 바
Local Navigation
로컬 네비게이션
현재 섹션 안에서만 표시되는 서브 메뉴. 글로벌 네비게이션의 하위 레벨이다.

예: 설정 화면의 카테고리 목록
Breadcrumb
브레드크럼
사용자가 현재 어디에 있는지 계층 경로를 보여 주는 보조 네비게이션. 홈 > 카테고리 > 현재 페이지 형태로 표시한다.

예: 이커머스 상품 상세 페이지 상단
Hamburger Menu
햄버거 메뉴
세 줄 아이콘(☰)을 눌러 숨겨진 네비게이션을 여는 패턴. 화면 공간을 절약하지만 메뉴 항목의 발견 가능성(discoverability)이 떨어진다는 비판도 있다.
패턴 특징 주로 쓰이는 곳
탭 바 (Tab Bar) 항상 노출, 최대 5개 항목 모바일 앱 (iOS/Android)
사이드바 (Sidebar) 계층 표현에 유리, 공간 차지 데스크톱 앱, 대시보드
햄버거 메뉴 공간 절약, 발견 어려움 모바일 웹, 보조 메뉴
브레드크럼 현재 위치 파악, 뒤로 이동 깊은 계층 구조를 가진 사이트
실무 팁: 네비게이션 패턴은 콘텐츠 양과 계층 깊이에 따라 달라진다. 모바일 앱에서 최상위 섹션이 5개 이하라면 탭 바가 가장 발견 가능성이 높다. 항목이 많아지면 사이드바나 햄버거 메뉴를 검토하되, 발견 가능성 저하를 테스트로 확인하자.
추천 1차 출처
IA vs. Navigation — Nielsen Norman Group

NN/g의 IA와 네비게이션 차이를 다룬 아티클. 카드 소팅 방법론은 Card Sorting: Uncover Users' Mental Models를 함께 읽으면 이해가 깊어진다.
