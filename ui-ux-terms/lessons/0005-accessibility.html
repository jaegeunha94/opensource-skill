접근성(a11y) — WCAG·ARIA·포커스 관리

2026-06-25  ·  예상 소요 시간 25분

이 레슨에서 배울 것

  • 접근성(a11y)이 왜 개발·디자인 협업에서 필수 언어인지 설명할 수 있다
  • WCAG의 4원칙(POUR)과 성공 기준 레벨(A / AA / AAA)을 구분할 수 있다
  • ARIA의 역할(role), 속성(property), 상태(state)를 정확히 구분할 수 있다
  • 포커스 관리와 색상 대비 기준을 실무 리뷰에서 적용할 수 있다

1. 접근성(Accessibility, a11y)이란

접근성(Accessibility)은 장애가 있거나 없는 모든 사람이 제품·서비스를 동등하게 이용할 수 있도록 설계하는 것이다. "a11y"는 accessibility 사이의 11글자를 숫자로 줄인 약어로, 현업 대화·이슈 트래커에서 흔히 쓰인다.

왜 협업에서 중요한가? 접근성은 법적 의무(한국 전자정부법, 미국 ADA·Section 508, 유럽 EN 301 549)이자 SEO·유지보수 품질 지표이기도 하다. 디자인 리뷰에서 "이 컴포넌트 a11y 고려됐어요?"라는 질문이 나오면 WCAG와 ARIA 용어로 답할 수 있어야 한다.
Accessibility (a11y)
접근성
시각·청각·운동·인지 장애를 포함한 모든 사람이 콘텐츠와 기능을 이용할 수 있도록 보장하는 설계 원칙.
Assistive Technology (AT)
보조 기술
스크린 리더(NVDA, VoiceOver, TalkBack), 스위치 제어, 화면 확대기, 자막 등 장애인이 디지털 콘텐츠에 접근하는 데 사용하는 도구.
Inclusive Design
포용적 디자인
특정 그룹을 위한 사후 조치가 아니라 처음부터 다양한 사용자를 고려해 설계하는 접근법. 결과적으로 모든 사용자에게 더 나은 경험을 제공한다.

2. WCAG — 웹 콘텐츠 접근성 지침

WCAG(Web Content Accessibility Guidelines)는 W3C의 WAI(Web Accessibility Initiative)가 발행하는 국제 표준이다. 2025년 현재 실무 기준은 WCAG 2.1 AA이며, WCAG 2.2(2023)가 최신 버전이다.

WCAG 4원칙 — POUR

모든 성공 기준은 다음 4원칙 중 하나에 속한다.

P
Perceivable (인식 가능)
정보와 UI 컴포넌트는 사용자가 인식할 수 있는 방식으로 제공되어야 한다.
예: 이미지에 대체 텍스트, 동영상에 자막 제공
O
Operable (운용 가능)
UI 컴포넌트와 네비게이션은 조작 가능해야 한다.
예: 키보드만으로 모든 기능 사용 가능, 충분한 시간 제공
U
Understandable (이해 가능)
정보와 UI 조작 방법은 이해할 수 있어야 한다.
예: 명확한 오류 메시지, 일관된 네비게이션
R
Robust (견고함)
콘텐츠는 보조 기술을 포함한 다양한 사용자 에이전트가 신뢰성 있게 해석할 수 있어야 한다.
예: 유효한 HTML, ARIA 올바른 사용

성공 기준 레벨

레벨 의미 실무 적용
A 최소 요구사항. 이를 충족하지 않으면 일부 사용자가 콘텐츠에 접근 불가 모든 프로젝트에서 필수
AA 국제·법적 기준. 대부분의 장애 유형에 대응 실무 최소 기준 (한국 정부·EU 의무)
AAA 최고 수준. 모든 콘텐츠에서 달성하기 어렵지만 중요 콘텐츠에 적용 공공 서비스·의료·교육에서 권장

3. 색상 대비(Color Contrast)

WCAG 1.4.3(AA)은 일반 텍스트의 전경색과 배경색 대비율이 최소 4.5:1이어야 하며, 큰 텍스트(18pt 이상 또는 14pt Bold)는 3:1을 요구한다. WCAG 1.4.11(AA)은 UI 컴포넌트의 초점 표시자나 아이콘 같은 비텍스트 요소도 3:1을 요구한다.

노란 배경 + 노란 텍스트
대비율 1.6:1 — 실패
파란 배경 + 흰 텍스트
대비율 8.6:1 — AA/AAA 통과
짙은 남색 + 흰 텍스트
대비율 17.5:1 — AAA 통과
Contrast Ratio
대비율
두 색상의 상대적 밝기(relative luminance) 차이를 나타내는 숫자. 1:1(동일)에서 21:1(흑백 최대)까지의 범위. Figma의 A11y 플러그인, WebAIM Contrast Checker로 측정한다.
Relative Luminance
상대 밝기
WCAG가 대비율 계산에 사용하는 색상 밝기 측정값. 선형 RGB 값에서 계산한다. 순수 흰색은 1.0, 순수 검정은 0. 두 밝기 값으로 대비율 = (L1 + 0.05) / (L2 + 0.05).
Color Independence
색상 독립성
정보를 색상만으로 전달하지 않아야 한다는 원칙(WCAG 1.4.1 A). 오류 상태는 빨간색 + 아이콘 + 텍스트를 함께 사용해야 한다.

4. ARIA — 접근 가능한 리치 인터넷 애플리케이션

ARIA(Accessible Rich Internet Applications)는 W3C 표준으로, HTML의 의미론적 정보가 부족할 때 스크린 리더 같은 보조 기술이 UI를 올바르게 해석하도록 속성을 추가하는 방법을 정의한다.

첫 번째 규칙: ARIA를 쓰기 전에 먼저 올바른 HTML 시맨틱 요소를 사용하라. <button> 대신 <div role="button">을 쓰는 것은 나쁜 패턴이다.

ARIA의 3가지 구성 요소

구분 의미 예시
Role (역할) 요소의 유형·목적을 정의. 보조 기술이 "이게 뭔지" 이해하게 한다 role="dialog", role="alert", role="tablist"
Property (속성) 요소의 성질을 설명하는 변하지 않는(또는 거의 변하지 않는) 정보 aria-label="닫기", aria-required="true", aria-describedby
State (상태) 사용자 인터랙션에 따라 동적으로 변하는 현재 상태 aria-expanded="false", aria-checked="true", aria-disabled="true"

자주 쓰는 ARIA 패턴

레이블 제공 aria-label="검색" 아이콘 버튼처럼 텍스트가 없을 때
설명 연결 aria-describedby="hint-1" 입력 필드에 추가 설명 연결
라이브 알림 aria-live="polite" 동적으로 바뀌는 메시지 영역
숨김 처리 aria-hidden="true" 장식용 아이콘 등 AT에서 제외
현재 위치 aria-current="page" 현재 활성화된 네비게이션 링크

5. 포커스 관리(Focus Management)

키보드 또는 스위치 기기로 탐색하는 사용자는 포커스(focus)를 통해 UI를 이동한다. 포커스 관리를 잘못하면 화면은 열렸지만 키보드 사용자는 막힌 상태가 되거나, 포커스가 숨겨진 요소로 이동하는 "포커스 트랩(focus trap)" 오류가 발생한다.

Focus Order
포커스 순서
Tab 키로 이동할 때의 순서. DOM 소스 순서와 시각적 순서가 일치해야 한다(WCAG 1.3.2, 2.4.3). tabindex로 순서를 강제하는 것은 유지보수 문제를 유발하므로 지양한다.
Focus Visible
포커스 가시성
키보드 포커스가 어디 있는지 시각적으로 보여야 한다(WCAG 2.4.7 AA, 2.4.11 AA). 기본 아웃라인을 outline: none으로 제거하면 이를 위반한다.
Focus Trap
포커스 트랩
모달이나 드로어가 열렸을 때 포커스가 그 내부에만 순환하도록 제한하는 패턴. 의도적 포커스 트랩은 모달에서 필수이며, 비의도적 포커스 트랩은 접근성 오류다.
Skip Link
건너뛰기 링크
반복되는 네비게이션을 건너뛰고 본문으로 바로 이동하는 숨겨진 링크. WCAG 2.4.1(A) 기준을 충족하며 페이지 상단에 숨겨두다가 Tab 시 나타나도록 구현한다.
Screen Reader
스크린 리더
화면 콘텐츠를 음성이나 점자로 변환해 주는 보조 기술. 대표 도구: macOS VoiceOver, NVDA(Windows, 무료), TalkBack(Android). 시맨틱 HTML과 올바른 ARIA가 없으면 정보를 올바르게 전달하지 못한다.
Keyboard Navigation
키보드 탐색
마우스 없이 Tab·Shift+Tab·Enter·Space·Arrow 키로만 모든 기능을 이용할 수 있어야 한다(WCAG 2.1 A). 커스텀 드롭다운·슬라이더·메뉴는 WAI-ARIA Authoring Practices의 키보드 패턴을 따른다.
