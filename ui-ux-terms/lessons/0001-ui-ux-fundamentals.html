1. UI와 UX — 무엇이 다른가?

가장 자주 혼동되는 두 용어부터 정리한다. UI(User Interface)는 사용자가 제품과 접촉하는 표면이고, UX(User Experience)는 그 접촉 전체를 통해 사용자가 느끼는 경험이다.

UI — User Interface
사용자 인터페이스
사용자가 제품과 상호작용하는 시각적·물리적 요소의 총합. 버튼, 색상, 타이포그래피, 아이콘, 레이아웃 등이 여기 속한다.

"UI는 식당의 인테리어다."
UX — User Experience
사용자 경험
사용자가 제품을 발견하고, 사용하고, 이탈하기까지 전 과정에서 쌓이는 인식·감정·행동의 총합. UI는 UX의 한 구성요소다.

"UX는 식당을 예약하고 음식을 먹고 집에 돌아가는 전체 경험이다."
비교 기준 UI UX
범위 시각·인터랙션 레이어 사용자 여정 전체
질문 "이 버튼이 예쁜가? 클릭하기 쉬운가?" "사용자가 목표를 달성하는가? 만족하는가?"
산출물 컴포넌트, 스타일 가이드, 목업 플로우 다이어그램, 사용자 여정 지도, 사용성 테스트 결과
담당자 UI 디자이너 UX 디자이너 / UX 리서처
핵심 공식: 훌륭한 UI + 나쁜 UX = 예쁘지만 쓰기 불편한 앱. 나쁜 UI + 훌륭한 UX = 못생겼지만 목적은 잘 달성하는 앱. 진짜 목표는 둘 다 잘 하는 것이다.

2. 설계 산출물 3단계 — 와이어프레임 · 목업 · 프로토타입

아이디어가 실제 제품이 되기까지 보통 세 단계를 거친다. 각 단계는 목적이 다르고, 쓰는 도구와 요구 충실도(fidelity)도 다르다.

Wireframe
와이어프레임
페이지 레이아웃과 요소 배치를 선(wire)과 상자로만 표현한 저충실도 스케치. 색상·이미지·폰트 없이 구조만 보여 준다.

목적: 정보 구조와 화면 흐름을 빠르게 검증.
도구: 손 스케치, Balsamiq, Figma(저충실도 모드)
Mockup
목업
와이어프레임에 색상·폰트·이미지를 입힌 고충실도 정적 시각물. 실제처럼 보이지만 클릭이 동작하지 않는다.

목적: 시각 디자인 검토, 이해관계자 승인.
도구: Figma, Sketch, Adobe XD
Prototype
프로토타입
실제로 클릭·탭할 수 있는 인터랙티브 모형. 코드 없이 만들기도 하고, 실제 코드로 만들기도 한다. 사용자 테스트에 활용한다.

목적: 사용 흐름 검증, 사용성 테스트.
도구: Figma(프로토타입 모드), ProtoPie, Framer
단계 충실도(Fidelity) 인터랙션 주요 목적
Wireframe 낮음 (Low) 없음 구조·배치 빠른 검토
Mockup 높음 (High) 없음 (정적) 시각 디자인 확정
Prototype 다양 (Low~High) 있음 (인터랙티브) 흐름 검증 / 사용성 테스트
실무 팁: "목업 만들었어요"라고 했을 때 클릭이 되지 않는다면 정상이다. 클릭이 된다면 그것은 프로토타입이다. 헷갈리면 "정적 목업인가요, 인터랙티브 프로토타입인가요?"라고 확인하자.
추천 1차 출처
UX vs. UI: What's the Difference? — Nielsen Norman Group

UI와 UX 구분에 대한 NN/g의 공식 아티클. 개념 정의와 예시가 명확하다. 와이어프레임·목업·프로토타입 비교는 같은 사이트의 Wireframe vs. Mockup vs. Prototype를 함께 읽자.
