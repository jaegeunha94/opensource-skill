이 레슨에서 배울 것

1. Prototype(프로토타입)이란

프로토타입(Prototype)은 최종 제품이 구현되기 전에 아이디어·인터랙션· 사용성을 검증하기 위해 만드는 시뮬레이션 가능한 시제품이다. "클릭하면 무슨 일이 일어나는가"를 실제 코드 없이 보여 줄 수 있다.

핵심 목적: 아이디어를 빠르게 실체화해서 사용자·이해관계자와 공유하고 피드백을 얻는 것. "만들기 전에 실패하라(Fail early, fail cheap)"는 원칙이 프로토타이핑의 철학이다.

산출물 스펙트럼 — Wireframe → Mockup → Prototype

산출물 정의 상호작용 주요 목적
Wireframe (와이어프레임) 레이아웃과 콘텐츠 배치를 흑백 선으로 표현한 구조 설계도 없음 정보 구조·레이아웃 검토
Mockup (목업) 실제 색상·타이포·이미지를 적용한 정적 시각 디자인 없음 비주얼 디자인 검토
Prototype (프로토타입) 화면 전환·클릭·입력 등 인터랙션을 시뮬레이션하는 시제품 있음 사용성 테스트·이해관계자 공유

2. 충실도(Fidelity) 스펙트럼

프로토타입의 충실도(Fidelity)란 최종 제품과 얼마나 가까운지를 나타내는 척도다. 낮은 충실도(Lo-Fi)는 빠르고 저렴하지만 추상적이고, 높은 충실도(Hi-Fi)는 현실감이 높지만 제작 비용이 크다.

Lo-Fi
스케치·종이
흑백·낮은 디테일
빠른 반복
→
Mid-Fi
디지털 와이어프레임
기본 인터랙션
그레이스케일
→
Hi-Fi
실제 디자인·컬러
전체 인터랙션
실제 콘텐츠
Visual Fidelity
시각적 충실도
색상·타이포·이미지 등 비주얼이 최종 제품과 얼마나 가까운가. Lo-Fi는 흑백 선, Hi-Fi는 완성 디자인.
Functional Fidelity
기능적 충실도
인터랙션·상태 변화·에러 처리 등 기능이 얼마나 구현되어 있는가. 클릭 가능한 버튼부터 실데이터 연동까지 범위가 다양하다.
Content Fidelity
콘텐츠 충실도
Lorem Ipsum 가짜 텍스트 vs 실제 콘텐츠. 실제 문구를 사용하면 더 현실적인 사용성 테스트가 가능하다.

3. 주요 프로토타입 유형

✏️
Lo-Fi Prototype
로우파이 프로토타입 · 저충실도 프로토타입
화이트보드 스케치나 종이로 만든 매우 단순한 프로토타입이다. 색상·타이포·이미지 없이 기능과 레이아웃에만 집중한다. 아이디어를 빠르게 검증하고 버리는 초기 발산 단계에 적합하다. 팀원 누구나 만들 수 있어 디자이너에 대한 의존을 줄인다.
제작 빠름 비용 낮음 초기 아이디어 검증
📄
Paper Prototype
페이퍼 프로토타입 · 종이 프로토타입
Lo-Fi의 한 형태로, 실제 종이에 화면을 그리고 오려 붙여 만드는 프로토타입이다. 진행자가 사용자의 입력에 맞춰 종이 조각을 수동으로 교체해 가며 인터랙션을 시뮬레이션한다. 도구 없이 즉시 만들 수 있어 브레인스토밍 직후 빠른 사용성 테스트에 자주 쓰인다.
도구 불필요 재료비 0원 팀 워크숍·브레인스토밍 직후
🖥️
Hi-Fi Prototype
하이파이 프로토타입 · 고충실도 프로토타입
최종 제품과 시각적으로 거의 동일하며 실제 인터랙션을 포함하는 프로토타입이다. Figma · Adobe XD · Framer 같은 디자인 툴로 제작하며, 실제 색상·타이포·이미지·트랜지션 애니메이션을 포함한다. 이해관계자 프레젠테이션, 개발 핸드오프 직전 사용성 테스트에 적합하다.
제작 시간 소요 비용 높음 이해관계자 검토·최종 사용성 테스트
🖱️
Clickable Prototype
클릭어블 프로토타입 · 인터랙티브 프로토타입
버튼·링크를 클릭하면 다음 화면으로 전환되는 인터랙션이 연결된 프로토타입이다. Mid-Fi 수준의 와이어프레임에 화면 전환만 연결한 단순한 형태부터, Hi-Fi 디자인에 애니메이션까지 포함한 고급 형태까지 범위가 넓다. Figma의 "Prototype 탭"이나 InVision으로 흔히 제작한다.
빠른 인터랙션 연결 사용 흐름(Task Flow) 검증
🧙
Wizard of Oz Prototype
오즈의 마법사 프로토타입
AI · 음성 인식 · 복잡한 알고리즘처럼 구현 비용이 높은 기능을 사람이 뒤에서 수동으로 조작해 마치 구현된 것처럼 시뮬레이션하는 기법이다. 사용자는 실제로 작동하는 시스템이라고 믿는다. "오즈의 마법사"처럼 커튼 뒤에서 사람이 조종한다는 의미에서 이름이 붙었다. 초기 AI 챗봇이나 음성 검색 UI 검증에 자주 활용된다.
AI·음성·복잡 알고리즘 조기 검증 개발 없이 가능

4. Paper Prototype — 어떻게 생겼나?

아래는 모바일 앱 로그인 화면을 종이로 스케치한 Lo-Fi Prototype의 예시다.

┌─────────────────────────────┐ │ ☰ App Name │ ← 상단 바 ├─────────────────────────────┤ │ │ │ [LOGO / 앱 아이콘] │ │ │ │ ┌─────────────────────┐ │ │ │ 이메일 입력 │ │ ← 텍스트 필드 │ └─────────────────────┘ │ │ │ │ ┌─────────────────────┐ │ │ │ 비밀번호 입력 │ │ ← 텍스트 필드 │ └─────────────────────┘ │ │ │ │ [ 로그인 버튼 (CTA) ] │ ← 주 버튼 │ │ │ 소셜 로그인 옵션 │ │ [G] [f] [애플] │ │ │ │ 회원가입 | 비번 찾기 │ └─────────────────────────────┘
Lo-Fi의 장점: 디자인이 "미완성처럼 보일수록" 사용자가 비주얼에 집중하지 않고 기능에 솔직한 피드백을 준다. Nielsen Norman Group은 이를 "낮은 완성도가 더 좋은 피드백을 유도한다"고 설명한다.

5. Lo-Fi vs Hi-Fi — 언제 무엇을 선택하나?

Lo-Fi 를 선택하는 경우
  • 초기 아이디어 발산 단계
  • 여러 컨셉을 빠르게 비교해야 할 때
  • 예산·시간이 제한적일 때
  • 비주얼보다 기능·흐름 검증이 목적일 때
  • 팀 내부 워크숍·스케치 세션
Hi-Fi 를 선택하는 경우
  • 이해관계자·투자자 프레젠테이션
  • 비주얼 디자인을 포함한 사용성 테스트
  • 개발 핸드오프 직전 최종 검증
  • 마케팅 데모 영상 제작
  • 신뢰도가 높은 사용자 테스트 결과가 필요할 때

프로토타입 선택 기준 요약

기준 Lo-Fi (Paper·Sketch) Mid-Fi (Clickable) Hi-Fi (Figma 인터랙티브)
제작 시간 수 분~수 시간 수 시간~1일 수 일~수 주
비용 매우 낮음 낮음~중간 높음
반복 속도 매우 빠름 빠름 느림
피드백 유형 기능·흐름 흐름·레이아웃 전반적 경험
적합한 단계 발산(Ideation) 수렴(Refinement) 검증·핸드오프

6. 프로토타이핑 프로세스

실무에서 프로토타입은 한 번에 Hi-Fi로 바로 가지 않는다. 충실도를 점차 높여 가며 각 단계에서 다른 질문에 답한다.

① 종이 스케치 (Paper Prototype)
"이 아이디어가 말이 되는가?" — 5분 만에 여러 컨셉을 그리고, 팀 내부에서 피드백을 주고받는다.
② Lo-Fi 디지털 와이어프레임
"화면 흐름이 논리적인가?" — Figma · Balsamiq 등으로 흑백 와이어프레임을 그리고, 화면 전환 링크를 연결한다.
③ Clickable Prototype (Mid-Fi)
"사용자가 태스크를 완료할 수 있는가?" — 실제 사용자에게 클릭해 보게 하면서 사용성 문제를 발견한다.
④ Hi-Fi Prototype
"전체 경험이 만족스러운가?" — 최종 비주얼을 적용하고, 이해관계자 승인 후 개발 핸드오프로 이동한다.

7. 핵심 용어 정리

Fidelity
충실도
프로토타입이 최종 제품과 얼마나 가까운지를 나타내는 척도. 시각적·기능적·콘텐츠 충실도로 세분된다.
Lo-Fi Prototype
로우파이 프로토타입
낮은 충실도의 프로토타입. 종이 스케치·흑백 와이어프레임 수준으로 빠르고 저렴하게 제작한다.
Hi-Fi Prototype
하이파이 프로토타입
최종 제품과 거의 동일한 수준의 프로토타입. Figma 등 디자인 툴로 제작하며 전체 인터랙션을 포함한다.
Paper Prototype
페이퍼 프로토타입
Lo-Fi의 한 형태. 종이에 화면을 그리고 오려 붙여 인터랙션을 수동으로 시뮬레이션한다.
Clickable Prototype
클릭어블 프로토타입
버튼·링크에 화면 전환 인터랙션이 연결된 프로토타입. Mid-Fi~Hi-Fi 수준에서 주로 사용한다.
Wizard of Oz
오즈의 마법사 프로토타입
구현이 어려운 기능을 사람이 수동으로 흉내 내어 시뮬레이션하는 프로토타이핑 기법.
Interactive Prototype
인터랙티브 프로토타입
Clickable Prototype과 혼용되며, 특히 애니메이션·상태 변화·제스처 인터랙션까지 포함한 높은 수준의 프로토타입을 가리킬 때 쓴다.
Throwaway Prototype
일회용 프로토타입
검증 후 폐기하는 것을 전제로 만드는 프로토타입. Lo-Fi가 대표적이다. 코드로 빌드하는 Throwaway Prototype도 있다.
