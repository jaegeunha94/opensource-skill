컴퓨터 구조 용어 사전

이 사전은 학습이 진행되면서 계속 확장됩니다. 개념을 처음 배울 때가 아니라, 충분히 이해한 후에 정의가 추가됩니다.

A

ALU (Arithmetic Logic Unit, 산술논리장치)
CPU 내부에서 실제 계산(덧셈, 뺄셈, AND, OR, 비교 등)을 수행하는 회로.
피하는 표현: 연산기, 계산기 유닛

Architecture (컴퓨터 구조)
프로그래머에게 보이는 컴퓨터의 속성 집합 — ISA, 레지스터 수, 메모리 모델, 데이터 타입.
피하는 표현: 하드웨어 구조 (마이크로아키텍처와 혼동 주의)

B

Bus (버스)
CPU, 메모리, I/O 장치 사이의 공유 통신 경로. 주소 버스, 데이터 버스, 제어 버스로 구성됨.

C

Control Unit (제어장치)
명령어를 해석해서 ALU, 레지스터, 메모리 등 각 구성 요소에 제어 신호를 전달하는 CPU 내부 회로.
비유: 오케스트라 지휘자

CPU (Central Processing Unit, 중앙처리장치)
명령어를 인출·해독·실행하는 컴퓨터의 핵심 처리 장치. ALU, 제어장치, 레지스터로 구성됨.

F

Fetch–Decode–Execute Cycle (명령어 사이클)
CPU가 반복하는 기본 동작 단위: 메모리에서 명령어 인출 → 해독 → 실행 → 결과 저장.
피하는 표현: 실행 사이클 (Fetch가 빠지면 의미 손실)

I

ISA (Instruction Set Architecture, 명령어 집합 아키텍처)
CPU가 이해하는 명령어 집합과 그 의미 — 소프트웨어와 하드웨어 사이의 계약.
예: x86-64, ARMv8, RISC-V

M

Micro-architecture (마이크로아키텍처)
동일한 ISA를 구현하는 구체적 하드웨어 설계 — 파이프라인 깊이, 캐시 크기, 분기 예측기 등.
예: Intel의 Alder Lake와 AMD의 Zen 4는 같은 x86-64 ISA를 다른 마이크로아키텍처로 구현함

R

Register (레지스터)
CPU 내부의 초고속 임시 저장 공간. 보통 수십 개(32~512비트 너비)가 존재하며, RAM보다 수백 배 빠름.

V

Von Neumann Architecture (폰 노이만 아키텍처)
프로그램 코드와 데이터를 같은 메모리에 저장하는 컴퓨터 설계 모델. 1945년 존 폰 노이만 제안. 현대 범용 컴퓨터의 기반.
피하는 표현: 폰 노이만 머신

Von Neumann Bottleneck (폰 노이만 병목)
CPU와 메모리 사이의 단일 버스로 인해 명령어 인출과 데이터 접근이 동시에 일어날 수 없는 구조적 한계.
