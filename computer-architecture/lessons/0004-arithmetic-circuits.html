1. 뺄셈 회로 — 2의 보수의 진가

Day 2에서 2의 보수를 배웠습니다. 2의 보수의 가장 큰 실용적 가치는 뺄셈을 덧셈 회로 하나로 처리할 수 있다는 것입니다. CPU는 별도의 뺄셈 회로를 갖지 않고, 가산기 하나로 덧셈과 뺄셈을 모두 수행합니다.

1-1. 원리: A − B = A + (−B)

2의 보수로 −B 만들기: 1단계: B의 각 비트를 XOR(NOT) 반전 → B̄ (1의 보수) 2단계: +1 더하기 → B̄ + 1 = −B (2의 보수) 따라서: A − B = A + B̄ + 1 회로 구현: ① B의 각 비트를 XOR 게이트로 조건부 반전 (선택 신호 Sub=0: B 통과 → 덧셈 Sub=1: B 반전 → 뺄셈 준비) ② 리플 캐리 가산기 최하위 Cin에 Sub 신호를 연결 (Sub=1일 때 Cin=1 → "+1" 자동 처리)

1-2. 덧셈/뺄셈 겸용 회로

Sub=0 (덧셈): B₃ B₂ B₁ B₀ 그대로 전달, Cin = 0 Sub=1 (뺄셈): B₃'B₂'B₁'B₀' 반전 전달, Cin = 1 A₃B₃ A₂B₂ A₁B₁ A₀B₀ ⊕Sub ⊕Sub ⊕Sub ⊕Sub ← XOR로 조건부 반전 │ │ │ │ [FA] ← [FA] ← [FA] ← [FA] ← Cin = Sub │ │ │ │ S₃ S₂ S₁ S₀ (+ 최종 Cout: 오버플로우 감지) 게이트 추가 비용: XOR 게이트 n개 + Sub 신호선 1개
오버플로우 감지
부호 있는 정수 연산에서 결과가 표현 범위를 벗어나면 오버플로우입니다.
감지 규칙: 최상위 비트(MSB) 자리로 들어오는 Carry와 나가는 Carry가 서로 다르면 오버플로우.
Overflow = Carry_in(MSB) XOR Carry_out(MSB)
CPU의 상태 레지스터(Status Register)에 V(Overflow) 플래그로 기록됩니다.

2. 비교기 (Comparator)

비교기는 두 수 A, B의 크기 관계(=, <, >)를 회로로 판별합니다. 조건 분기(if, jmp)를 처리하는 CPU에서 필수 회로입니다.

2-1. 동일 비교 (Equality)

A = B 판별: 모든 비트 쌍이 같아야 한다 → 각 비트 쌍에 XNOR 적용 (같으면 1) → 결과를 모두 AND Equal = XNOR(A₃,B₃) · XNOR(A₂,B₂) · XNOR(A₁,B₁) · XNOR(A₀,B₀) XNOR 진리표: A B │ XNOR ─────┼────── 0 0 │ 1 (같음) 0 1 │ 0 (다름) 1 0 │ 0 (다름) 1 1 │ 1 (같음)

2-2. 크기 비교 (A < B, A > B)

방법 1 — 뺄셈 이용: A − B를 계산하고 결과의 부호 비트(MSB)를 본다 MSB = 1 → A − B < 0 → A < B MSB = 0, 결과≠0 → A > B 결과 = 0 → A = B 방법 2 — MSB 우선 비교 (직접 비교기): 가장 높은 비트부터 순서대로 비교: Aᵢ = 0, Bᵢ = 1 → 즉시 A < B 확정 Aᵢ = 1, Bᵢ = 0 → 즉시 A > B 확정 Aᵢ = Bᵢ → 다음 낮은 비트로 이동 → 4비트 비교기: 4단계 비교 논리를 게이트로 병렬 구현
CPU 플래그 레지스터와의 연결
비교 연산(CMP 명령어)은 내부적으로 뺄셈을 실행하되 결과를 저장하지 않고 플래그만 업데이트합니다.
  • Z(Zero): 결과 = 0 → A = B
  • N(Negative): MSB = 1 → A < B (부호 있는)
  • C(Carry): Cout = 1 → A < B (부호 없는)
  • V(Overflow): 부호 있는 오버플로우 발생
조건 분기 명령어(BEQ, BLT 등)는 이 플래그를 읽어 분기를 결정합니다.

3. 올림 예측 가산기 (Carry-Lookahead Adder, CLA)

Day 3의 리플 캐리 가산기는 올림(Carry)이 LSB부터 MSB까지 순차적으로 전파되어 n비트일 때 지연이 O(n)입니다. 32비트라면 32단계를 기다려야 합니다. CLA는 수학적 공식으로 모든 자리의 Carry를 동시에 미리 계산해 지연을 O(log n)으로 줄입니다.

3-1. Generate와 Propagate

자리 i에서 두 가지 신호를 정의: Gᵢ (Generate, 생성): Aᵢ · Bᵢ → Cin과 무관하게 이 자리가 Carry를 "생성" → 1·1 = 1 이므로 무조건 올림 발생 Pᵢ (Propagate, 전파): Aᵢ ⊕ Bᵢ → 이 자리는 Cin이 1이면 Carry를 "전파" → A=0,B=1 또는 A=1,B=0 → Cin이 있으면 올림 전달 올림 식: C₁ = G₀ + P₀·C₀ C₂ = G₁ + P₁·C₁ = G₁ + P₁·G₀ + P₁·P₀·C₀ C₃ = G₂ + P₂·C₂ = G₂ + P₂·G₁ + P₂·P₁·G₀ + P₂·P₁·P₀·C₀ C₄ = G₃ + P₃·G₂ + P₃·P₂·G₁ + P₃·P₂·P₁·G₀ + P₃·P₂·P₁·P₀·C₀ 모든 Carry가 C₀(초기 올림)와 G, P만으로 표현됨 → G와 P는 A, B로부터 1단계 지연 → 모든 Carry는 추가 2단계 지연 (AND-OR) → 합계 Sum = Pᵢ ⊕ Cᵢ로 1단계 더 총 지연: 4단계 (비트 수와 무관!)

3-2. 리플 캐리 vs CLA 비교

항목리플 캐리 가산기CLA 가산기
지연 O(n) — 비트마다 1단계 추가 O(log n) — 계층적 구성 시
게이트 수 적음 (FA n개) 많음 (G/P 계산 + 올림 네트워크)
면적 작음 큼
실제 사용 소형 임베디드, 교육용 범용 CPU ALU
현대 CPU의 실제 가산기
실제 고성능 CPU는 접두사 가산기(Prefix Adder) 계열 (Kogge-Stone, Brent-Kung, Han-Carlson)을 사용합니다. 이는 CLA 개념을 계층적 트리로 확장한 것으로, 지연과 면적 사이의 균형을 최적화합니다. 하지만 기본 원리는 모두 Generate/Propagate 개념에 기반합니다.

4. ALU — 산술논리 연산 장치

ALU(Arithmetic Logic Unit)는 CPU에서 실제 연산을 담당하는 핵심 회로입니다. 가산기, 비교기, 논리 게이트를 하나로 묶고 제어 신호(연산 코드)로 원하는 연산을 선택합니다.

4-1. ALU의 구성

A [n비트] B [n비트] │ │ ┌────┴───────────┴────┐ │ 산술 유닛 │ 논리 유닛 │ ┌──────────────┐ │ ┌──────────────┐ │ │ 가산기/뺄셈기 │ │ │ AND │ │ │ (CLA 기반) │ │ │ OR │ │ └──────────────┘ │ │ XOR │ │ │ │ NOT │ │ 시프트 유닛 │ └──────────────┘ │ ┌──────────────┐ │ │ │ 논리/산술 │ │ │ │ 좌/우 시프트 │ │ │ └──────────────┘ │ └─────────┬───────────┘ │ 여러 연산 결과 ┌────┴────┐ │ MUX │ ← ALU Op 코드 (연산 선택) └────┬────┘ │ Result [n비트] 플래그 출력: Zero, Carry, Overflow, Negative

4-2. 연산 코드 예시 (간단한 4비트 ALU)

Op 코드연산설명
000A + B덧셈
001A − B뺄셈 (2의 보수)
010A AND B비트 AND
011A OR B비트 OR
100A XOR B비트 XOR
101NOT A비트 반전
110A << 1논리 좌시프트 (× 2)
111A >> 1논리 우시프트 (÷ 2)

4-3. MUX(멀티플렉서)로 연산 선택

멀티플렉서(MUX): n개 입력 중 1개를 선택 신호에 따라 출력하는 회로 2-to-1 MUX: 입력: D₀, D₁, 선택 신호 S 출력: Y = S'·D₀ + S·D₁ S=0 → D₀ 선택 S=1 → D₁ 선택 4-to-1 MUX (선택 신호 2비트 S₁S₀): 00 → D₀, 01 → D₁, 10 → D₂, 11 → D₃ ALU는 각 유닛(가산기·논리·시프트)의 결과를 MUX 입력으로 받고, Op 코드를 MUX 선택 신호로 써서 정확한 결과를 라우팅합니다.
시프트 연산과 곱셈
곱셈 하드웨어는 비싸므로 2의 거듭제곱 곱셈은 시프트로 최적화합니다.
  • x << 1 → x × 2 (좌시프트)
  • x << 3 → x × 8
  • x >> 1 → x ÷ 2 (부호 있는: 산술 시프트, 부호 없는: 논리 시프트)
컴파일러는 상수 곱셈을 시프트 + 덧셈 조합으로 자동 변환합니다.
예: x × 5 = x × 4 + x = (x << 2) + x

4-4. ALU가 CPU 데이터패스에서 하는 역할

단순화된 단일 사이클 데이터패스: [명령어 메모리] → 명령어 디코더 → ALU Op 코드 │ │ [레지스터 파일] → A, B ────────────→ [ALU] → 결과 → [레지스터 파일] │ [데이터 메모리] (Load/Store 시) - ALU의 연산 결과는 레지스터에 쓰거나(산술/논리 명령) 메모리 주소로 사용됩니다(Load/Store 명령). - 분기 명령에서는 ALU가 A−B를 계산해 Zero 플래그로 분기 여부 결정.

오늘의 핵심 요약

  • 뺄셈 = 덧셈: A − B = A + B̄ + 1. B를 XOR로 반전하고 Cin=1 주입, 별도 회로 불필요
  • 오버플로우: MSB Carry-in ≠ Carry-out이면 발생. V 플래그로 표시
  • 비교기: 뺄셈 결과의 플래그(Z, N, C)로 =, <, > 판별. CMP 명령어가 이 방식
  • CLA: Generate(G=A·B)와 Propagate(P=A⊕B)로 올림을 미리 계산. 지연 O(log n)
  • ALU: 가산기·논리·시프트 유닛을 MUX로 묶어 Op 코드로 연산 선택. CPU의 연산 엔진
  • 시프트 최적화: 2의 거듭제곱 곱셈은 컴파일러가 시프트+덧셈으로 변환해 ALU 연산 최소화
