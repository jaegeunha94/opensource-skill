오늘 배울 것

1. 진법 체계 (Positional Number Systems)

컴퓨터의 모든 데이터는 결국 0과 1의 나열로 저장됩니다. 사람은 10진수에 익숙하지만, 하드웨어는 전압의 높고 낮음(on/off)으로 값을 표현하므로 2진수가 가장 자연스럽습니다.

진법 기저(Base) 사용 숫자 주요 쓰임새
2진수 (Binary) 2 0, 1 CPU 내부 연산·메모리 저장
8진수 (Octal) 8 0–7 Unix 파일 권한(chmod 755)
16진수 (Hexadecimal) 16 0–9, A–F 메모리 주소·색상값·기계어 덤프
10진수 (Decimal) 10 0–9 사람이 읽는 숫자

1-1. 위치값(Positional Value) 원리

어떤 진법이든 자릿수 위치에 따라 기저의 거듭제곱을 곱합니다.

10진수 → 1 3 7 = 1×10² + 3×10¹ + 7×10⁰ = 100 + 30 + 7 = 137 2진수 → 1 0 1 1 = 1×2³ + 0×2² + 1×2¹ + 1×2⁰ = 8 + 0 + 2 + 1 = 11 (10진수) 16진수 → 0x1F = 1×16¹ + 15×16⁰ = 16 + 15 = 31 (10진수)

1-2. 진법 변환 요령

변환 방향방법예시
10진수 → 2진수 2로 반복 나눗셈, 나머지를 역순 기록 25 → 11001₂
2진수 → 16진수 오른쪽부터 4비트씩 묶어 16진 자리로 변환 1010 1111₂ → AF₁₆
16진수 → 2진수 각 16진 자리를 4비트 이진수로 전개 3C₁₆ → 0011 1100₂
실무 팁
2진수 ↔ 16진수 변환은 4비트 묶음 덕분에 매우 쉽습니다. 메모리 덤프·기계어 출력 도구가 모두 16진수를 사용하는 이유입니다. 0x 접두사는 "이 숫자는 16진수"를 의미하는 C/C++ 표기법입니다.

2. 부호 없는 정수 (Unsigned Integer)

n비트 부호 없는 정수는 0부터 2ⁿ−1까지 표현합니다.

8비트 (1바이트) → 0 ~ 255 (2⁸−1) 16비트 → 0 ~ 65,535 32비트 → 0 ~ 4,294,967,295 (약 42억) 64비트 → 0 ~ 18,446,744,073,709,551,615 (약 1.8×10¹⁹)

오버플로우 (Overflow)

최대값에서 1을 더하면 모든 비트가 0으로 돌아갑니다. 이것이 오버플로우입니다.

8비트 예: 1111 1111 (255) + 0000 0001 (+1) ────────── 1 0000 0000 → 최상위 비트 캐리 버림 → 0000 0000 (0)

3. 2의 보수 (Two's Complement)

음수를 표현하는 방법은 여러 가지가 있지만, 현대 컴퓨터는 거의 예외 없이 2의 보수를 사용합니다.

왜 2의 보수인가?

방식장점단점
부호 크기(Sign-Magnitude) 직관적 +0, −0 두 가지 존재; 덧셈 회로 별도 필요
1의 보수(One's Complement) 비트 반전으로 간단 역시 +0, −0 두 가지 존재
2의 보수(Two's Complement) 0이 하나; 덧셈·뺄셈 회로 공유 양수보다 음수 표현 범위가 1 더 넓음

2의 보수 계산 방법

방법: 비트를 전부 반전(NOT) 한 뒤 1을 더한다. 예) 8비트에서 −5 구하기 5 → 0000 0101 반전 → 1111 1010 +1 → 1111 1011 ← 이것이 −5의 2의 보수 표현

8비트 부호 있는 정수 범위

최솟값: 1000 0000 = −128 최댓값: 0111 1111 = +127 범위: −2^(n−1) ~ 2^(n−1)−1 → −128 ~ 127 (8비트)
왜 덧셈 회로 하나로 충분한가
2의 보수 체계에서 뺄셈은 음수의 덧셈입니다. 예: 7 − 5 = 7 + (−5) = 0000 0111 + 1111 1011 = 1 0000 0010 → 캐리 버림 → 0000 0010 = 2 ✓
덧셈 회로 하나(ALU의 가산기)만으로 부호 있는 덧셈과 뺄셈 모두 처리할 수 있어 하드웨어가 단순해집니다.

부호 확장 (Sign Extension)

8비트 값을 16비트 레지스터로 옮길 때 최상위 비트(부호 비트)를 복사해서 채웁니다. 이를 부호 확장(sign extension)이라 합니다.

8비트 −5 → 1111 1011 16비트 −5 → 1111 1111 1111 1011 (앞을 1로 채움) 8비트 +5 → 0000 0101 16비트 +5 → 0000 0000 0000 0101 (앞을 0으로 채움)

4. IEEE 754 부동소수점 (Floating-Point)

정수만으로는 3.14나 0.001 같은 소수를 표현할 수 없습니다. IEEE 754는 부동소수점 수를 저장하는 국제 표준입니다.

형식: 부호 · 지수 · 가수

32비트 단정밀도(Single Precision) float: ┌─┬──────────┬───────────────────────┐ │S│ Exponent│ Mantissa │ │1│ 8 bit │ 23 bit │ └─┴──────────┴───────────────────────┘ S = 부호 비트 (0: 양수, 1: 음수) Exponent= 지수 (bias=127로 이동) Mantissa= 가수 (소수점 아래, 암묵적 선행 1) 값 = (−1)^S × 1.Mantissa × 2^(Exponent−127)
타입총 비트부호지수가수대략 정밀도
float (단정밀도) 32 1 8 23 유효 숫자 약 7자리
double (배정밀도) 64 1 11 52 유효 숫자 약 15~16자리

0.1 + 0.2 ≠ 0.3 이유

10진수 0.1은 2진 부동소수점으로 정확히 표현할 수 없습니다. 무한 반복 소수(0.000110011001100…₂)이기 때문에 23비트에서 반올림·잘림이 발생합니다. 이것이 Python에서 0.1 + 0.2 == 0.30000000000000004가 나오는 이유입니다.

0.1 (10진) → 0.0001100110011001100110011… (2진, 무한반복) 23비트에서 잘리면 → 미세한 오차 발생 두 오차를 더하면 → 눈에 보이는 오차
실무 시사점
금융·회계 계산에는 부동소수점 대신 고정소수점이나 decimal 타입(Python의 decimal.Decimal, Java의 BigDecimal)을 사용하세요. 비교할 때는 abs(a − b) < ε 방식으로 허용 오차를 두어야 합니다.

특수값

패턴의미발생 예
지수 모두 0, 가수 모두 0±00.0 초기화
지수 모두 1, 가수 모두 0±∞ (Infinity)1.0 / 0.0
지수 모두 1, 가수 ≠ 0NaN (Not a Number)0.0 / 0.0, sqrt(−1)
지수 모두 0, 가수 ≠ 0비정규화수 (Subnormal)0에 매우 가까운 값

오늘의 핵심 요약

  • 2진수: 컴퓨터 내부 표현. 4비트 묶음 = 16진수 한 자리
  • 부호 없는 정수: n비트 → 0 ~ 2ⁿ−1. 최대값 초과 시 오버플로우로 0으로 감김
  • 2의 보수: NOT 후 +1. 뺄셈을 덧셈으로 통일 → ALU 단순화. 8비트 범위 −128 ~ 127
  • IEEE 754: 부호(1) + 지수(8) + 가수(23) = 32비트. 0.1은 정확히 표현 불가
  • NaN / ±Infinity: 특수 비트 패턴으로 예외 상황 처리
