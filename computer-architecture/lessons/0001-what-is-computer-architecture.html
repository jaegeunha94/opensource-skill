1. 컴퓨터를 보는 세 가지 관점

컴퓨터 시스템은 보통 세 층위로 나뉘어 논의됩니다. 어느 층위에서 보느냐에 따라 "컴퓨터 구조"의 의미가 달라집니다.

층위 다루는 것 대표 질문
컴퓨터 구조
(Architecture)		 프로그래머에게 보이는 기계의 속성 — ISA, 레지스터, 메모리 모델 "이 명령어로 무엇을 할 수 있는가?"
마이크로아키텍처
(Micro-architecture)		 ISA를 구현하는 구체적 설계 — 파이프라인, 캐시 크기, 분기 예측기 "이 명령어가 어떻게 실행되는가?"
디지털 로직
(Logic Design)		 게이트, 플립플롭, 트랜지스터 수준 "비트 0/1이 물리적으로 어떻게 표현되는가?"
핵심 인사이트
같은 ISA(예: x86-64)를 구현하더라도 마이크로아키텍처는 제조사마다 다릅니다. Intel Core i9과 AMD Ryzen 9은 같은 프로그램을 실행하지만, 내부 파이프라인 설계는 전혀 다릅니다. 이 과정에서는 주로 컴퓨터 구조와 마이크로아키텍처 층위를 함께 다룹니다.

2. 폰 노이만 아키텍처

현대 컴퓨터의 대부분은 1945년 존 폰 노이만(John von Neumann)이 제안한 저장 프로그램 컴퓨터(stored-program computer) 모델을 따릅니다. 핵심 아이디어는 간단합니다:

데이터와 명령어(프로그램)를 같은 메모리에 저장한다.

이 아이디어 이전에는 프로그램을 바꾸려면 기계 배선 자체를 다시 연결해야 했습니다 (에니악, ENIAC).

구성 요소

┌─────────────────────────────────────────────────────┐ │ 컴퓨터 시스템 │ │ │ │ ┌──────────────────────┐ ┌────────────────────┐ │ │ │ CPU (중앙처리장치) │ │ 메모리 (Memory) │ │ │ │ │◄──►│ │ │ │ │ ┌────────┐ │ │ 프로그램 코드 │ │ │ │ │ ALU │ 계산 │ │ + 데이터 │ │ │ │ └────────┘ │ │ (같은 공간!) │ │ │ │ ┌────────┐ │ └────────────────────┘ │ │ │ │ 제어장치│ 순서 제어 │ │ │ │ └────────┘ │ ┌────────────────────┐ │ │ │ ┌────────┐ │ │ 입출력 장치 (I/O) │ │ │ │ │레지스터│ 임시 저장 │◄──►│ 키보드, 모니터, │ │ │ │ └────────┘ │ │ 디스크 등 │ │ │ └──────────────────────┘ └────────────────────┘ │ └─────────────────────────────────────────────────────┘

그림 1 — 폰 노이만 아키텍처의 핵심 구성 요소

각 구성 요소의 역할

구성 요소역할비유
ALU
(산술논리장치)		 덧셈, 뺄셈, AND, OR 등 실제 계산 수행 계산기의 + − × ÷ 버튼
제어장치
(Control Unit)		 명령어를 읽고 해석해서 각 부품에 신호 전달 오케스트라 지휘자
레지스터
(Register)		 CPU 내부의 초고속 임시 저장소. 보통 수십 개 요리사의 양손 (지금 당장 쓸 재료)
메모리
(RAM)		 프로그램 코드와 데이터를 저장. 레지스터보다 느리지만 훨씬 큼 주방의 조리대
입출력
(I/O)		 외부 세계(사용자, 디스크, 네트워크)와 통신 주방의 출입문

3. 명령어 사이클 (Instruction Cycle)

CPU는 전원이 켜진 순간부터 꺼질 때까지 다음 사이클을 반복합니다. 이것이 컴퓨터가 "동작"하는 원리입니다.

① Fetch (인출) → 메모리에서 다음 명령어를 가져온다 ↓ ② Decode (해독) → 명령어가 무엇을 뜻하는지 해석한다 ↓ ③ Execute (실행) → ALU나 메모리 접근 등 실제 작업을 한다 ↓ ④ (Write-back) → 결과를 레지스터나 메모리에 저장한다 ↓ ↑___________다시 ①로 반복___________↑

그림 2 — 명령어 사이클 (Fetch–Decode–Execute)

왜 중요한가
파이프라이닝(Pipelining), 분기 예측(Branch Prediction), 비순서 실행(Out-of-Order Execution) 등 현대 CPU의 거의 모든 최적화 기법은 이 명령어 사이클을 더 빠르게 만들기 위한 것입니다. Day 1의 이 개념이 이후 모든 수업의 토대가 됩니다.

4. 폰 노이만 병목 (Von Neumann Bottleneck)

폰 노이만 아키텍처에는 유명한 단점이 있습니다. CPU와 메모리가 하나의 버스(Bus)로만 연결되기 때문에, 명령어를 가져오는 것과 데이터를 읽는 것이 동시에 일어날 수 없습니다.

CPU ←────── 단일 버스 ──────→ 메모리 (명령어 OR 데이터, 동시 불가)

이 병목을 완화하기 위한 기법이 바로 캐시 메모리와 하버드 아키텍처입니다. (이후 레슨에서 자세히 다룹니다.)
