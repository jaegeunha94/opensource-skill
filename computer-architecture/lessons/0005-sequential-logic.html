오늘 배울 것

  • 조합 논리 회로와 순서 논리 회로의 차이를 이해한다

  • 클럭(Clock)이 회로 동기화에 미치는 역할을 파악한다

1. 조합 논리 vs 순서 논리

Day 3~4에서 다룬 게이트·가산기·ALU는 모두 조합 논리 회로입니다. 현재 입력만으로 출력이 결정되며, 이전 상태를 기억하지 않습니다.

조합 논리 (Combinational Logic): 출력 = f(현재 입력) 기억 없음, 클럭 없음 예: 게이트, 가산기, ALU, MUX 순서 논리 (Sequential Logic): 출력 = f(현재 입력, 현재 상태) 상태(state)를 저장하고 클럭에 맞춰 갱신 예: 플립플롭, 레지스터, 카운터, FSM
왜 기억이 필요한가?
CPU는 이전 명령어의 결과를 다음 명령어에서 사용해야 합니다. 레지스터 파일에 값을 저장하고, 프로그램 카운터(PC)가 다음 실행할 명령어 주소를 기억하며, 파이프라인 스테이지 사이에 데이터가 전달되려면 모두 "기억하는 회로"가 필수입니다.

2. SR 래치 (SR Latch)

가장 단순한 기억 회로입니다. NAND 게이트 두 개를 교차 결합(cross-coupled)하면 출력이 입력에 되먹임(feedback)되어 상태를 유지합니다.

2-1. NAND SR 래치 구조

S̄ ──┬──[NAND]── Q │ │ └───────┘ (피드백) R̄ ──┬──[NAND]── Q̄ │ │ └───────┘ (피드백) (S̄: Set bar, R̄: Reset bar — 둘 다 active-low 입력)

2-2. SR 래치 진리표

S̄R̄Q(다음)Q̄(다음)설명
11Q(유지)Q̄(유지)기억 — 상태 유지
0110Set — Q를 1로 세팅
1001Reset — Q를 0으로 리셋
001*1*금지 — 정의되지 않은 상태
금지 상태가 문제인 이유
S̄=R̄=0이면 Q와 Q̄ 모두 1이 되어 보수 관계가 깨집니다. 이 상태를 해제할 때 둘 중 어느 쪽이 먼저 바뀌느냐에 따라 최종 상태가 결정되지 않는 경쟁 조건(Race Condition)이 발생합니다. SR 래치의 이 한계를 해결하기 위해 클럭과 D 플립플롭이 등장합니다.

3. D 플립플롭 (D Flip-Flop)

SR 래치의 금지 상태 문제를 해결한 회로가 D 플립플롭입니다. 입력이 D(Data) 하나뿐이고, 클럭 엣지(Clock Edge)에서만 상태가 바뀝니다.

3-1. 클럭과 엣지 트리거

클럭 신호: ┌──┐ ┌──┐ ┌──┐ 0 ───┘ └──┘ └──┘ └─── 상승 엣지(Rising Edge) ↑: 0 → 1 전환 순간 하강 엣지(Falling Edge) ↓: 1 → 0 전환 순간 엣지 트리거 플립플롭: 클럭이 특정 엣지(보통 ↑)에 도달하는 그 순간에만 D를 캡처해 Q에 저장 클럭 엣지 사이에는 D가 아무리 바뀌어도 Q는 변하지 않음

3-2. D 플립플롭 동작

CLK (엣지)DQ(다음)설명
↑ (상승)00D=0을 캡처
↑ (상승)11D=1을 캡처
유지 (엣지 아님)XQ(유지)상태 변화 없음
타이밍 다이어그램: CLK: ─┐┌─┐┌─┐┌─ D: ──┤0├─┤1├── Q: ──┤ └─┤ └── (↑에서 D 값 캡처) 실제: CLK ───┐ ┌──┐ ┌── D ─X───┤1 │0 │── Q ─Q₀──┤1─┘0─┘── ↑ ↑ 캡처 순간

3-3. 셋업/홀드 타임 (Setup/Hold Time)

올바른 캡처를 위해 D가 안정해야 하는 시간: Setup Time (t_su): CLK 상승 엣지 이전에 D가 안정 상태를 유지해야 하는 최소 시간 → 위반 시: 플립플롭이 불안정한 값을 캡처 Hold Time (t_h): CLK 상승 엣지 이후에도 D를 유지해야 하는 최소 시간 → 위반 시: 내부 래치가 안정화되기 전에 D가 바뀌어 오류 발생 타이밍: D ─────[안정 구간]───── ↑셋업 ↑홀드 CLK ─────────↑───────── 이것이 CPU 클럭 속도의 물리적 한계를 결정합니다. (클럭 주기 > 조합 논리 지연 + 셋업 타임 + 플립플롭 전파 지연)

4. 레지스터 (Register)

D 플립플롭을 n개 병렬로 묶으면 n비트 레지스터가 됩니다. CPU의 레지스터 파일(Register File)은 이 구조의 배열입니다.

4-1. n비트 레지스터 구조

4비트 레지스터: D₃ ─[FF]─ Q₃ D₂ ─[FF]─ Q₂ CLK ─┬─┬─┬─┬─ D₁ ─[FF]─ Q₁ │ │ │ │ D₀ ─[FF]─ Q₀ FF FF FF FF CLK 한 번에 D₃D₂D₁D₀ 전체가 Q₃Q₂Q₁Q₀에 동시 저장 → 4비트 값을 하나의 원자적 동작으로 캡처

4-2. Write Enable이 있는 레지스터

실제 레지스터는 매 클럭마다 갱신되면 안 됩니다. Write Enable(WE) 신호로 갱신 시점을 제어합니다. WE=0: D = Q (현재 값을 D로 되먹임 → 상태 유지) WE=1: D = 새 입력값 (클럭에 새 값 캡처) 구현: D = MUX(현재Q, 새입력, WE)

4-3. CPU 레지스터 파일

ARM Cortex-A 기준: 31개 범용 레지스터(x0~x30), 각 64비트 RISC-V 기준: 32개 범용 레지스터(x0~x31) 레지스터 파일 구조: ┌─────────────────────────────────────┐ │ 레지스터 배열 (32 × 64비트 FF) │ │ │ │ 읽기 포트 2개: rs1, rs2 주소 입력 │ │ → 조합 논리로 해당 레지스터 값 출력│ │ 쓰기 포트 1개: rd 주소 + 쓸 값 │ │ → 클럭에 맞춰 해당 레지스터 갱신 │ └─────────────────────────────────────┘ 한 사이클에 2개 읽기 + 1개 쓰기가 동시에 가능 (ALU가 두 피연산자를 읽고 결과를 다시 저장)

5. 카운터 (Counter)

카운터는 클럭 펄스마다 숫자를 1씩 증가(또는 감소)시키는 순서 회로입니다. CPU에서 프로그램 카운터(PC)가 가장 중요한 카운터입니다.

5-1. 비동기 이진 카운터 (리플 카운터)

T 플립플롭(Toggle FF: 클럭마다 상태 반전)을 직렬 연결: CLK ─[T-FF₀]─ Q₀(LSB) └─[T-FF₁]─ Q₁ └─[T-FF₂]─ Q₂ └─[T-FF₃]─ Q₃(MSB) Q₀가 1→0 될 때마다 Q₁이 토글 (Q₀의 하강 엣지가 Q₁의 클럭) → 2비트씩 체인으로 연결 카운트 순서: 0000 → 0001 → 0010 → 0011 → ... → 1111 → 0000 단점: 각 비트가 순차적으로 토글 → 일시적으로 중간 값이 나타남 (글리치)

5-2. 동기 이진 카운터

모든 플립플롭이 같은 CLK을 사용: Q₀의 D = Q₀ ⊕ 1 = NOT(Q₀) 항상 토글 Q₁의 D = Q₁ ⊕ Q₀ Q₀=1일 때 토글 Q₂의 D = Q₂ ⊕ (Q₁·Q₀) Q₁Q₀=11일 때 토글 Q₃의 D = Q₃ ⊕ (Q₂·Q₁·Q₀) Q₂Q₁Q₀=111일 때 토글 장점: 모든 비트가 같은 클럭에 동시 갱신 → 글리치 없음 → 실제 CPU PC는 동기 카운터 사용

5-3. 프로그램 카운터 (PC)

PC = 현재 실행 중인 명령어의 메모리 주소를 저장하는 레지스터 기본 동작: PC ← PC + 4 (명령어 하나 = 4바이트인 경우, 순차 실행) 분기 명령어: PC ← 분기 목적지 주소 (조건부 / 무조건 점프) 인터럽트: PC ← 인터럽트 핸들러 주소 (이전 PC는 스택에 저장) PC는 특수 레지스터로, 매 클럭 사이클 자동 증가 또는 ALU가 계산한 분기 주소로 갱신됩니다.

6. 유한 상태 기계 (FSM) 개요

순서 논리 회로의 일반적인 모델이 유한 상태 기계(Finite State Machine)입니다. CPU의 제어 유닛(Control Unit)이 FSM으로 구현됩니다.

FSM 구성 요소: ① 상태 레지스터 : 현재 상태를 저장하는 D 플립플롭 ② 다음 상태 논리 : 현재 상태 + 입력 → 다음 상태 (조합 논리) ③ 출력 논리 : 현재 상태 (+ 입력) → 출력 (조합 논리) 두 가지 FSM 유형: Mealy 기계: 출력 = f(현재 상태, 현재 입력) — 빠름, 상태 수 적음 Moore 기계: 출력 = f(현재 상태만) — 안전, 글리치 적음 CPU 제어 유닛 예: 상태: Fetch → Decode → Execute → Memory → Writeback 각 상태에서 제어 신호(RegWrite, MemRead 등)를 출력
클럭 주파수와 성능의 관계
클럭 1주기 = 1 사이클 = 최소 실행 단위
클럭이 빠를수록 초당 더 많은 상태 전이가 발생 → 더 빠른 연산
최대 클럭 = 1 / (조합 논리 지연 + 셋업 타임 + 클럭 지터)
Intel Core i9-13900K의 최대 부스트 클럭 ≈ 5.8 GHz는 이 물리적 한계를 반도체 공정 기술로 밀어붙인 결과입니다.

오늘의 핵심 요약

  • 순서 논리: 출력 = f(현재 입력, 현재 상태). 클럭과 피드백으로 상태를 기억
  • SR 래치: 가장 단순한 기억 소자. S̄=R̄=0 금지 상태(경쟁 조건)가 한계
  • D 플립플롭: 엣지 트리거로 금지 상태 제거. 클럭 ↑에서만 D를 Q에 캡처
  • 셋업/홀드 타임: 클럭 엣지 전후로 D를 안정 유지해야 하는 최소 시간. CPU 클럭 속도 한계
  • 레지스터: D-FF n개 병렬 = n비트 저장. Write Enable로 갱신 시점 제어
  • 카운터: 동기 이진 카운터로 글리치 없이 매 클럭 증가. PC가 대표적
  • FSM: 순서 논리의 일반 모델. CPU 제어 유닛은 Fetch→Decode→Execute FSM
