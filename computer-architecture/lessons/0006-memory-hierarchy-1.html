빠름 vs 용량 — 메모리 계층과 SRAM/DRAM

오늘 배울 것

  • 왜 단 하나의 "완벽한" 메모리가 존재할 수 없는지 이해한다
  • 메모리 계층 피라미드(레지스터 → 캐시 → 주기억장치 → 보조기억장치)를 파악한다
  • SRAM(플립플롭 기반)과 DRAM(커패시터 기반)의 구조적 차이를 비교한다

1. 왜 하나의 메모리로 충분하지 않은가?

이상적인 메모리는 크고, 빠르고, 저렴해야 합니다. 그러나 현실에서는 세 가지를 동시에 만족시킬 수 없습니다.

메모리의 3가지 상충 관계 (Trilemma): 빠름(Speed) ▲ │ SRAM (캐시) │ → 빠름 + 비쌈 + 용량 소 │ │ └────────────────▶ 저렴함(Cost) / / DRAM (주기억장치) / → 중간 속도 + 중간 가격 + 용량 중 / ▼ 큰 용량(Capacity) HDD/SSD (보조기억장치) → 느림 + 저렴 + 용량 대
Moore의 법칙과 메모리 격차
CPU 속도는 매 18개월마다 2배씩 빨라졌지만(Moore의 법칙), DRAM 속도는 훨씬 느리게 개선되었습니다. 1980년대 초에는 CPU와 DRAM 속도가 비슷했으나, 2020년대에는 CPU가 DRAM보다 수백 배 빠릅니다. 이 격차를 메모리 장벽(Memory Wall)이라 하며, 캐시 계층이 이를 완화하는 핵심 장치입니다.

3. SRAM vs DRAM — 구조적 차이

3-1. SRAM (Static RAM)

SRAM 비트셀 구조 (6-트랜지스터 셀): BL ─┤M5├─ Q ─┤M1├─ GND Word Line(WL)이 선택되면 B̄L ─┤M6├─ Q̄─┤M3├─ GND BL/B̄L을 통해 Q/Q̄ 읽기/쓰기 ┌───────────────────┐ │ M1 ──── M3 │ ← 인버터 쌍 (교차 결합) │ M2 ──── M4 │ Day 5의 SR 래치와 동일 원리 │ M5 M6 (액세스) │ └───────────────────┘ 특징: ① 1비트 = 트랜지스터 6개 ② 전원이 있는 한 refresh 없이 상태 유지 (Static) ③ 속도: 0.5~5 ns (매우 빠름) ④ 면적: 넓음 → 고가, 집적도 낮음 ⑤ 소비 전력: 스탠바이 시 낮음, 스위칭 시 높음 ⑥ 사용처: L1/L2/L3 캐시, 레지스터 파일

3-2. DRAM (Dynamic RAM)

DRAM 비트셀 구조 (1-트랜지스터 1-커패시터 셀): Word Line(WL) ─┤ M ├─ Bit Line(BL) │ C (커패시터: 전하 저장) 1 = 커패시터 충전 상태 (전하 있음) 0 = 커패시터 방전 상태 (전하 없음) 특징: ① 1비트 = 트랜지스터 1개 + 커패시터 1개 (1T1C) ② 커패시터 전하가 시간 지나면 누설 → 주기적 재충전(Refresh) 필요 → 약 64 ms마다 전체 행을 읽고 다시 씀 ③ 속도: 50~200 ns (SRAM보다 10~100배 느림) ④ 면적: 좁음 → 저가, 높은 집적도 (GB 수준 가능) ⑤ 소비 전력: refresh로 인한 정적 소비 전력 있음 ⑥ 사용처: 주기억장치 (메인 메모리)

DRAM의 Refresh가 왜 문제인가?
Refresh 중에는 해당 행의 DRAM이 읽기/쓰기 요청을 처리하지 못합니다. 현대 DRAM은 이를 숨기기 위해 자동 갱신(Auto-Refresh) 기능과 전체 DRAM 뱅크를 교번하며 refresh를 분산하는 방식을 사용합니다. 그러나 대용량 DRAM일수록 refresh 오버헤드가 커지는 한계가 있습니다.

4. 지역성의 원리 (Principle of Locality)

캐시가 작음에도 불구하고 높은 히트율을 달성할 수 있는 이유는 실제 프로그램의 메모리 접근 패턴에 지역성이 존재하기 때문입니다.

4-1. 시간적 지역성 (Temporal Locality)

"최근에 접근한 데이터는 곧 다시 접근될 가능성이 높다" 예: 루프 변수 for (int i = 0; i < 1000; i++) { sum += arr[i]; // 'sum'은 1000번 반복 접근 } ↑ sum은 루프 내내 계속 사용됨 → 캐시에 유지하면 이득 예: 함수 자체 코드 printf를 1000번 호출하면 printf 명령어 코드는 첫 번째 호출 이후 캐시에 남아 이후 999번은 캐시에서 처리

4-2. 공간적 지역성 (Spatial Locality)

"최근에 접근한 주소 근처의 데이터도 곧 접근될 가능성이 높다" 예: 배열 순차 접근 for (int i = 0; i < 1024; i++) { sum += arr[i]; // arr[0], arr[1], arr[2], ... 순서로 접근 } ↑ arr[0] 접근 시 인근 arr[1]~arr[15]도 캐시 라인에 함께 적재 → 이후 15번은 캐시 히트 (캐시 라인 크기가 64바이트인 경우) 예: 명령어 실행 CPU는 보통 순차적으로 명령어를 실행 → 현재 명령어 다음 명령어도 이미 캐시에 존재

4-3. 지역성을 활용하는 캐시 동작

캐시 적재 단위: 캐시 라인 (Cache Line) 일반적으로 64바이트 = 16개의 32비트 워드 캐시 미스 발생 시: ① CPU가 주소 A의 데이터 요청 ② L1 캐시 미스 → L2 확인 ③ L2 캐시 미스 → L3 확인 ④ L3 캐시 미스 → DRAM에서 64바이트 블록 통째로 가져옴 ⑤ 블록을 L3 → L2 → L1 순서로 채움 ⑥ CPU에 데이터 전달 공간적 지역성 덕분에: A 주변 63바이트도 함께 캐시에 올라옴 → 근처 데이터를 곧 접근하면 캐시 히트

5. 캐시 히트·미스와 평균 메모리 접근 시간 (AMAT)

5-1. 히트와 미스

캐시 히트 (Cache Hit): CPU가 요청한 데이터가 캐시에 이미 있음 접근 시간 = 캐시 접근 시간 (L1 기준 1~4 사이클) 캐시 미스 (Cache Miss): CPU가 요청한 데이터가 캐시에 없음 접근 시간 = 캐시 접근 시간 + 하위 메모리 접근 시간 (패널티) 히트율 (Hit Rate, h): h = 캐시 히트 횟수 / 전체 메모리 접근 횟수 미스율 (Miss Rate): Miss Rate = 1 - h

5-2. 평균 메모리 접근 시간 (AMAT)

AMAT (Average Memory Access Time) 공식: AMAT = 히트 시간 + 미스율 × 미스 패널티 예시: L1 캐시 접근 시간 = 2 사이클 L1 미스율 = 5% (= 0.05) DRAM 접근 시간 (패널티) = 200 사이클 AMAT = 2 + 0.05 × 200 = 2 + 10 = 12 사이클 비교: L1 캐시 없이 매번 DRAM 접근 = 200 사이클 L1 캐시 있을 때 평균 = 12 사이클 → 캐시로 인한 성능 향상 = 200 / 12 ≈ 16.7배

5-3. 다단 캐시의 AMAT

L1 → L2 → DRAM 계층 구조: AMAT = T_L1 + MR_L1 × (T_L2 + MR_L2 × T_DRAM) 예시 (Intel Core i7 근사치): T_L1 = 4 사이클 MR_L1 = 5% T_L2 = 12 사이클 MR_L2 = 20% (L1 미스 중 20%가 L2 미스) T_DRAM = 200 사이클 AMAT = 4 + 0.05 × (12 + 0.20 × 200) = 4 + 0.05 × (12 + 40) = 4 + 0.05 × 52 = 4 + 2.6 = 6.6 사이클 캐시 없을 때: 200 사이클 다단 캐시: 6.6 사이클 → 약 30배 향상
캐시가 항상 도움이 되는 것은 아니다
공간적·시간적 지역성이 낮은 접근 패턴에서는 캐시 히트율이 낮아집니다.
예시: 열(column) 우선 행렬 접근
C언어 배열은 행(row) 우선으로 저장됩니다.
행 기준 이중 루프: 공간적 지역성 높음 → 캐시 효율적
열 기준 이중 루프: 캐시 라인을 매번 건너뜀 → 캐시 비효율적 (수십 배 느릴 수 있음)

6. 캐시 미스의 3C 분류

Compulsory Miss (강제 미스, Cold Miss): 처음으로 접근하는 데이터 → 캐시에 있을 수 없음 프로그램 시작 초기에 집중적으로 발생 해결: prefetch(선제적 적재)로 완화 Capacity Miss (용량 미스): 캐시 용량이 작아 필요한 데이터 전체를 담지 못함 워킹 셋(Working Set)이 캐시보다 클 때 발생 해결: 캐시 크기 증가 Conflict Miss (충돌 미스): 직접 매핑(Direct-Mapped) 캐시에서 서로 다른 주소가 같은 캐시 슬롯에 경쟁할 때 발생 A와 B가 같은 슬롯을 사용: A 접근 → A 적재 → B 접근 → A 교체 → A 접근 → B 교체 → ... 해결: 집합 연관 캐시(Set-Associative Cache) 사용 (Day 7에서 상세 학습)

오늘의 핵심 요약

  • 메모리 트릴레마: 빠름·큰 용량·저렴함 세 가지를 동시에 만족하는 메모리는 없다
  • 메모리 계층: 레지스터 → L1 → L2 → L3 → DRAM → SSD/HDD 순으로 속도↓, 용량↑, 가격↓
  • SRAM: 6-트랜지스터 플립플롭 기반. Refresh 불필요. 빠르고 비쌈. 캐시에 사용
  • DRAM: 1T1C 커패시터 기반. 주기적 Refresh 필요. 느리고 저렴. 주기억장치에 사용
  • 시간적 지역성: 최근 접근 데이터는 곧 다시 접근될 가능성이 높다 (루프 변수 등)
  • 공간적 지역성: 최근 접근 주소 인근도 곧 접근될 가능성이 높다 (배열 순차 접근 등)
  • AMAT: = 히트 시간 + 미스율 × 미스 패널티. 다단 캐시로 최소화
  • 3C 미스: Compulsory(최초 접근) / Capacity(용량 부족) / Conflict(슬롯 충돌)
