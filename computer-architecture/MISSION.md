# Mission: 컴퓨터 구조 (Computer Architecture)

## Why
소프트웨어 개발자 또는 CS 학습자로서, 컴퓨터가 내부적으로 어떻게 동작하는지 이해함으로써 더 효율적인 코드를 작성하고, 성능 병목을 진단하고, 하드웨어-소프트웨어 인터페이스를 깊이 이해하는 것이 목표입니다. 매일 아침 꾸준한 학습을 통해 컴퓨터 구조의 핵심 개념들을 습득합니다.

## Success looks like
- CPU, 메모리, I/O 서브시스템이 어떻게 상호작용하는지 설명할 수 있다
- 파이프라이닝, 캐시 계층, 명령어 집합 아키텍처(ISA)의 핵심 개념을 이해한다
- 특정 코드 패턴이 왜 빠르거나 느린지 하드웨어 관점에서 설명할 수 있다
- 컴퓨터 구조 관련 기술 면접 질문에 자신있게 답변할 수 있다
- 간단한 프로세서 설계 개념을 이해하고 도식화할 수 있다

## Constraints
- 매일 아침 단일 세션으로 학습 (1회 보고서 기준)
- 이론과 실용적 이해를 균형있게 다룸
- 이전 학습 내용과 연계하여 점진적으로 심화

## Out of scope
- FPGA 프로그래밍이나 실제 하드웨어 제작
- 특정 마이크로아키텍처 세부사항 (Intel/AMD 내부 구조의 극세부)
- 임베디드 시스템 프로그래밍 (학습 후 연계 가능)
