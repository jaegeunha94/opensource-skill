# Mission: 시스템 설계 Interview

## Why

10년차 이상 시니어 개발자로서 시스템 설계 인터뷰에서 평가자가 기대하는 수준의 답변을 일관되게 낼 수 있어야 한다.
단순히 컴포넌트를 나열하는 게 아니라, 요구사항 정리 → 규모 산정 → trade-off 판단 → 장애 격리 → 확장성 설계로 이어지는 구조화된 사고를 실시간으로 보여주는 능력을 기른다.

## Success looks like

- 주어진 시스템 설계 문제에서 5분 안에 functional/non-functional requirements를 구조적으로 정리할 수 있다
- DAU, QPS, 저장 용량, 대역폭을 빠르게 추정하고 설계 결정에 근거로 사용할 수 있다
- 수평 확장, 캐시, DB 샤딩, 메시지 큐, 장애 격리 등 각 선택의 trade-off를 설명할 수 있다
- "왜 이 방식을 선택했나요?" "다른 방법은 없었나요?"에 즉각 답할 수 있다
- CAP theorem, 분산 시스템 consistency 모델, 장애 시나리오를 production 경험과 연결해 설명할 수 있다
- 면접에서 45분 안에 complete system design을 논리적으로 전달할 수 있다

## Constraints

- 면접 대비 목적이므로 구현 코드보다는 설계 논리와 커뮤니케이션 방식에 집중한다
- 이미 백엔드 실무 경험이 있으므로 기초 HTTP, DB 개념은 최소화하고 scale/trade-off 판단에 집중한다
- 레슨당 1개의 핵심 시스템 설계 주제 또는 패턴을 다룬다

## Out of scope

- 특정 언어나 프레임워크 구현 코드
- 알고리즘 코딩 문제 (별도 트랙)
- 기초 CS (별도 트랙에서 다룸)
