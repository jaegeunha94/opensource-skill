Day 1 — 네트워크 기초 & OSI 모델

학습 목표

- 네트워크가 무엇인지, 왜 필요한지 설명할 수 있다.

- OSI 7계층 각각의 역할과 데이터 단위(PDU)를 말할 수 있다.

- 캡슐화(Encapsulation)와 역캡슐화(Decapsulation) 과정을 그림으로 설명할 수 있다.

1. 네트워크란 무엇인가?

네트워크(Network) 는 두 대 이상의 장치(device)가 데이터를 주고받을 수 있도록 연결된 시스템이다.

[PC A] ──── [스위치] ──── [라우터] ──── 인터넷
              │
           [PC B]

핵심 용어

용어설명
호스트(Host)네트워크에 연결된 장치. 데이터를 보내거나 받는 주체.
노드(Node)네트워크 상의 모든 장치 (호스트 + 스위치/라우터 등 중간 장비 포함).
대역폭(Bandwidth)단위 시간당 전송할 수 있는 최대 데이터 양 (단위: bps).
지연(Latency)데이터가 출발지에서 목적지까지 걸리는 시간.
패킷(Packet)네트워크에서 전송되는 데이터의 기본 단위.

네트워크 분류

종류범위예시
LAN (Local Area Network)건물 / 캠퍼스 내부사무실 Wi-Fi, 이더넷
WAN (Wide Area Network)도시 / 국가 / 대륙인터넷, 전용선
MAN (Metropolitan Area Network)도시 규모도시 단위 케이블 망
PAN (Personal Area Network)수 미터 이내블루투스, USB

2. 프로토콜(Protocol)이란?

네트워크 통신에서 프로토콜은 두 장치가 데이터를 주고받기 위해 약속한 규칙의 집합이다.

프로토콜이 정의하는 것:

  • 구문(Syntax): 데이터의 형식과 인코딩 방식
  • 의미(Semantics): 각 비트·필드가 의미하는 것
  • 타이밍(Timing): 언제 데이터를 보내고, 얼마나 빨리 보낼지

인간의 언어에 비유하면, 프로토콜은 "문법 + 어휘 + 대화 순서"다.

3. OSI 7계층 모델

OSI(Open Systems Interconnection) 모델은 ISO가 1984년에 제정한 네트워크 통신 표준 참조 모델이다. 복잡한 네트워크 통신 과정을 7개의 계층으로 추상화한다.

┌─────────────────────────────────────┐
│  7. 응용 계층 (Application)          │  ← 사용자·앱과 직접 상호작용
├─────────────────────────────────────┤
│  6. 표현 계층 (Presentation)         │  ← 데이터 형식 변환·암호화
├─────────────────────────────────────┤
│  5. 세션 계층 (Session)              │  ← 세션 수립·유지·종료
├─────────────────────────────────────┤
│  4. 전송 계층 (Transport)            │  ← 종단 간 신뢰성·흐름 제어
├─────────────────────────────────────┤
│  3. 네트워크 계층 (Network)           │  ← 논리 주소(IP), 라우팅
├─────────────────────────────────────┤
│  2. 데이터링크 계층 (Data Link)       │  ← 물리 주소(MAC), 오류 감지
├─────────────────────────────────────┤
│  1. 물리 계층 (Physical)             │  ← 비트 전송, 전기·광 신호
└─────────────────────────────────────┘

암기법

"All People Seem To Need Data Processing"

Application / Presentation / Session / Transport / Network / Data Link / Physical

또는 아래에서 위로:

"Please Do Not Throw Sausage Pizza Away"

4. 각 계층 상세 설명

계층 1 — 물리 계층 (Physical Layer)

  • 역할: 비트(0과 1)를 전기 신호, 광 신호, 무선 전파로 변환해 전송.
  • PDU: 비트(Bit)
  • 장비: 케이블, 허브(Hub), 리피터(Repeater), NIC(랜카드)
  • 대표 기술: 이더넷 케이블(Cat5e/Cat6), 광섬유, 802.11 Wi-Fi 무선
[PC] ──Cat6 케이블──> [허브] ──> 전기 신호로 비트 전달

계층 2 — 데이터링크 계층 (Data Link Layer)

  • 역할: 같은 네트워크 내 장치 간 신뢰성 있는 데이터 전송. MAC 주소로 장치를 식별.
  • PDU: 프레임(Frame)
  • 장비: 스위치(Switch), 브리지(Bridge)
  • 주요 기능:

- 프레이밍: 비트 스트림을 프레임 단위로 묶음

- 오류 감지: CRC(Cyclic Redundancy Check)로 전송 오류 탐지

- MAC 주소: 48비트 물리 주소 (예: 00:1A:2B:3C:4D:5E)

프레임 구조: [목적지 MAC | 출발지 MAC | 타입 | 데이터 페이로드 | FCS]

계층 3 — 네트워크 계층 (Network Layer)

  • 역할: 서로 다른 네트워크 간 데이터 전달. IP 주소로 경로(Route) 결정.
  • PDU: 패킷(Packet)
  • 장비: 라우터(Router)
  • 대표 프로토콜: IP, ICMP, ARP (일부 모델 기준)
[192.168.1.10] ──라우터──> [10.0.0.5]
                  ↑
           IP 주소 기반 경로 결정

계층 4 — 전송 계층 (Transport Layer)

  • 역할: 출발지에서 목적지 프로세스 간 종단 간(End-to-End) 통신 보장.
  • PDU: 세그먼트(Segment, TCP) / 데이터그램(Datagram, UDP)
  • 대표 프로토콜: TCP, UDP
  • 포트(Port) 번호로 프로세스 구분 (예: HTTP=80, HTTPS=443, DNS=53)
TCPUDP
연결 지향 (3-way handshake)비연결 지향
신뢰성 보장 (재전송)빠르고 경량
순서 보장순서 보장 없음
파일 전송, 웹스트리밍, DNS

계층 5 — 세션 계층 (Session Layer)

  • 역할: 두 애플리케이션 간 세션(Session) 수립, 유지, 종료 관리.
  • 동기화(Synchronization) 포인트로 중단 후 재개 가능.
  • 대표 프로토콜: NetBIOS, RPC, PPTP

실무에서는 TCP/IP 모델에서 세션·표현·응용 계층을 합쳐 응용 계층으로 처리한다.

계층 6 — 표현 계층 (Presentation Layer)

  • 역할: 데이터의 형식 변환, 압축, 암호화/복호화.
  • 송신측: 애플리케이션 데이터 → 네트워크 표준 형식으로 변환
  • 수신측: 네트워크 형식 → 애플리케이션이 이해할 수 있는 형식으로 변환
  • 대표 예: JPEG, MP3, TLS 암호화, ASCII/UTF-8 변환

계층 7 — 응용 계층 (Application Layer)

  • 역할: 사용자나 애플리케이션이 직접 사용하는 서비스 제공.
  • PDU: 데이터(Data) / 메시지(Message)
  • 대표 프로토콜: HTTP, HTTPS, FTP, SMTP, DNS, DHCP, SSH
사용자가 브라우저에 URL 입력
       ↓
응용 계층: HTTP 요청 생성
       ↓
... 아래 계층들이 순서대로 처리 ...
       ↓
상대방 서버에 도달

5. 캡슐화(Encapsulation)와 역캡슐화(Decapsulation)

데이터가 송신측에서 수신측으로 이동할 때, 각 계층이 헤더(Header) 를 붙이는 과정이 캡슐화다.

캡슐화 — 송신 방향 (위 → 아래)

[응용 계층]     데이터 (Data)
     ↓ + HTTP 헤더
[전송 계층]     세그먼트 [TCP Header | Data]
     ↓ + TCP 헤더
[네트워크 계층]  패킷    [IP Header | TCP Header | Data]
     ↓ + IP 헤더
[데이터링크]     프레임  [ETH Header | IP Header | TCP Header | Data | FCS]
     ↓ 비트로 변환
[물리 계층]     비트 스트림 ──────────────────────────────> 전선

역캡슐화 — 수신 방향 (아래 → 위)

수신측은 반대로 각 계층에서 헤더를 제거(벗겨내기)하며 상위 계층으로 데이터를 전달한다.

[물리 계층]     비트 수신
     ↑ ETH 헤더 제거
[데이터링크]     프레임 확인 → MAC 주소 검증
     ↑ IP 헤더 제거
[네트워크 계층]  패킷 확인 → IP 주소 검증
     ↑ TCP 헤더 제거
[전송 계층]     세그먼트 재조합 → 포트 번호로 프로세스 전달
     ↑
[응용 계층]     최종 데이터 사용자에게 전달

핵심 포인트: 각 계층은 자신의 계층 헤더만 보고 처리한다. 하위 계층은 상위 계층의 내용을 알 필요 없다(계층 독립성).

6. PDU(Protocol Data Unit) 정리

계층PDU 이름
7 — 응용데이터/메시지 (Data/Message)
6 — 표현데이터 (Data)
5 — 세션데이터 (Data)
4 — 전송세그먼트 / 데이터그램 (Segment/Datagram)
3 — 네트워크패킷 (Packet)
2 — 데이터링크프레임 (Frame)
1 — 물리비트 (Bit)

7. OSI vs TCP/IP 모델 비교

실무에서는 OSI 7계층보다 TCP/IP 4계층 모델이 더 많이 쓰인다.

OSI 7계층                   TCP/IP 4계층
─────────────────────────   ─────────────────
7. 응용 계층   ┐
6. 표현 계층   ├──────────> 4. 응용 계층 (Application)
5. 세션 계층   ┘
4. 전송 계층  ──────────>   3. 전송 계층 (Transport)
3. 네트워크 계층 ────────>  2. 인터넷 계층 (Internet)
2. 데이터링크 ┐
1. 물리 계층  ├──────────>  1. 네트워크 액세스 계층 (Network Access)
              ┘

OSI는 이론적 참조 모델, TCP/IP는 실제 인터넷에서 구현된 모델이다.

9. 핵심 요약

  • 네트워크는 데이터를 주고받기 위한 장치들의 연결이며, 프로토콜로 규칙을 정한다.
  • OSI 7계층은 물리 → 응용 방향으로 역할이 분리된 참조 모델이다.
  • 각 계층은 PDU라는 데이터 단위를 가지며, 송신 시 헤더를 추가(캡슐화), 수신 시 헤더를 제거(역캡슐화)한다.
  • 실무 인터넷은 TCP/IP 4계층 모델로 동작한다.
