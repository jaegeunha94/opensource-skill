Day 7 — 네트워크 보안 기초

오늘은 네트워크를 보호하는 핵심 기술과 대표적인 공격 패턴을 배웁니다. NAT로 사설 IP를 숨기는 원리, 방화벽이 트래픽을 필터링하는 방식, VPN의 터널링 개념, 그리고 실제로 자주 발생하는 공격 유형까지 다룹니다.

학습 목표
NAT 동작 원리와 포트 변환 테이블 이해 · 방화벽 규칙(ACL) 적용 순서 파악 · VPN 터널링과 암호화 개념 설명 · 주요 네트워크 공격(DoS/DDoS, MitM, IP Spoofing 등) 식별

1. NAT — Network Address Translation

NAT는 사설 IP 주소(192.168.x.x, 10.x.x.x, 172.16–31.x.x)를 공인 IP로 변환해 인터넷에 내보내는 기술입니다. IPv4 주소 부족 문제를 해결하고, 내부 망 구조를 외부에 숨기는 보안 효과도 줍니다.

1-1. NAT 종류

종류설명사용 예
Static NAT 사설 IP 1개 ↔ 공인 IP 1개 (1:1 고정 매핑) 내부 서버를 외부에 공개할 때
Dynamic NAT 사설 IP 풀 → 공인 IP 풀 (동적 매핑, 공인 IP 부족 시 거부) 공인 IP 풀이 여럿인 기업망
PAT / NAPT 여러 사설 IP → 공인 IP 1개 + 포트 번호로 구분 (N:1) 가정용 공유기(가장 흔한 형태)

PAT(Port Address Translation)가 일반 NAT와 다른 핵심은 포트 번호를 추가 식별자로 사용한다는 점입니다. 공유기 한 대에 연결된 스마트폰·노트북·TV가 각각 다른 포트로 변환되어 인터넷을 공유할 수 있습니다.

1-2. NAT 변환 테이블 (PAT 예시)

사설 IP:포트공인 IP:포트목적지프로토콜
192.168.1.10:5001203.0.113.1:400018.8.8.8:53UDP
192.168.1.20:5002203.0.113.1:40002142.250.80.46:443TCP
192.168.1.30:5003203.0.113.1:40003104.26.10.78:80TCP

응답 패킷이 돌아오면 라우터는 목적지 포트(40001, 40002, …)를 보고 변환 테이블에서 원래 사설 IP:포트를 찾아 내부로 전달합니다.

1-3. 인터랙티브 — NAT 패킷 흐름 시뮬레이터

내부 PC 192.168.1.10:5001 → 8.8.8.8:53 (DNS 쿼리)
라우터 NAT 변환: src 192.168.1.10:5001 → 203.0.113.1:40001
인터넷 203.0.113.1:40001 → 8.8.8.8:53 (출발)
인터넷 8.8.8.8:53 → 203.0.113.1:40001 (응답 도착)
라우터 역변환: dst 203.0.113.1:40001 → 192.168.1.10:5001
내부 PC 192.168.1.10:5001 수신 완료 (DNS 응답)
▶ 버튼을 눌러 패킷 흐름을 단계별로 확인하세요.

2. 방화벽 (Firewall)

방화벽은 미리 정의된 규칙(ACL)에 따라 패킷을 허용(ALLOW) 또는 차단(DENY)합니다. 규칙은 위에서 아래로 순서대로 평가하며, 첫 번째로 일치하는 규칙이 적용됩니다.

2-1. 패킷 필터링 vs 상태 검사 vs 애플리케이션 방화벽

유형검사 수준특징
패킷 필터링 IP/포트/프로토콜 빠르지만 연결 상태 인식 없음; 1세대 방화벽
상태 검사 (Stateful) 연결 상태 테이블 TCP 연결 확립 여부 추적; 현재 대부분의 방화벽
NGFW / 애플리케이션 L7 페이로드 HTTP URL, DNS 도메인, 앱 프로토콜까지 필터링

2-2. 방화벽 규칙 예시

#방향출발 IP목적지 IP포트프로토콜동작
1INany10.0.0.10443TCPALLOW
2INany10.0.0.1080TCPALLOW
3IN192.168.1.0/24any22TCPALLOW
4INanyany22TCPDENY
5INanyanyanyanyDENY

묵시적 거부 (Implicit Deny) — 마지막 규칙 5처럼 모든 트래픽을 거부하는 규칙을 두어, 위에서 매칭되지 않은 패킷은 모두 차단합니다. "허용 목록(allowlist)" 방식의 기본 원칙입니다.

2-3. DMZ (비무장지대)

웹 서버처럼 외부에 공개되어야 하는 시스템은 DMZ 구역에 배치합니다. 인터넷 ↔ DMZ 사이에 외부 방화벽, DMZ ↔ 내부망 사이에 내부 방화벽을 두어 이중으로 보호합니다.

인터넷
    │
  [외부 방화벽]  — 443, 80만 허용
    │
  [DMZ: 웹 서버 / 메일 서버]
    │
  [내부 방화벽]  — DB 포트만 내부 허용
    │
  [내부망: DB, 업무 시스템]

3. VPN — Virtual Private Network

VPN은 공용 인터넷 위에 암호화된 터널을 만들어 마치 전용 회선처럼 안전하게 통신합니다.

3-1. 핵심 개념

  • 터널링 — 원본 IP 패킷을 새 패킷으로 캡슐화해 전송. 수신 측에서 역캡슐화.
  • 암호화 — AES-256 등으로 페이로드를 암호화해 도청 방지.
  • 인증 — 사전 공유 키(PSK), 인증서, 사용자 ID/PW 등으로 신원 확인.

3-2. 주요 VPN 프로토콜 비교

프로토콜계층포트특징
IPsec L3 UDP 500/4500 기업 사이트 간 VPN(S2S) · AH(인증)+ESP(암호화) 모드
OpenVPN L4/TLS UDP 1194 / TCP 443 오픈소스, TLS 기반, 방화벽 우회 용이
WireGuard L3 UDP 51820 간결한 코드(~4000줄), 빠른 속도, 최신 표준
SSL VPN L4/TLS TCP 443 브라우저 기반 원격 접속; 클라이언트 설치 불필요

3-3. Site-to-Site vs Remote Access VPN

Site-to-SiteRemote Access
목적두 거점 사무실 연결재택·출장 직원이 내부망 접속
장치양쪽 라우터/방화벽직원 단말 + VPN 서버
예시본사-지사 IPsec 터널GlobalProtect, Cisco AnyConnect

4. 주요 네트워크 공격 유형

4-1. DoS / DDoS

DoS/DDoS 서비스 거부 공격 — 대량의 트래픽(패킷)을 서버에 쏟아 정상 사용자가 접근하지 못하도록 만드는 공격입니다.

유형원리대응
SYN Flood TCP 3-way handshake 중 SYN만 대량 전송 → 서버 연결 테이블 소진 SYN Cookie, Rate Limiting
UDP Flood 랜덤 포트로 대량 UDP 패킷 전송 → 서버 ICMP 응답 부하 트래픽 기반 필터링
HTTP Flood 정상처럼 보이는 HTTP GET/POST 대량 전송 (L7 공격) CAPTCHA, 행동 분석, CDN
DNS 증폭 소스 IP를 피해자로 위조 → DNS 응답(큰 패킷)이 피해자에게 쏟아짐 BCP38 Ingress Filtering

DDoS는 수천~수백만 대의 감염된 기기(봇넷)가 동시에 공격해 단일 차단으로 막기 어렵습니다. CDN, Anycast 흡수, ISP 협력 차단(Blackholing)을 조합해 대응합니다.

4-2. Man-in-the-Middle (MitM)

MitM 통신 경로에 공격자가 끼어들어 패킷을 가로채거나 변조합니다.

  • ARP Spoofing — 위조된 ARP 응답을 브로드캐스트해 게이트웨이 MAC을 자신의 MAC으로 교체.
  • DNS Spoofing — DNS 캐시를 오염시켜 도메인이 공격자 서버로 연결되게 함.
  • SSL Stripping — HTTPS → HTTP 다운그레이드 후 평문 트래픽 도청.

대응 — HTTPS 강제(HSTS), 인증서 고정(Certificate Pinning), Dynamic ARP Inspection(DAI), DNSSEC.

4-3. IP Spoofing

IP 스푸핑 출발지 IP를 위조해 출처를 숨기거나 신뢰된 IP인 척합니다. DDoS 증폭 공격에 주로 쓰입니다. 대응책은 BCP38 — ISP가 자신이 할당하지 않은 출발지 IP를 가진 패킷을 출구에서 버리는 규칙입니다.

4-4. 포트 스캔

정찰 공격 전 대상 시스템의 열린 포트와 서비스를 파악하는 단계입니다. nmap이 대표적인 도구이며, 반열린(half-open) SYN 스캔으로 로그를 남기지 않고 스캔할 수 있습니다.

nmap -sS -p 1-65535 192.168.1.10    # SYN 스캔 (반열린 스캔)

탐지 · 대응 — IDS(Intrusion Detection System) 경보, Rate Limiting, 포트 노킹(port knocking).

4-5. 재전송 공격 (Replay Attack)

재전송 정상 인증 패킷을 캡처해 나중에 다시 전송해 인증을 통과하는 공격입니다. Nonce(일회용 난수), Timestamp, Sequence Number로 방어합니다.

4-6. 피싱 / 소셜 엔지니어링

피싱 기술적 취약점보다 사람을 속이는 공격입니다. 가짜 로그인 페이지, 악성 이메일 첨부 등이 대표적입니다. 스피어 피싱은 특정 대상을 겨냥해 정교하게 꾸민 형태입니다.
