Day 3 — 네트워크 계층 : IP와 라우팅

학습 목표

- IPv4 주소 구조와 클래스를 설명할 수 있다.

- 서브넷 마스크와 CIDR 표기법으로 네트워크/호스트 범위를 계산할 수 있다.

- 라우팅 테이블의 구조와 Longest Prefix Match 규칙을 이해한다.

- IPv6가 필요한 이유와 기본 표기법을 안다.

1. 네트워크 계층이란?

Day 2에서 다룬 데이터링크 계층은 같은 네트워크(LAN) 안에서 인접 장치끼리 프레임을 전달하는 역할을 했다. 하지만 서울 PC가 뉴욕 서버에 데이터를 보내려면 수십 개의 서로 다른 네트워크를 통과해야 한다.

네트워크 계층(Network Layer)은 이 역할을 담당한다. OSI 3계층 / TCP/IP 인터넷 계층에 해당하며, 핵심 프로토콜은 IP(Internet Protocol)다.

역할설명
논리 주소 부여각 장치에 전 세계적으로 고유한 IP 주소 할당
패킷 라우팅출발지 → 목적지까지 최적 경로 결정
단편화(Fragmentation)MTU를 초과하는 패킷을 분할하여 전송

MAC vs IP 주소 — MAC 주소는 물리적 위치를 나타내며 같은 LAN 안에서만 유효하다. IP 주소는 논리적 위치를 나타내며 인터넷 전체에서 패킷을 라우팅하는 데 사용된다. 라우터는 IP 헤더를 보고 다음 홉(hop)을 결정한다.

2. IPv4 주소 구조

IPv4 주소는 32비트로 이루어지며, 8비트씩 점(.)으로 구분한 점-표기법(dotted-decimal)으로 표현한다.

192    .  168   .   1   .   1
11000000.10101000.00000001.00000001  (이진수)

네트워크 파트 vs 호스트 파트

IP 주소는 두 부분으로 나뉜다.

┌────────────────────────┬───────────────────────────┐
│   네트워크 파트        │     호스트 파트            │
│ (어느 네트워크인가?)   │ (그 네트워크 안 어느 장치?) │
└────────────────────────┴───────────────────────────┘

어디서 잘리느냐는 서브넷 마스크가 결정한다.

클래스형 주소 (역사적 배경)

클래스첫 옥텟 범위네트워크 비트호스트 수용도
A1 – 1268약 1,677만대규모 기관
B128 – 19116약 65,534중규모 기관
C192 – 22324254소규모 네트워크
D224 – 239——멀티캐스트
E240 – 255——연구/예비용

클래스형 방식은 주소 낭비가 심해 현재는 CIDR로 대체되었지만, 사설 주소 대역은 여전히 자주 사용된다.

사설(Private) IP 주소 대역

대역CIDR주소 수
10.0.0.0 – 10.255.255.25510.0.0.0/8약 1,677만
172.16.0.0 – 172.31.255.255172.16.0.0/12약 104만
192.168.0.0 – 192.168.255.255192.168.0.0/1665,536

사설 IP는 인터넷 라우터가 전달하지 않는다. 내부 LAN 전용이며, 인터넷에 나갈 때는 NAT(Network Address Translation)로 공인 IP로 변환된다.

3. 서브넷 마스크와 CIDR

서브넷 마스크

서브넷 마스크는 IP 주소와 AND 연산을 통해 네트워크 주소를 구한다.

IP 주소     :  192.168.1.100  →  11000000.10101000.00000001.01100100
서브넷 마스크:  255.255.255.0  →  11111111.11111111.11111111.00000000
                                   ─────────────────────────── ───────
AND 결과     :  192.168.1.0    →  네트워크 주소           호스트 부분

CIDR (Classless Inter-Domain Routing)

서브넷 마스크의 1 비트 수를 슬래시(/) 뒤에 표기한다.

192.168.1.0/24
              └─ 앞 24비트가 네트워크 파트 (= 마스크 255.255.255.0)

사용 가능한 호스트 수 = 2^(32-24) - 2 = 2^8 - 2 = 254개
  (전체 256개 중 네트워크 주소 192.168.1.0, 브로드캐스트 192.168.1.255 제외)

CIDR 계산 요령

CIDR마스크호스트 수전형적 용도
/8255.0.0.0약 1,677만클래스 A 대역
/16255.255.0.065,534대기업 내부망
/24255.255.255.0254일반 가정/사무실
/25255.255.255.128126/24 반분할
/30255.255.255.2522라우터 간 점대점 링크

공식 — 호스트 비트 수를 h라 하면, 사용 가능한 호스트 수 = 2^h − 2. 뺄 두 개는 네트워크 주소(모두 0)와 브로드캐스트 주소(모두 1)다.

서브넷 계산기 (직접 해보기)

CIDR 표기 입력: (예: 192.168.10.0/26)

4. IPv4 헤더 구조

IP 패킷은 헤더 + 데이터(페이로드)로 구성된다. 헤더의 주요 필드를 알아두면 Wireshark 분석 시 매우 유용하다.

0        8        16       24       32 (bit)
├────────┼────────┼────────┼────────┤
│Version │  IHL   │DSCP/ECN│  Total Length  │
├────────┴────────┼────────┴────────────────┤
│  Identification │ Flags │ Fragment Offset │
├─────────────────┼────────┬────────────────┤
│   TTL           │Protocol│ Header Checksum│
├─────────────────┴────────┴────────────────┤
│              Source IP Address            │
├───────────────────────────────────────────┤
│           Destination IP Address          │
└───────────────────────────────────────────┘
필드설명
Version4 (IPv4) 또는 6 (IPv6)
TTL (Time to Live)라우터를 거칠 때마다 1 감소. 0이 되면 폐기 → 루프 방지
Protocol페이로드 프로토콜: 6=TCP, 17=UDP, 1=ICMP
Source / Dest IP각각 4바이트(32비트) 출발지·목적지 주소
Fragment Offset단편화된 패킷 재조합 순서

5. 라우팅과 라우팅 테이블

라우터의 역할

라우터는 서로 다른 네트워크를 연결하는 장비다. 패킷이 도착하면 IP 헤더의 목적지 주소를 보고 라우팅 테이블을 참조하여 다음 홉(Next Hop)으로 전달한다.

인터넷
   │
[라우터 A] ── 192.168.1.0/24 ── [PC 그룹 1]
   │
   └─── 10.0.0.0/8 ────────── [PC 그룹 2]

라우팅 테이블 예시

# Linux: ip route show
default via 192.168.1.1 dev eth0
10.0.0.0/8      via 10.0.0.1   dev eth1
192.168.1.0/24  dev eth0       proto kernel scope link src 192.168.1.100
항목설명
목적지 네트워크 (Destination)패킷이 향하는 네트워크 대역 (CIDR)
Next Hop (게이트웨이)패킷을 전달할 다음 라우터의 IP
인터페이스 (Interface)패킷을 내보낼 로컬 네트워크 카드
메트릭 (Metric)경로 비용. 낮을수록 우선
기본 경로 (default / 0.0.0.0/0)어떤 항목도 매칭되지 않을 때 사용하는 경로

Longest Prefix Match (LPM)

라우터는 목적지 IP와 가장 길게 일치하는(= 가장 구체적인) 항목을 우선 선택한다.

라우팅 테이블:
  10.0.0.0/8      via GW-A
  10.1.0.0/16     via GW-B
  10.1.2.0/24     via GW-C
  default         via GW-D

목적지: 10.1.2.50
  → /8  일치 (○)
  → /16 일치 (○)
  → /24 일치 (○)  ← 가장 길게 일치 → GW-C 선택

라우팅 종류

종류설명프로토콜 예시
정적 라우팅관리자가 직접 경로 입력ip route add
동적 라우팅라우터끼리 자동으로 경로 정보 교환OSPF, BGP, RIP
기본 경로어디로 갈지 모를 때 단일 출구로 보냄0.0.0.0/0

6. IPv6 개요

IPv4 주소는 약 43억 개(2³²)로 2011년에 고갈되었다. IPv6는 이를 해결하기 위해 설계된 차세대 프로토콜이다.

항목IPv4IPv6
주소 길이32비트128비트
주소 수약 43억약 3.4 × 10³⁸
표기법점-십진수 (192.168.1.1)콜론-16진수 (2001:db8::1)
브로드캐스트있음없음 (멀티캐스트/애니캐스트로 대체)
헤더 단순화복잡고정 40바이트로 단순화
보안선택 (IPsec)기본 포함 (IPsec 권장)

IPv6 주소 표기

전체:  2001:0db8:0000:0000:0000:ff00:0042:8329
단축1: 2001:db8:0:0:0:ff00:42:8329   (앞쪽 0 생략)
단축2: 2001:db8::ff00:42:8329        (연속된 0 그룹을 :: 하나로 생략)
                                       ↑ ::는 주소 전체에서 한 번만 사용 가능

주요 IPv6 주소

주소의미
::1루프백 (IPv4의 127.0.0.1)
fe80::/10링크-로컬 주소 (같은 링크 내 자동 설정)
2001:db8::/32문서/예제용 예약 대역
ff00::/8멀티캐스트

7. ICMP — 네트워크의 진단 도구

ICMP(Internet Control Message Protocol)는 IP와 쌍을 이루는 제어 프로토콜이다. 데이터 전송보다 네트워크 상태 진단에 쓰인다.

ICMP 메시지설명관련 도구
Echo Request / Reply상대방이 살아있는지 확인ping
Time ExceededTTL=0 → 패킷 폐기 알림traceroute
Destination Unreachable목적지에 도달 불가포트 닫힘 알림 등
 
# ping: 왕복 시간 측정
$ ping 8.8.8.8
64 bytes from 8.8.8.8: icmp_seq=1 ttl=117 time=12.4 ms

# traceroute: 경로상 홉 추적
$ traceroute 8.8.8.8
 1  192.168.1.1   1.2 ms    (홈 라우터)
 2  10.0.0.1      5.8 ms    (ISP 첫 홉)
 3  ...
 9  8.8.8.8       12.4 ms

8. 오늘의 핵심 정리

개념한 줄 요약
IPv432비트 논리 주소. 점-십진수 표기. 약 43억 개.
서브넷 마스크 / CIDR네트워크 파트 길이 지정. /24 = 마스크 255.255.255.0.
라우팅 테이블목적지 네트워크 → 다음 홉 매핑. LPM으로 선택.
기본 경로0.0.0.0/0 — 어디로 갈지 모를 때의 출구.
TTL라우터 거칠 때마다 −1. 0이 되면 폐기 → 루프 방지.
IPv6128비트. 주소 고갈 해결. 콜론-16진수 + :: 단축 표기.
ICMPping, traceroute 등 진단 도구의 기반 프로토콜.
