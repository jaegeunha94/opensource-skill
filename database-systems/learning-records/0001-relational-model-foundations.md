# 관계형 모델 기초 확립 (Day 1)

Day 1 레슨을 통해 관계형 모델의 핵심 개념(릴레이션·튜플·속성·도메인·키 종류)과 ERD 읽기, 관계 대수의 σ·π·⋈ 연산을 학습했다. 이 개념들은 이후 모든 SQL과 DB 설계 레슨의 공통 어휘가 되므로, Day 2 이후에는 별도 설명 없이 해당 용어를 사용해도 된다.

## Evidence

Day 1 실습 과제(온라인 서점 시나리오)의 기본키·외래키·카디널리티·관계 대수 SQL 변환 문제를 직접 풀었다.

## Implications

- 슈퍼키·후보키·기본키 구분이 확립되었으므로 Day 2 DDL에서 `PRIMARY KEY` 제약조건을 "왜" 명시하는지 이해 가능
- ERD 카디널리티를 읽을 수 있으므로 M:N 관계가 교차 테이블로 변환되는 이유를 바로 설명 가능
- 관계 대수 `σ`/`π`가 SQL `WHERE`/`SELECT`에 대응됨을 알고 있어 Day 3 DML 학습 부담 감소
