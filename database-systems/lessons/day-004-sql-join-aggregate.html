2. JOIN이란?

JOIN은 두 개 이상의 테이블을 공통 컬럼(주로 FK)으로 연결해 하나의 결과 집합으로 만드는 연산이다.

departments employees ┌──────────┬──────────┐ ┌────────┬──────────┬─────────┐ │ dept_id │ dept_name│ │ emp_id │ name │ dept_id │ ├──────────┼──────────┤ ├────────┼──────────┼─────────┤ │ 1 │ 개발팀 │──┐│ 101 │ 김철수 │ 1 │ │ 2 │ 마케팅팀 │ └│ 102 │ 이영희 │ 1 │ │ 3 │ 인사팀 │──┐│ 103 │ 박민준 │ 2 │ │ 4 │ 기획팀 │ ││ 104 │ 최수진 │ 3 │ └──────────┴──────────┘ ││ 105 │ 정도윤 │ 2 │ │└─────────────────────────────┘ 연결 키: employees.dept_id = departments.dept_id

3. JOIN 종류

JOIN 종류결과NULL 포함 여부
INNER JOIN두 테이블에서 매칭되는 행만매칭 안 되면 제외
LEFT JOIN왼쪽 테이블 전체 + 오른쪽 매칭오른쪽 없으면 NULL
RIGHT JOIN오른쪽 테이블 전체 + 왼쪽 매칭왼쪽 없으면 NULL
FULL OUTER JOIN양쪽 전체 (합집합)양쪽 모두 NULL 가능

3-1. INNER JOIN

두 테이블 모두에 존재하는 행만 반환한다. 부서가 없는 직원(한지민)과 직원이 없는 부서(기획팀)는 결과에서 제외된다.

SELECT e.name, e.salary, d.dept_name
FROM   employees e
INNER JOIN departments d ON e.dept_id = d.dept_id;
namesalarydept_name
김철수6000000개발팀
이영희7500000개발팀
박민준5500000마케팅팀
최수진6200000인사팀
정도윤5000000마케팅팀

테이블 별칭(alias): employees e처럼 테이블에 짧은 이름을 붙이면 e.name처럼 쓸 수 있어 쿼리가 간결해진다. 같은 컬럼명이 두 테이블에 있을 때 반드시 테이블 별칭으로 구분해야 한다.

3-2. LEFT JOIN

왼쪽 테이블(employees)의 모든 행을 반환한다. 오른쪽 테이블(departments)에 매칭이 없으면 NULL로 채운다.

SELECT e.name, e.salary, d.dept_name
FROM   employees e
LEFT JOIN departments d ON e.dept_id = d.dept_id;
namesalarydept_name
김철수6000000개발팀
이영희7500000개발팀
박민준5500000마케팅팀
최수진6200000인사팀
정도윤5000000마케팅팀
한지민6800000NULL

부서 미배정 직원(한지민)이 포함되고 dept_name은 NULL이다.

3-3. RIGHT JOIN

오른쪽 테이블(departments)의 모든 행을 반환한다. 왼쪽 테이블에 매칭이 없으면 NULL로 채운다.

SELECT e.name, e.salary, d.dept_name
FROM   employees e
RIGHT JOIN departments d ON e.dept_id = d.dept_id;
namesalarydept_name
김철수6000000개발팀
이영희7500000개발팀
박민준5500000마케팅팀
정도윤5000000마케팅팀
최수진6200000인사팀
NULLNULL기획팀

직원이 없는 기획팀이 포함되고 name, salary는 NULL이다.

3-4. FULL OUTER JOIN

두 테이블의 모든 행을 반환한다. 매칭되지 않는 쪽은 NULL로 채운다.

SELECT e.name, e.salary, d.dept_name
FROM   employees e
FULL OUTER JOIN departments d ON e.dept_id = d.dept_id;
namesalarydept_name
김철수6000000개발팀
이영희7500000개발팀
박민준5500000마케팅팀
최수진6200000인사팀
정도윤5000000마케팅팀
한지민6800000NULL
NULLNULL기획팀

MySQL은 FULL OUTER JOIN을 지원하지 않는다. LEFT JOIN과 RIGHT JOIN을 UNION으로 결합해 같은 결과를 얻을 수 있다.

4. 집계 함수

집계 함수(Aggregate Function)는 여러 행을 하나의 값으로 요약한다.

함수역할NULL 처리
COUNT(*)전체 행 수NULL 포함
COUNT(컬럼)NULL이 아닌 값의 수NULL 제외
SUM(컬럼)합계NULL 제외
AVG(컬럼)평균NULL 제외
MAX(컬럼)최댓값NULL 제외
MIN(컬럼)최솟값NULL 제외
 
-- 전체 직원 수
SELECT COUNT(*) AS 직원수 FROM employees;          -- 6

-- 부서가 배정된 직원 수 (NULL 제외)
SELECT COUNT(dept_id) AS 부서배정직원수 FROM employees;  -- 5

-- 전체 급여 합계 / 평균 / 최고 / 최저
SELECT
  SUM(salary)  AS 총급여,
  AVG(salary)  AS 평균급여,
  MAX(salary)  AS 최고급여,
  MIN(salary)  AS 최저급여
FROM employees;

COUNT(*) vs COUNT(컬럼): COUNT(*)는 NULL을 포함한 모든 행을 세고, COUNT(컬럼)은 해당 컬럼이 NULL이 아닌 행만 센다. 두 값이 다르다면 해당 컬럼에 NULL이 존재한다는 의미다.

5. GROUP BY — 그룹별 집계

GROUP BY는 지정한 컬럼의 값이 같은 행들을 하나의 그룹으로 묶어 집계를 수행한다.

5-1. 기본 문법

SELECT   그룹기준컬럼, 집계함수(...)
FROM     테이블
[WHERE   행필터조건]
GROUP BY 그룹기준컬럼
[HAVING  그룹필터조건]
[ORDER BY ...];

5-2. 부서별 직원 수와 평균 급여

SELECT   dept_id,
         COUNT(*)        AS 직원수,
         AVG(salary)     AS 평균급여,
         MAX(salary)     AS 최고급여
FROM     employees
GROUP BY dept_id
ORDER BY dept_id;
dept_id직원수평균급여최고급여
1267500007500000
2252500005500000
3162000006200000
NULL168000006800000

5-3. JOIN + GROUP BY 조합

부서 이름을 함께 보여주려면 JOIN을 먼저 수행하고 집계한다.

SELECT   d.dept_name,
         COUNT(e.emp_id) AS 직원수,
         ROUND(AVG(e.salary), 0) AS 평균급여
FROM     departments d
LEFT JOIN employees e ON d.dept_id = e.dept_id
GROUP BY d.dept_id, d.dept_name
ORDER BY 평균급여 DESC NULLS LAST;
dept_name직원수평균급여
개발팀26750000
인사팀16200000
마케팅팀25250000
기획팀0NULL

LEFT JOIN을 사용했기 때문에 직원이 없는 기획팀도 결과에 포함된다. INNER JOIN을 쓰면 기획팀이 사라진다.

6. HAVING — 그룹 필터링

HAVING은 GROUP BY로 만들어진 그룹에 조건을 적용한다. WHERE이 개별 행을 필터링한다면, HAVING은 그룹 결과를 필터링한다.

6-1. WHERE vs HAVING 비교

항목WHEREHAVING
적용 대상개별 행GROUP BY 결과 그룹
실행 순서GROUP BY 이전GROUP BY 이후
집계 함수 사용불가가능

6-2. 직원 수 2명 이상인 부서만 조회

SELECT   d.dept_name,
         COUNT(e.emp_id) AS 직원수,
         AVG(e.salary)   AS 평균급여
FROM     departments d
LEFT JOIN employees e ON d.dept_id = e.dept_id
GROUP BY d.dept_id, d.dept_name
HAVING   COUNT(e.emp_id) >= 2
ORDER BY 직원수 DESC;
dept_name직원수평균급여
개발팀26750000
마케팅팀25250000

6-3. WHERE와 HAVING 동시 사용

-- 급여 600만 원 이상 직원들(WHERE) 중
-- 부서 평균 급여가 650만 원 이상인 부서만(HAVING) 조회
SELECT   d.dept_name,
         COUNT(e.emp_id) AS 고액연봉자수,
         AVG(e.salary)   AS 평균급여
FROM     departments d
INNER JOIN employees e ON d.dept_id = e.dept_id
WHERE    e.salary >= 6000000          -- 개별 행 필터 (GROUP BY 전)
GROUP BY d.dept_id, d.dept_name
HAVING   AVG(e.salary) >= 6500000    -- 그룹 필터 (GROUP BY 후)
ORDER BY 평균급여 DESC;

핵심 구분: "WHERE는 행을 거르고, HAVING은 그룹을 거른다." — 집계 함수(COUNT, AVG 등)를 조건에 쓰고 싶다면 반드시 HAVING을 사용해야 한다.

8. 종합 실습

문제 1

모든 직원의 이름과 소속 부서명을 조회하라. 부서가 미배정된 직원도 포함하고, 그 경우 부서명은 '미배정'으로 표시하라.

정답 보기 
SELECT e.name,
       COALESCE(d.dept_name, '미배정') AS dept_name
FROM   employees e
LEFT JOIN departments d ON e.dept_id = d.dept_id
ORDER BY e.name;

COALESCE(값, 대체값): 첫 번째 인자가 NULL이면 두 번째 인자를 반환한다.

문제 2

각 부서의 직원 수, 평균 급여, 최고 급여를 조회하라. 부서 이름도 함께 표시하고, 직원이 없는 부서도 포함하라. 평균 급여 내림차순으로 정렬하라.

정답 보기 
SELECT   d.dept_name,
         COUNT(e.emp_id)        AS 직원수,
         ROUND(AVG(e.salary), 0) AS 평균급여,
         MAX(e.salary)           AS 최고급여
FROM     departments d
LEFT JOIN employees e ON d.dept_id = e.dept_id
GROUP BY d.dept_id, d.dept_name
ORDER BY 평균급여 DESC NULLS LAST;

문제 3

평균 급여가 600만 원 이상인 부서와 그 평균 급여를 조회하라.

정답 보기 
SELECT   d.dept_name,
         AVG(e.salary) AS 평균급여
FROM     departments d
INNER JOIN employees e ON d.dept_id = e.dept_id
GROUP BY d.dept_id, d.dept_name
HAVING   AVG(e.salary) >= 6000000
ORDER BY 평균급여 DESC;

문제 4

각 부서에서 가장 최근에 입사한 직원의 이름, 부서명, 입사일을 조회하라. (힌트: GROUP BY + MAX 후 JOIN)

정답 보기 
-- 방법 1: 서브쿼리 활용 (Day 5에서 자세히 다룸)
SELECT e.name, d.dept_name, e.hire_date
FROM   employees e
INNER JOIN departments d ON e.dept_id = d.dept_id
WHERE  e.hire_date = (
  SELECT MAX(e2.hire_date)
  FROM   employees e2
  WHERE  e2.dept_id = e.dept_id
)
ORDER BY d.dept_name;

-- 방법 2: 최신 입사일만 먼저 구하기
SELECT e.name, d.dept_name, e.hire_date
FROM   employees e
INNER JOIN departments d ON e.dept_id = d.dept_id
INNER JOIN (
  SELECT dept_id, MAX(hire_date) AS latest
  FROM   employees
  GROUP BY dept_id
) latest ON e.dept_id = latest.dept_id AND e.hire_date = latest.latest
ORDER BY d.dept_name;

9. 핵심 요약

개념핵심 포인트
INNER JOIN두 테이블 모두 매칭되는 행만 — 매칭 안 되면 제외
LEFT JOIN왼쪽 전체 보존 — 오른쪽 없으면 NULL
RIGHT JOIN오른쪽 전체 보존 — 왼쪽 없으면 NULL
FULL OUTER JOIN양쪽 모두 보존 (MySQL 미지원)
GROUP BY같은 값 행을 묶어 집계 — SELECT에는 GROUP BY 기준 컬럼 또는 집계 함수만 쓸 수 있다
HAVING그룹 필터링 — 집계 함수 조건은 WHERE가 아닌 HAVING에
집계 함수COUNT(*) vs COUNT(컬럼): NULL 포함 여부 차이

자주 하는 실수: GROUP BY 없이 집계 함수와 일반 컬럼을 함께 SELECT하면 오류가 발생한다.

-- 오류: dept_id가 GROUP BY에 없음
SELECT dept_id, name, COUNT(*) FROM employees;

-- 올바름
SELECT dept_id, COUNT(*) FROM employees GROUP BY dept_id;
