Day 3 — SQL 기초: DML (Data Manipulation Language)

날짜: 2026-06-22

소요 시간: 약 60분

선행 조건: Day 2 — SQL 기초: DDL (Data Definition Language)

학습 목표

이 레슨을 마치면 다음을 할 수 있다:

  1. SELECT로 테이블 데이터를 조회하고 원하는 컬럼만 뽑을 수 있다
  2. WHERE 절로 조건 필터링을 적용할 수 있다
  3. ORDER BY와 LIMIT으로 결과를 정렬하고 개수를 제한할 수 있다
  4. INSERT로 새 행을 삽입할 수 있다
  5. UPDATE로 기존 행을 수정할 수 있다
  6. DELETE로 행을 삭제하고, WHERE 없이 쓸 때의 위험을 알 수 있다

1. DML이란?

DML(Data Manipulation Language)은 테이블 구조가 아닌 데이터를 다루는 SQL 명령 집합이다.

명령어역할트랜잭션 대상
SELECT데이터 조회읽기 전용 (변경 없음)
INSERT새 행 삽입O
UPDATE기존 행 수정O
DELETE행 삭제O

DDL(CREATE/DROP 등)은 대부분의 DBMS에서 자동 커밋(Auto-commit)된다. 반면 DML은 트랜잭션 제어(COMMIT/ROLLBACK)가 가능하다.

2. 실습용 스키마

이 레슨 전체에서 아래 두 테이블을 사용한다. Day 2에서 배운 DDL로 만든 것과 같은 구조다.

CREATE TABLE departments (
  dept_id   INT          PRIMARY KEY,
  dept_name VARCHAR(100) NOT NULL
);

CREATE TABLE employees (
  emp_id    INT          PRIMARY KEY,
  name      VARCHAR(100) NOT NULL,
  dept_id   INT          REFERENCES departments(dept_id),
  salary    NUMERIC(10,2),
  hire_date DATE
);

3. SELECT — 데이터 조회

3-1. 기본 문법

SELECT 컬럼1, 컬럼2, ...
FROM   테이블명;

모든 컬럼을 조회하려면 *를 사용한다:

SELECT * FROM employees;

특정 컬럼만 선택:

SELECT name, salary FROM employees;

3-2. 컬럼 별칭 (Alias)

SELECT name AS 이름, salary AS 연봉 FROM employees;

3-3. DISTINCT — 중복 제거

SELECT DISTINCT dept_id FROM employees;
-- 결과: 1, 2, 3

4. WHERE — 조건 필터링

4-1. 비교 연산자

연산자의미예시
=같다dept_id = 1
<> 또는 !=다르다dept_id <> 2
>, <, >=, <=크기 비교salary >= 6000000
BETWEEN a AND b범위 포함salary BETWEEN 5000000 AND 6500000
IS NULL / IS NOT NULLNULL 검사dept_id IS NULL
LIKE패턴 매칭name LIKE '김%'
IN (...)목록 포함dept_id IN (1, 3)

4-2. 논리 연산자

-- AND: 두 조건 모두 참
SELECT * FROM employees
WHERE  dept_id = 1 AND salary >= 7000000;

-- OR: 하나 이상 참
SELECT * FROM employees
WHERE  dept_id = 2 OR dept_id = 3;

-- NOT: 조건 반전
SELECT * FROM employees
WHERE  NOT dept_id = 1;

4-3. LIKE 패턴

패턴의미예시 결과
'김%''김'으로 시작김철수, 김OOO
'%준''준'으로 끝남박민준, OO준
'%수%''수' 포함김철수, 최수진
'_수_'정확히 3글자, 가운데 '수'김수진 형태
 
SELECT * FROM employees WHERE name LIKE '김%';
-- 결과: 김철수

5. ORDER BY — 정렬

5-1. 기본 문법

SELECT 컬럼1, 컬럼2
FROM   테이블
ORDER BY 정렬기준컬럼 [ASC | DESC];
  • ASC(기본값): 오름차순 (작은 값 → 큰 값)
  • DESC: 내림차순 (큰 값 → 작은 값)
-- 급여 높은 순
SELECT name, salary FROM employees ORDER BY salary DESC;

-- 이름 가나다 순
SELECT name FROM employees ORDER BY name ASC;

5-2. 다중 정렬 기준

-- dept_id 오름차순, 같은 부서 내 salary 내림차순
SELECT name, dept_id, salary
FROM   employees
ORDER BY dept_id ASC, salary DESC;

6. LIMIT — 결과 개수 제한

-- 급여 상위 3명
SELECT name, salary
FROM   employees
ORDER BY salary DESC
LIMIT 3;

페이지네이션: OFFSET을 함께 쓴다.

-- 2번째 페이지 (페이지당 2건)
SELECT name, salary
FROM   employees
ORDER BY emp_id
LIMIT 2 OFFSET 2;   -- 3번째 행부터 2건

Oracle은 LIMIT 대신 ROWNUM이나 FETCH FIRST n ROWS ONLY를 사용한다. SQL Server는 TOP n을 사용한다.

7. INSERT — 데이터 삽입

7-1. 단일 행 삽입

-- 컬럼 명시 방식 (권장)
INSERT INTO employees (emp_id, name, dept_id, salary, hire_date)
VALUES (106, '한지민', 1, 6800000, '2025-01-05');

-- 컬럼 생략 (모든 컬럼 순서대로 값 입력)
INSERT INTO employees
VALUES (107, '오태양', 3, 5800000, '2025-06-01');

팁: 컬럼 이름을 명시하는 방식이 안전하다. 나중에 테이블에 컬럼이 추가되어도 INSERT문이 깨지지 않는다.

7-2. NULL 삽입

-- dept_id를 아직 모르는 직원 추가
INSERT INTO employees (emp_id, name, dept_id, salary, hire_date)
VALUES (108, '신입사원', NULL, NULL, '2026-06-22');

7-3. 다중 행 삽입

INSERT INTO departments (dept_id, dept_name)
VALUES (4, '기획팀'),
       (5, '법무팀'),
       (6, '재무팀');

8. UPDATE — 데이터 수정

8-1. 기본 문법

UPDATE 테이블명
SET    컬럼1 = 값1, 컬럼2 = 값2, ...
WHERE  조건;
-- 김철수의 급여를 6,500,000으로 인상
UPDATE employees
SET    salary = 6500000
WHERE  emp_id = 101;

-- 개발팀(dept_id=1) 전원 급여 10% 인상
UPDATE employees
SET    salary = salary * 1.10
WHERE  dept_id = 1;

주의: WHERE을 생략하면 테이블의 모든 행이 수정된다.

-- 위험! 모든 직원의 급여가 0이 됨
UPDATE employees SET salary = 0;

실행 전 SELECT로 대상 행을 먼저 확인하는 습관을 들이자.

9. DELETE — 데이터 삭제

9-1. 기본 문법

DELETE FROM 테이블명
WHERE  조건;
-- emp_id가 108인 직원 삭제
DELETE FROM employees WHERE emp_id = 108;

-- 2024년 이후 입사자 삭제
DELETE FROM employees WHERE hire_date >= '2024-01-01';

주의: WHERE 없이 사용하면 테이블의 모든 행이 삭제된다.

-- 위험! 모든 직원 데이터 삭제
DELETE FROM employees;

9-2. DELETE vs TRUNCATE 비교

항목DELETETRUNCATE
분류DMLDDL
WHERE 조건가능불가 (전체 삭제)
ROLLBACK가능대부분 불가 (DBMS마다 다름)
속도느림 (행별 로깅)빠름 (페이지 단위)
트리거실행됨실행 안 됨

10. SELECT 완전한 실행 순서

SELECT 문의 각 절은 작성 순서와 실행 순서가 다르다.

실행 순서절역할
1FROM대상 테이블 결정
2WHERE행 필터링
3GROUP BY그룹화 (Day 4에서 학습)
4HAVING그룹 필터링 (Day 4에서 학습)
5SELECT컬럼 선택 및 계산
6ORDER BY결과 정렬
7LIMIT결과 개수 제한

핵심 포인트: WHERE에서는 SELECT에서 정의한 별칭(alias)을 사용할 수 없다. 실행 순서상 WHERE가 SELECT보다 먼저 실행되기 때문이다.

-- 오류: WHERE에서 alias 사용 불가
SELECT salary * 12 AS annual_salary
FROM   employees
WHERE  annual_salary > 80000000;   -- 오류

-- 올바른 방법
SELECT salary * 12 AS annual_salary
FROM   employees
WHERE  salary * 12 > 80000000;

11. 종합 실습

문제 1

급여가 600만 원 이상이고 개발팀(dept_id=1)에 속한 직원의 이름과 급여를 급여 내림차순으로 조회하라.

정답 보기 
SELECT name, salary
FROM   employees
WHERE  salary >= 6000000 AND dept_id = 1
ORDER BY salary DESC;

문제 2

2023년 이후 입사자 중 급여 상위 2명의 이름, 급여, 입사일을 조회하라.

정답 보기 
SELECT name, salary, hire_date
FROM   employees
WHERE  hire_date >= '2023-01-01'
ORDER BY salary DESC
LIMIT 2;

문제 3

박민준의 부서를 인사팀(dept_id=3)으로 이동하고 급여를 580만 원으로 조정하라.

정답 보기 
UPDATE employees
SET    dept_id = 3, salary = 5800000
WHERE  name = '박민준';

-- 확인
SELECT * FROM employees WHERE name = '박민준';

문제 4

'이름'에 '수'가 포함된 직원을 모두 조회하라.

정답 보기 
SELECT * FROM employees WHERE name LIKE '%수%';
-- 결과: 김철수, 최수진

12. 핵심 요약

명령어핵심 포인트
SELECT원하는 컬럼만 선택, *은 전체, DISTINCT로 중복 제거
WHERE비교·논리·LIKE·IN·BETWEEN·IS NULL 연산자 조합
ORDER BY기본값 ASC, 다중 기준 가능, 실행 순서 6번째
LIMIT결과 개수 제한, OFFSET과 조합해 페이지네이션
INSERT컬럼 명시 권장, 다중 행 삽입 가능
UPDATE반드시 WHERE 확인 — 없으면 전체 수정
DELETE반드시 WHERE 확인 — 없으면 전체 삭제, TRUNCATE와 차이 인지
