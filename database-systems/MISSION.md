# Mission: 데이터베이스

## Why
백엔드 개발이나 데이터 엔지니어링 직무에서 실무 수준의 DB 설계·쿼리·최적화를 직접 처리할 수 있도록, 관계형 데이터베이스의 원리부터 실전 SQL까지 체계적으로 익힌다.

## Success looks like
- ERD를 보고 정규화된 테이블 스키마를 직접 설계할 수 있다
- 복잡한 JOIN·서브쿼리·집계를 포함한 SQL을 오류 없이 작성할 수 있다
- 슬로우 쿼리를 EXPLAIN으로 분석하고 인덱스 전략으로 개선할 수 있다
- 트랜잭션 격리 수준과 ACID 속성을 설명하고 동시성 문제를 예방할 수 있다
- 관계형 DB와 NoSQL의 차이를 이해하고 상황에 맞는 선택 근거를 제시할 수 있다

## Constraints
- 사전 지식: 프로그래밍 기초(변수·함수·반복문) 보유
- 하루 1개 레슨 페이스, 레슨당 40~60분 학습 목표
- 실습 환경: SQLite 또는 PostgreSQL (로컬 설치 또는 온라인 플레이그라운드)

## Out of scope
- Oracle PL/SQL 또는 SQL Server T-SQL 전용 문법
- 분산 데이터베이스 / 샤딩 아키텍처 (심화 과정으로 분리)
- ORM 프레임워크 내부 구현 세부사항
