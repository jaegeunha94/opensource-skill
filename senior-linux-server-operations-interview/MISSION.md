# Mission: Linux 서버 운영 인터뷰 대비

## Why
10년차 이상 백엔드/DevOps 개발자로서 Linux 서버 운영 인터뷰에서 평가받을 서버 장애 대응 능력, 원인 분석 사고방식, 실무 운영 판단력을 면접 수준에서 명확히 설명할 수 있도록 준비한다. 단순히 "어떤 명령어를 쓰나요?" 수준이 아니라, "왜 그 명령어인지", "데이터를 어떻게 해석하는지", "어떤 trade-off를 고려해서 조치를 결정했는지"까지 구조적으로 답변할 수 있는 수준을 목표로 한다.

## Success looks like
- 서버 장애 신고를 받았을 때 체계적인 순서로 접근 방법을 면접관에게 설명할 수 있다
- `top`, `vmstat`, `iostat`, `journalctl`, `ss`, `netstat`, `df`, `ps`, `systemctl` 등 명령어를 단순 암기가 아니라 "왜 이 상황에서 이 명령어인지"까지 설명할 수 있다
- CPU 병목 / 메모리 압박 / Disk I/O 병목 / 네트워크 문제를 각각 진단하는 접근법을 설명할 수 있다
- load average, iowait, OOM killer, swap 사용 등 지표를 해석하고 의미를 설명할 수 있다
- systemd 서비스 장애를 분석하고 복구하는 흐름을 설명할 수 있다
- 권한 관리와 보안 설정 이슈를 운영 관점에서 설명할 수 있다
- Senior-level follow-up 질문에 trade-off와 실무 판단 근거를 포함해서 답변할 수 있다

## Constraints
- 단순 명령어 암기가 아니라 실무 판단 능력 평가에 초점
- 10년차 이상 기준이므로 "어떤 명령어 쓰나요?" 수준의 답변은 충분하지 않음
- 각 개념은 면접 답변 형태로 정리 (나열식 암기 X, 구조적 설명 + 판단 근거 O)
- 전제 개념을 쉬운 한국어로 먼저 설명하고 기술 용어로 연결하는 방식 유지

## Out of scope
- 커널 개발, 드라이버 작성 등 저수준 시스템 프로그래밍
- Ansible, Terraform 등 인프라 자동화 도구 심화 (별도 과목)
- AWS/GCP 등 클라우드 서비스 특화 내용 (별도 과목)
- 데이터베이스 튜닝 심화 (별도 과목)
