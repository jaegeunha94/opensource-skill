# Mission: 파일구조 (File Structures)

## Why

파일구조 과목을 체계적으로 학습하여 보조기억장치에 데이터를 효율적으로 저장·검색하는 원리를 이해하고, 데이터베이스 시스템과 파일 시스템 설계의 기반을 갖추기 위해 학습한다. 이 지식은 대용량 데이터를 다루는 소프트웨어 개발 및 시스템 설계 역량으로 이어진다.

## Success looks like

- 순차 파일, 인덱스 파일, 직접 파일(해싱)의 구조와 장단점을 비교 설명할 수 있다
- B-트리와 B+-트리를 직접 그리고, 삽입·삭제 연산을 단계별로 수행할 수 있다
- 주어진 요구사항에 맞는 적절한 파일구조를 선택하고 그 이유를 논리적으로 설명할 수 있다
- 외부 정렬(External Sort) 알고리즘의 작동 원리를 설명할 수 있다
- 파일 시스템(FAT, inode)의 구조를 이해하고 디렉터리 탐색 과정을 설명할 수 있다

## Constraints

- 한국 대학교 파일구조 과목 커리큘럼 기준으로 학습
- 이론 중심 학습 (구현보다 개념과 원리 이해 우선)
- 각 세션에서 하나의 Day 레슨을 완료하는 페이스 유지

## Out of scope

- 특정 DBMS(MySQL, PostgreSQL 등)의 내부 구현 세부사항
- 운영체제 커널 수준의 파일 시스템 구현 코드
- 네트워크 파일 시스템(NFS, SMB) 프로토콜
