1. 운영체제의 정의

운영체제(Operating System, OS)는 하드웨어 자원을 관리하고, 응용 프로그램에게 일관된 환경을 제공하는 소프트웨어 계층입니다. 사용자와 하드웨어 사이의 중재자(intermediary) 역할을 합니다.

핵심 정의
운영체제는 자원 관리자(Resource Manager)이자 가상 기계(Virtual Machine)입니다. 하드웨어의 복잡성을 숨기고, 프로그램이 하드웨어를 직접 다루지 않아도 되도록 추상화합니다.

OS가 없다면 모든 프로그램이 CPU 점유, 메모리 주소, 디스크 섹터를 직접 협상해야 합니다. OS는 이 혼돈을 막기 위해 중앙 집중적 자원 할당을 담당합니다.

2. OS의 세 가지 핵심 역할

역할 설명 예시
자원 관리 CPU, 메모리, 디스크, I/O 장치를 프로세스들에게 공정하게 배분 여러 탭이 열려 있어도 브라우저가 멈추지 않는 이유
추상화 하드웨어 세부사항을 숨기고 일관된 인터페이스 제공 파일 시스템 — SSD·HDD 관계없이 open()/read() 호출
보호·격리 프로세스 간 메모리 침범 방지, 권한 없는 자원 접근 차단 한 프로그램의 크래시가 전체 시스템을 멈추지 않는 이유

3. OS 구조: 커널 · 셸 · 시스템 콜

┌─────────────────────────────────────────┐ │ 사용자 응용 프로그램 │ ← User Space │ (Chrome, VS Code, Python script …) │ ├─────────────────────────────────────────┤ │ 시스템 콜 인터페이스 │ ← 경계선 │ open(), read(), fork(), exec(), … │ ├─────────────────────────────────────────┤ │ 커 널 (Kernel) │ ← Kernel Space │ [프로세스 관리] [메모리 관리] [파일 시스템] │ │ [디바이스 드라이버] [네트워크 스택] │ ├─────────────────────────────────────────┤ │ 하드웨어 (Hardware) │ │ CPU · RAM · Disk · NIC · GPU … │ └─────────────────────────────────────────┘

그림 1: OS 계층 구조 — 커널이 하드웨어와 사용자 공간 사이에 위치

커널 (Kernel)

OS의 핵심 부분으로 항상 메모리에 상주합니다. 특권 모드(Kernel Mode)에서 실행되어 하드웨어에 직접 접근할 수 있습니다. 프로세스 스케줄링, 메모리 관리, 인터럽트 처리가 커널의 주요 책임입니다.

시스템 콜 (System Call)

사용자 프로그램이 OS 서비스를 요청하는 유일한 공식 경로입니다. 사용자 모드에서 커널 모드로 전환(trap)을 발생시킵니다. 

# 예: 파일 읽기에 숨겨진 시스템 콜 흐름
f = open("data.txt")   →  open() 시스템 콜
data = f.read()        →  read() 시스템 콜
f.close()              →  close() 시스템 콜

셸 (Shell)

사용자와 OS 사이의 명령 해석기입니다. bash, zsh, PowerShell 등이 여기에 해당합니다. 셸 자체는 커널이 아닌 일반 사용자 프로그램으로, 내부적으로 시스템 콜을 사용합니다.

4. 운영체제의 주요 유형

유형 특징 예시
배치 처리 OS 작업을 모아 순서대로 실행, 사용자 상호작용 없음 초기 IBM 메인프레임
시분할 OS CPU 시간을 잘게 쪼개어 여러 사용자/프로세스가 동시에 사용하는 것처럼 보임 Unix, Linux, Windows
실시간 OS (RTOS) 응답 시간이 엄격하게 보장되어야 하는 시스템 FreeRTOS, VxWorks (자동차 ECU, 항공기)
분산 OS 네트워크로 연결된 여러 머신을 단일 시스템처럼 관리 Google의 내부 분산 시스템
모바일 OS 배터리·터치·센서 최적화, 앱 샌드박스 강화 Android (Linux 기반), iOS (Darwin/XNU 기반)

5. 커널 구조: 모놀리식 vs 마이크로커널

핵심 트레이드오프
모놀리식 커널: 성능 우선. 모든 OS 서비스가 커널 공간에서 실행. Linux가 대표 사례.
마이크로커널: 안정성·보안 우선. 최소한의 서비스만 커널에 두고, 나머지는 사용자 공간. Mach, seL4가 대표 사례.
구조 장점 단점
모놀리식 빠른 내부 통신, 높은 성능 한 모듈 버그가 커널 전체에 영향
마이크로커널 격리된 서비스, 높은 신뢰성 IPC 오버헤드로 인한 성능 저하
하이브리드 (Windows NT) 두 방식을 절충 설계 복잡성 증가
