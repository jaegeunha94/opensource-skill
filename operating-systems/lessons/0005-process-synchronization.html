프로세스 동기화 기초

여러 프로세스 또는 스레드가 공유 자원에 동시에 접근할 때 발생하는 문제와, 이를 안전하게 다루는 방법을 배웁니다.

학습 목표
  • 경쟁 조건(Race Condition)이 무엇인지 설명할 수 있다.
  • 임계 구역(Critical Section) 문제의 세 가지 요구 조건을 열거할 수 있다.
  • 뮤텍스(Mutex)와 세마포어(Semaphore)의 차이를 구분할 수 있다.
  • 생산자-소비자 문제를 세마포어로 해결하는 구조를 이해한다.

1. 공유 자원과 경쟁 조건

두 스레드가 같은 변수를 동시에 읽고 쓰면 결과가 실행 순서에 따라 달라집니다. 이런 상황을 경쟁 조건(Race Condition)이라 합니다.

예시: 은행 계좌 잔액 갱신

// 공유 변수
int balance = 1000;

// Thread A: 입금 +500
temp_A = balance;   // temp_A = 1000
temp_A += 500;      // temp_A = 1500
balance = temp_A;   // balance = 1500

// Thread B: 출금 -200 (A와 동시에 실행될 경우)
temp_B = balance;   // temp_B = 1000  ← A가 쓰기 전에 읽음!
temp_B -= 200;      // temp_B = 800
balance = temp_B;   // balance = 800  ← A의 입금이 사라짐!

기대값 1300이지만 실제로는 800이 될 수 있습니다. 실행 순서가 결과를 결정합니다.

2. 임계 구역 (Critical Section)

공유 자원에 접근하는 코드 영역을 임계 구역이라 합니다. 임계 구역 문제를 해결하려면 다음 세 가지 조건을 모두 만족해야 합니다.

조건설명
상호 배제 (Mutual Exclusion) 한 프로세스가 임계 구역에 있으면 다른 프로세스는 진입 불가
진행 (Progress) 임계 구역 밖의 프로세스가 진입을 방해해서는 안 됨
한계 대기 (Bounded Waiting) 진입 요청 후 무한정 기다리는 프로세스가 없어야 함 (기아 방지)
[Entry Section] ← 진입 허가를 요청하는 코드 ↓ [Critical Section] ← 공유 자원에 접근하는 코드 ↓ [Exit Section] ← 임계 구역 진입 해제 ↓ [Remainder Section]← 나머지 코드

그림 1 — 임계 구역 구조

3. 뮤텍스 (Mutex)

뮤텍스(Mutual Exclusion)는 잠금(lock) / 해제(unlock) 두 가지 연산으로 임계 구역을 보호합니다. 잠금을 획득한 스레드만 임계 구역에 진입할 수 있습니다. 소유권이 있어서, 잠근 스레드만 해제할 수 있습니다.

mutex_lock(m);
  // 임계 구역 — 한 번에 하나의 스레드만 실행
  balance += 500;
mutex_unlock(m);
스핀락 vs 블로킹 뮤텍스
  • 스핀락(Spinlock): CPU를 점유한 채 반복 확인 (busy-wait). 대기 시간이 짧을 때 유리.
  • 블로킹 뮤텍스: 잠금 획득 실패 시 스레드를 sleep. CPU 낭비 없지만 컨텍스트 스위치 비용.

4. 세마포어 (Semaphore)

세마포어는 정수 카운터와 두 가지 원자적 연산으로 동기화를 제공합니다.

연산동작구 이름
wait(S) / P(S) S > 0이면 S -= 1 후 진행, 그렇지 않으면 블록 proberen (검사하다)
signal(S) / V(S) S += 1, 대기 중인 프로세스가 있으면 깨움 verhogen (증가시키다)

이진 세마포어 vs 계수 세마포어

종류카운터 범위용도
이진 세마포어0 또는 1상호 배제 (뮤텍스와 유사)
계수 세마포어0 이상 임의 정수자원 개수 관리 (예: 버퍼 슬롯)

5. 생산자-소비자 문제 (Bounded Buffer)

크기 n의 공유 버퍼에 생산자가 항목을 넣고 소비자가 꺼냅니다. 버퍼가 가득 차면 생산자가, 비어 있으면 소비자가 기다려야 합니다.

생산자 → [■ ■ ■ □ □] → 소비자 버퍼 (n=5) 세마포어 설정: mutex = 1 (버퍼 접근 상호 배제) empty = n (빈 슬롯 수) full = 0 (채워진 슬롯 수)

그림 2 — 생산자-소비자 공유 버퍼

/* 생산자 */
while (true) {
    produce_item();
    wait(empty);    // 빈 슬롯 기다림
    wait(mutex);
      add_to_buffer();
    signal(mutex);
    signal(full);   // 소비 가능 슬롯 증가
}

/* 소비자 */
while (true) {
    wait(full);     // 채워진 슬롯 기다림
    wait(mutex);
      remove_from_buffer();
    signal(mutex);
    signal(empty);  // 빈 슬롯 증가
    consume_item();
}
순서에 주의
wait(mutex)를 wait(empty/full)보다 먼저 호출하면 교착 상태(Deadlock)가 발생할 수 있습니다. 항상 자원 세마포어(empty, full)를 먼저 확인해야 합니다.

6. 뮤텍스 vs 세마포어 비교

특성뮤텍스세마포어
카운터 값0 또는 1 (잠김/열림)0 이상 임의 정수
소유권있음 (잠근 스레드만 해제)없음 (다른 스레드가 signal 가능)
주 용도임계 구역 보호자원 접근 제어, 스레드 간 시그널링
재귀 잠금일부 구현에서 지원지원하지 않음
