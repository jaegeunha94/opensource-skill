1. LIFO 원리 (Last In, First Out)

스택(Stack)은 "마지막에 넣은 것이 가장 먼저 나온다"는 LIFO(Last In, First Out) 규칙을 따르는 자료구조다. 식당의 쟁반 더미를 생각해 보자. 가장 마지막에 올려놓은 쟁반을 가장 먼저 꺼내 쓴다.

push → TOP ┌─────┐ │ E │ ← 가장 최근에 삽입된 원소 ├─────┤ │ D │ ├─────┤ │ C │ ├─────┤ │ B │ ├─────┤ │ A │ ← 가장 먼저 삽입된 원소 └─────┘ BOTTOM pop 시 E → D → C → B → A 순서로 꺼낸다.

스택의 구조 — TOP에서만 삽입(push)과 제거(pop)가 일어난다

1-1. 핵심 연산

연산설명시간복잡도
push(x)스택 TOP에 원소 x를 추가O(1)
pop()TOP 원소를 제거하고 반환. 빈 스택에서 호출 시 underflow 오류O(1)
peek() / top()TOP 원소를 제거 없이 반환O(1)
is_empty()스택이 비었는지 확인O(1)
size()현재 원소 수 반환O(1)
스택 vs 큐 — 헷갈릴 때 기억할 것

스택(Stack): LIFO — 브라우저 뒤로 가기, 실행 취소(Undo), 함수 호출 스택
큐(Queue): FIFO — 프린터 대기열, BFS, 요청 처리 순서
"마지막 것 먼저"가 자연스러운 문제라면 스택, "먼저 온 것 먼저"라면 큐.

2. 배열 기반 스택 구현

파이썬의 list(동적 배열)를 그대로 스택처럼 쓸 수 있다. list.append()가 push, list.pop()이 pop 역할을 한다 — 둘 다 상각 O(1). 

class ArrayStack:
    def __init__(self):
        self._data = []          # 내부 저장 배열

    def push(self, x):
        self._data.append(x)     # 배열 끝 = TOP

    def pop(self):
        if self.is_empty():
            raise IndexError("pop from empty stack")
        return self._data.pop()  # 배열 끝에서 제거 — O(1) amortized

    def peek(self):
        if self.is_empty():
            raise IndexError("peek from empty stack")
        return self._data[-1]    # 배열 끝 원소 조회

    def is_empty(self):
        return len(self._data) == 0

    def size(self):
        return len(self._data)

    def __repr__(self):
        return f"Stack(bottom→top): {self._data}"


# 사용 예
s = ArrayStack()
s.push(10)
s.push(20)
s.push(30)
print(s)          # Stack(bottom→top): [10, 20, 30]
print(s.pop())    # 30
print(s.peek())   # 20
print(s.size())   # 2
push(10) → [10] push(20) → [10, 20] push(30) → [10, 20, 30] pop() → [10, 20] 반환값: 30 peek() → [10, 20] 반환값: 20 (변화 없음)

3. 연결 리스트 기반 스택 구현

단일 연결 리스트의 head를 TOP으로 사용하면, push와 pop 모두 head 앞에서만 일어나므로 포인터 이동 없이 엄격한 O(1)이 보장된다. 

class _Node:
    __slots__ = ('data', 'next')
    def __init__(self, data, next_node=None):
        self.data = data
        self.next = next_node


class LinkedStack:
    def __init__(self):
        self._head = None        # TOP을 가리키는 포인터
        self._size = 0

    def push(self, x):
        self._head = _Node(x, self._head)   # 새 노드를 HEAD 앞에 연결
        self._size += 1

    def pop(self):
        if self.is_empty():
            raise IndexError("pop from empty stack")
        val = self._head.data
        self._head = self._head.next         # HEAD를 한 칸 뒤로
        self._size -= 1
        return val

    def peek(self):
        if self.is_empty():
            raise IndexError("peek from empty stack")
        return self._head.data

    def is_empty(self):
        return self._head is None

    def size(self):
        return self._size
push(A): [A] → None head = A push(B): [B] → [A] → None head = B (B가 TOP) push(C): [C] → [B] → [A] → None head = C pop() → C 반환, head = B [B] → [A] → None

4. 함수 호출 스택 (Call Stack)

프로그래밍 언어 런타임은 함수 호출을 관리하기 위해 콜 스택(call stack)을 사용한다. 함수를 호출하면 스택 프레임(stack frame)이 PUSH되고, 함수가 반환되면 POP된다. 

def c():
    return 42

def b():
    return c() + 1

def a():
    return b() + 1

result = a()   # 어떤 순서로 스택 프레임이 쌓일까?
① a() 호출 ② b() 호출 ┌────────────┐ │ b() 프레임 │ ← TOP ┌────────────┐ ├────────────┤ │ a() 프레임 │ ← TOP │ a() 프레임 │ └────────────┘ └────────────┘ ③ c() 호출 ④ c() 반환 → pop ┌────────────┐ │ c() 프레임 │ ← TOP ┌────────────┐ ├────────────┤ │ b() 프레임 │ ← TOP │ b() 프레임 │ ├────────────┤ ├────────────┤ │ a() 프레임 │ │ a() 프레임 │ └────────────┘ └────────────┘ ⑤ b() 반환 → pop ⑥ a() 반환 → pop ┌────────────┐ (스택 비어 있음) │ a() 프레임 │ ← TOP └────────────┘

콜 스택의 변화 — 호출할 때마다 PUSH, 반환할 때마다 POP

4-1. 재귀 vs 스택 반복문

재귀(recursion)는 내부적으로 콜 스택을 사용한다. 재귀 깊이가 너무 깊으면 RecursionError: maximum recursion depth exceeded가 발생한다. 이 경우 명시적 스택으로 재귀를 대체할 수 있다. 

# 재귀로 DFS
def dfs_recursive(graph, node, visited=None):
    if visited is None:
        visited = set()
    visited.add(node)
    for neighbor in graph[node]:
        if neighbor not in visited:
            dfs_recursive(graph, neighbor, visited)
    return visited

# 명시적 스택으로 DFS — 재귀 없이 동일 결과
def dfs_iterative(graph, start):
    visited = set()
    stack = [start]              # 명시적 스택 사용
    while stack:
        node = stack.pop()       # LIFO — 가장 최근 방문 후보
        if node not in visited:
            visited.add(node)
            for neighbor in graph[node]:
                if neighbor not in visited:
                    stack.append(neighbor)
    return visited

5. 응용: 괄호 검사 (Bracket Matching)

스택의 가장 유명한 응용 문제다. 주어진 문자열에서 괄호의 종류와 순서가 올바른지 확인한다.

5-1. 알고리즘 설계

  1. 문자를 왼쪽에서 오른쪽으로 순회한다.
  2. 여는 괄호 (, [, {를 만나면 스택에 PUSH한다.
  3. 닫는 괄호를 만나면:
    • 스택이 비어 있으면 → 실패 (대응하는 여는 괄호 없음)
    • 스택 TOP의 여는 괄호와 짝이 맞으면 → POP하고 계속
    • 짝이 맞지 않으면 → 실패
  4. 순회가 끝났을 때 스택이 비어 있으면 → 성공, 그렇지 않으면 실패.
입력: "({[]})" 문자 스택 상태 동작 ( [(] push ( { [(, {] push { [ [(, {, [] push [ ] [(, {] pop [ — ] 와 짝 맞음 ✓ } [(] pop { — } 와 짝 맞음 ✓ ) [] pop ( — ) 와 짝 맞음 ✓ 스택 비어 있음 → 올바른 괄호열 ✓ 입력: "([)]" 문자 스택 상태 동작 ( [(] push ( [ [(, [] push [ ) 실패 ✗ TOP=[ 이지만 ) 가 들어옴 → 짝 불일치

괄호 검사 알고리즘 추적

def is_valid_brackets(s: str) -> bool:
    MATCH = {')': '(', ']': '[', '}': '{'}
    OPEN  = set('([{')
    stack = []

    for ch in s:
        if ch in OPEN:
            stack.append(ch)
        elif ch in MATCH:
            if not stack or stack[-1] != MATCH[ch]:
                return False
            stack.pop()
    return len(stack) == 0


# 테스트
cases = [
    ("()",        True),
    ("()[]{}",   True),
    ("({[]})",   True),
    ("([)]",     False),
    ("{[]",      False),
    ("",          True),
]
for expr, expected in cases:
    result = is_valid_brackets(expr)
    status = "✓" if result == expected else "✗"
    print(f"{status}  is_valid_brackets({expr!r}) = {result}")

핵심 정리

  • LIFO — 마지막에 넣은 원소가 가장 먼저 나온다.
  • push, pop, peek 모두 O(1)이 목표 — 배열과 연결 리스트 둘 다 달성 가능.
  • 콜 스택은 함수 호출·반환 순서를 관리하는 언어 런타임의 스택이다.
  • 괄호 검사, Undo, DFS 등 "최근 상태를 빠르게 되돌아봐야 하는" 문제에 스택이 적합하다.
  • 파이썬에서는 list가 사실상 표준 스택이다. append = push, pop = pop.
