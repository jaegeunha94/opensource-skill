해시 테이블 (Hash Tables)

Day 5 2026-06-26  ·  Hash Tables — 해시 함수, 충돌 해결, O(1) 평균 탐색

학습 목표

이 레슨을 마치면 다음을 할 수 있다:

  • 해시 테이블이 O(1) 평균 탐색·삽입·삭제를 달성하는 원리를 설명할 수 있다.
  • 좋은 해시 함수의 조건을 나열하고 간단한 해시 함수를 직접 작성할 수 있다.
  • 체이닝(Chaining)과 오픈 어드레싱(Open Addressing) 두 충돌 해결 방식의 차이를 비교할 수 있다.
  • 로드 팩터(load factor)가 성능에 미치는 영향을 설명하고 리해싱이 필요한 시점을 알 수 있다.
  • Python의 dict와 set이 해시 테이블임을 알고 올바르게 활용할 수 있다.

1. 해시 테이블이란?

해시 테이블(Hash Table)은 키(key)를 해시 함수로 변환해 얻은 인덱스에 값을 저장하는 자료구조다. 배열 처럼 인덱스로 직접 접근하기 때문에 평균 O(1)의 탐색·삽입·삭제가 가능하다.

해시 테이블의 기본 구조: 키(key) → 해시 함수 → 인덱스 → 배열(버킷) 접근 "apple" → hash() → 3 → table[3] = "사과" "banana" → hash() → 7 → table[7] = "바나나" "cherry" → hash() → 1 → table[1] = "체리" 인덱스 0 1 2 3 4 5 6 7 ┌──┬────────┬──┬────────┬──┬──┬──┬────────┐ │ │"체리" │ │"사과" │ │ │ │"바나나"│ └──┴────────┴──┴────────┴──┴──┴──┴────────┘

해시 테이블 — 키를 인덱스로 변환해 배열에 직접 저장한다

왜 해시 테이블이 필요한가?
  • 배열: 인덱스로 O(1) 접근, 하지만 키가 정수여야 하고 범위가 크면 메모리 낭비.
  • 연결 리스트: 키가 무엇이든 저장 가능, 하지만 탐색에 O(n) 시간이 걸린다.
  • 해시 테이블: 어떤 키든 해시 함수로 정수 인덱스로 변환 → O(1) 접근, 메모리 효율.

2. 해시 함수 (Hash Function)

해시 함수는 임의의 키를 받아 배열 인덱스(정수)를 반환하는 함수다. index = hash(key) % table_size 형태가 가장 기본이다.

좋은 해시 함수의 조건

조건설명
결정론적 (Deterministic)같은 키는 항상 같은 인덱스를 반환해야 한다.
균등 분포 (Uniform Distribution)인덱스가 배열 전체에 고루 분산되어야 충돌이 줄어든다.
빠른 계산 (Fast Computation)O(1) 또는 O(키 길이) 내에 계산되어야 한다.
눈사태 효과 (Avalanche)키의 작은 변화가 인덱스를 크게 바꿔야 한다 (보안 해시에서 더 중요).

간단한 해시 함수 예시 (정수 키)

def hash_int(key: int, size: int) -> int:
    return key % size

# 예: 테이블 크기 10
print(hash_int(42, 10))    # 2
print(hash_int(153, 10))   # 3
print(hash_int(0, 10))     # 0

문자열 해시 함수 — 다항 롤링 해시

def hash_str(key: str, size: int, base: int = 31) -> int:
    """다항 롤링 해시: h = (c0*b^n + c1*b^(n-1) + ... + cn) % size"""
    h = 0
    for ch in key:
        h = (h * base + ord(ch)) % size
    return h

print(hash_str("apple",  1000))   # 예: 927
print(hash_str("Apple",  1000))   # 예: 895 (대소문자만 달라도 다른 값)
print(hash_str("appel",  1000))   # 예: 해당 값 (철자 순서가 달라도 다른 값)
나쁜 해시 함수 — 길이만 반환

hash_str = lambda key, size: len(key) % size 같은 함수는 키 길이가 같으면 항상 같은 인덱스가 되어 충돌이 폭증한다. O(1)은커녕 O(n)도 보장하지 못한다.

3. 충돌 (Collision)

두 개 이상의 키가 같은 인덱스로 매핑되는 현상을 충돌(Collision)이라 한다. 비둘기집 원리에 의해 키의 범위가 테이블 크기보다 크면 충돌은 필연적으로 발생한다.

충돌 예시 (테이블 크기 = 5): "apple" → hash % 5 = 2 ─┐ "grape" → hash % 5 = 2 ─┤ → 인덱스 2에서 충돌! "lemon" → hash % 5 = 4 "mango" → hash % 5 = 2 ─┘ → 세 번째 충돌!

서로 다른 키가 같은 인덱스로 매핑 — 충돌 처리 전략이 필요하다

4. 충돌 해결 1 — 체이닝 (Separate Chaining)

각 버킷(배열 슬롯)에 연결 리스트(또는 다른 자료구조)를 두고, 충돌하는 키를 같은 버킷의 리스트에 이어 붙이는 방식이다.

체이닝 예시 (테이블 크기 = 5): 인덱스 0: → None 인덱스 1: → ("fig", 1500) → None 인덱스 2: → ("apple", 800) → ("grape", 300) → ("mango", 200) → None 인덱스 3: → ("banana", 500) → None 인덱스 4: → ("lemon", 900) → None

체이닝 — 같은 인덱스의 충돌 항목을 연결 리스트로 관리한다

class ChainingHashTable:
    def __init__(self, size: int = 16):
        self._size = size
        self._buckets = [[] for _ in range(size)]  # 각 버킷은 (key, value) 리스트

    def _hash(self, key) -> int:
        return hash(key) % self._size

    def put(self, key, value) -> None:
        idx = self._hash(key)
        for i, (k, v) in enumerate(self._buckets[idx]):
            if k == key:
                self._buckets[idx][i] = (key, value)  # 키 존재 시 업데이트
                return
        self._buckets[idx].append((key, value))        # 새 항목 추가

    def get(self, key):
        idx = self._hash(key)
        for k, v in self._buckets[idx]:
            if k == key:
                return v
        raise KeyError(key)

    def delete(self, key) -> None:
        idx = self._hash(key)
        bucket = self._buckets[idx]
        for i, (k, v) in enumerate(bucket):
            if k == key:
                bucket.pop(i)
                return
        raise KeyError(key)


# 사용 예
ht = ChainingHashTable()
ht.put("apple", 800)
ht.put("grape", 300)
ht.put("apple", 850)    # 업데이트
print(ht.get("apple"))  # 850
ht.delete("grape")

체이닝의 시간복잡도

연산평균최악 (모든 키 충돌)
탐색(get)O(1)O(n)
삽입(put)O(1)O(n)
삭제(delete)O(1)O(n)

5. 충돌 해결 2 — 오픈 어드레싱 (Open Addressing)

충돌이 발생하면 같은 배열 내의 다른 빈 슬롯을 탐색해서 저장하는 방식이다. 모든 데이터가 배열 안에만 존재하므로 포인터 오버헤드가 없다.

선형 탐사 (Linear Probing)

충돌 시 (index + 1) % size씩 순차적으로 다음 빈 슬롯을 탐색한다.

선형 탐사 예시 (테이블 크기 = 7): "apple" → idx 2 → 비어 있음 → table[2] = "apple" "grape" → idx 2 → 충돌! → 탐사 → table[3] = "grape" "cherry" → idx 3 → 충돌! → 탐사 → table[4] = "cherry" 인덱스: 0 1 2 3 4 5 6 [ _ | _ |"apple" |"grape" |"cherry" | _ | _ ] "lime" → idx 4 → 충돌! → idx5 비어있음 → table[5] = "lime"

선형 탐사 — 충돌 시 한 칸씩 오른쪽으로 이동해 빈 슬롯을 찾는다 (클러스터링 위험)

이차 탐사 (Quadratic Probing)

충돌 시 탐사 간격을 1, 4, 9, 16, …(i²)로 늘려 클러스터링을 줄인다.

# 이차 탐사 인덱스 계산
index = (initial_hash + i * i) % size   # i = 1, 2, 3, …

이중 해싱 (Double Hashing)

두 번째 해시 함수로 탐사 간격을 결정해 클러스터링을 크게 줄인다.

# 이중 해싱 인덱스 계산
step = hash2(key)                         # 두 번째 해시 함수
index = (initial_hash + i * step) % size  # i = 1, 2, 3, …

class OpenAddressingHashTable:
    _DELETED = object()   # 삭제된 슬롯을 나타내는 sentinel

    def __init__(self, size: int = 16):
        self._size = size
        self._slots = [None] * size
        self._count = 0

    def _hash(self, key) -> int:
        return hash(key) % self._size

    def put(self, key, value) -> None:
        if self._count / self._size > 0.7:
            self._resize()
        idx = self._hash(key)
        i = 0
        while self._slots[(idx + i) % self._size] not in (None, self._DELETED):
            slot_key, _ = self._slots[(idx + i) % self._size]
            if slot_key == key:
                self._slots[(idx + i) % self._size] = (key, value)  # 업데이트
                return
            i += 1
        self._slots[(idx + i) % self._size] = (key, value)
        self._count += 1

    def get(self, key):
        idx = self._hash(key)
        i = 0
        while self._slots[(idx + i) % self._size] is not None:
            slot = self._slots[(idx + i) % self._size]
            if slot is not self._DELETED and slot[0] == key:
                return slot[1]
            i += 1
            if i == self._size:
                break
        raise KeyError(key)

    def delete(self, key) -> None:
        idx = self._hash(key)
        i = 0
        while self._slots[(idx + i) % self._size] is not None:
            slot = self._slots[(idx + i) % self._size]
            if slot is not self._DELETED and slot[0] == key:
                self._slots[(idx + i) % self._size] = self._DELETED  # lazy 삭제
                self._count -= 1
                return
            i += 1
            if i == self._size:
                break
        raise KeyError(key)

    def _resize(self) -> None:
        old_slots = self._slots
        self._size *= 2
        self._slots = [None] * self._size
        self._count = 0
        for slot in old_slots:
            if slot and slot is not self._DELETED:
                self.put(slot[0], slot[1])
삭제 주의 — Lazy 삭제가 필요한 이유

오픈 어드레싱에서 슬롯을 단순히 None으로 비우면 탐사 체인이 끊겨 이후 탐색 시 존재하는 키를 찾지 못한다. 반드시 sentinel(삭제 마커)을 두고 나중에 일괄 정리(리해싱)한다.

7. 로드 팩터와 리해싱 (Load Factor & Rehashing)

로드 팩터(α) = 저장된 항목 수 / 테이블 크기. α가 높을수록 충돌이 많아져 성능이 저하된다.

로드 팩터별 성능 (체이닝 기준): α = 0.1 → 버킷당 평균 0.1개 → 거의 충돌 없음 → O(1) 탐색 α = 0.5 → 버킷당 평균 0.5개 → 충돌 적음 → O(1) 탐색 α = 1.0 → 버킷당 평균 1.0개 → 충돌 보통 → O(1) 탐색 (체이닝은 여전히 O(1)) α = 5.0 → 버킷당 평균 5.0개 → 충돌 많음 → 점점 O(n) 탐색에 수렴 오픈 어드레싱 권장 범위: α < 0.7 (Python dict는 α > 2/3 시 자동 리해싱)

로드 팩터가 높을수록 충돌 빈도가 늘어 성능이 저하된다

리해싱(Rehashing): 로드 팩터가 임계값을 초과하면 더 큰 배열(보통 2배)을 새로 만들고 모든 항목을 재삽입한다. 리해싱 자체는 O(n)이지만, 분할 상환 분석(amortized analysis)으로 삽입 1회당 O(1)이 된다. 

# Python dict의 리해싱 임계값 확인 (CPython 내부)
# dict는 로드 팩터가 2/3을 넘으면 자동으로 리해싱된다
import sys

d = {}
prev_size = sys.getsizeof(d)
for i in range(20):
    d[i] = i
    cur_size = sys.getsizeof(d)
    if cur_size != prev_size:
        print(f"크기 증가: len={len(d)}, size={cur_size} bytes")
        prev_size = cur_size

8. Python dict와 set

Python의 dict와 set은 둘 다 해시 테이블로 구현되어 평균 O(1)의 탐색·삽입·삭제를 제공한다. 

# dict — 해시 테이블 (키: 해시 가능한 값, 값: 임의의 값)
prices = {"apple": 800, "banana": 500, "grape": 300}
prices["lemon"] = 900           # 삽입 O(1)
print(prices["apple"])          # 탐색 O(1)  → 800
del prices["banana"]            # 삭제 O(1)
print("grape" in prices)        # 멤버십 O(1) → True

# set — 해시 테이블 (중복 없는 키만 저장)
fruits = {"apple", "banana", "apple"}
print(fruits)                   # {'apple', 'banana'} — 중복 제거

# 교집합·합집합·차집합 모두 O(n)
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}
print(a & b)   # 교집합: {3, 4}
print(a | b)   # 합집합: {1, 2, 3, 4, 5, 6}
print(a - b)   # 차집합: {1, 2}

해시 가능한(hashable) vs 불가능한 타입

해시 가능 (dict 키 가능)해시 불가능 (dict 키 불가)
int, float, str, tuple(불변 요소만), frozensetlist, dict, set (변경 가능하기 때문)

9. 실전 응용 — 빈도 집계와 중복 탐지

빈도 집계 (Frequency Count)

from collections import Counter

words = ["apple", "banana", "apple", "cherry", "banana", "apple"]

# Counter는 dict를 상속한 해시 테이블
freq = Counter(words)
print(freq)            # Counter({'apple': 3, 'banana': 2, 'cherry': 1})
print(freq["apple"])   # 3

# 직접 구현
freq2 = {}
for w in words:
    freq2[w] = freq2.get(w, 0) + 1
print(freq2)           # {'apple': 3, 'banana': 2, 'cherry': 1}

두 합계 (Two Sum) — LeetCode #1

def two_sum(nums: list, target: int) -> list:
    """해시 테이블로 O(n) 풀이 — 나머지(complement)를 저장한다."""
    seen = {}                           # {값: 인덱스}
    for i, n in enumerate(nums):
        complement = target - n
        if complement in seen:          # O(1) 탐색
            return [seen[complement], i]
        seen[n] = i
    return []

print(two_sum([2, 7, 11, 15], 9))      # [0, 1]
print(two_sum([3, 2, 4], 6))           # [1, 2]
해시 테이블이 어울리는 문제 패턴
  • 중복 탐지 (Duplicate detection): set으로 방문 여부 O(1) 확인.
  • 빈도 집계 (Frequency counting): Counter 또는 defaultdict(int).
  • 빠른 탐색 (Fast lookup): 값→인덱스 매핑 (Two Sum 패턴).
  • 그룹화 (Grouping): defaultdict(list)로 키별 항목 묶기.
  • 캐싱 (Memoization): 계산 결과를 dict에 저장해 재계산 방지.

핵심 정리

이 레슨의 핵심
  • 해시 테이블은 해시 함수로 키를 인덱스로 변환해 O(1) 평균 탐색·삽입·삭제를 달성한다.
  • 충돌은 필연적이다 — 체이닝(외부 연결 리스트)이나 오픈 어드레싱(내부 탐사)으로 해결한다.
  • 로드 팩터(α)가 높아지면 충돌이 늘고 성능이 저하 — 임계값 초과 시 리해싱으로 배열을 2배 확장한다.
  • Python의 dict와 set이 해시 테이블 구현이다 — 변경 불가능한 타입만 키로 사용할 수 있다.
  • 최악의 경우 O(n)이지만, 좋은 해시 함수 + 적절한 로드 팩터 유지 시 실제로는 거의 O(1)이다.
