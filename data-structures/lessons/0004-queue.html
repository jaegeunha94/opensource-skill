1. FIFO 원리

큐(Queue)는 한쪽(rear)에서 데이터를 넣고 반대쪽(front)에서 꺼내는 선형 자료구조다. 가장 먼저 들어온 것이 가장 먼저 나가기 때문에 FIFO(First-In First-Out)라고 부른다. 줄 서기(대기열)를 생각하면 된다.

enqueue(A) → enqueue(B) → enqueue(C) front rear ↓ ↓ ┌───┬───┬───┐ │ A │ B │ C │ └───┴───┴───┘ dequeue() → A 반환 front rear ↓ ↓ ┌───┬───┐ │ B │ C │ └───┴───┘

큐의 FIFO 구조 — front에서 꺼내고 rear에 넣는다

일상 속 큐
  • 은행·카페 대기줄: 먼저 온 손님을 먼저 처리한다.
  • 프린터 대기열: 먼저 보낸 문서를 먼저 출력한다.
  • 키보드 입력 버퍼: 입력 순서대로 처리한다.
  • BFS 탐색: 가까운 노드부터 탐색하기 위해 큐를 사용한다.
  • OS 프로세스 스케줄링: 라운드 로빈 방식에서 실행 대기 큐 사용.

2. 핵심 연산과 시간복잡도

연산설명시간복잡도
enqueue(x)rear에 원소 x를 추가O(1)
dequeue()front 원소를 제거하고 반환O(1)*
peek() / front()front 원소를 제거 없이 확인O(1)

배열 기반 큐의 함정

Python list에서 pop(0)으로 dequeue를 구현하면 O(n)이 된다. 앞 원소를 제거한 후 나머지 원소를 한 칸씩 앞으로 당겨야 하기 때문이다. 올바른 방법은 원형 큐(circular queue) 또는 collections.deque를 사용하는 것이다.

3. 원형 큐 (Circular Queue)

고정 크기 배열로 큐를 구현할 때, 매번 원소를 앞으로 당기는 대신 front/rear 포인터를 원형으로 이동시켜 공간을 재활용한다. 이 구조를 원형 큐(circular queue, ring buffer)라고 한다.

크기 5인 원형 큐 (인덱스 0~4): 초기: front=0, rear=0, size=0 [ _ | _ | _ | _ | _ ] ↑f,r enqueue(A,B,C): front=0, rear=3, size=3 [ A | B | C | _ | _ ] ↑f ↑r dequeue() → A: front=1, rear=3, size=2 [ _ | B | C | _ | _ ] ↑f ↑r enqueue(D,E): front=1, rear=0 (wrap!), size=4 [ E | B | C | D | _ ] ↑r ↑f 포인터가 배열 끝을 넘으면 0으로 돌아온다 (mod 연산)

원형 큐 — rear가 끝에 닿으면 배열 앞으로 돌아가 공간을 재활용한다

class CircularQueue:
    def __init__(self, capacity: int):
        self._data = [None] * capacity
        self._capacity = capacity
        self._front = 0
        self._size = 0

    def enqueue(self, value):
        if self._size == self._capacity:
            raise OverflowError("queue is full")
        rear = (self._front + self._size) % self._capacity
        self._data[rear] = value
        self._size += 1

    def dequeue(self):
        if self._size == 0:
            raise IndexError("dequeue from empty queue")
        value = self._data[self._front]
        self._data[self._front] = None          # GC를 위해 참조 제거
        self._front = (self._front + 1) % self._capacity
        self._size -= 1
        return value

    def peek(self):
        if self._size == 0:
            raise IndexError("peek from empty queue")
        return self._data[self._front]

    def is_empty(self):
        return self._size == 0

    def is_full(self):
        return self._size == self._capacity

    def size(self):
        return self._size


# 사용 예
cq = CircularQueue(4)
cq.enqueue(10)
cq.enqueue(20)
cq.enqueue(30)
print(cq.dequeue())   # 10
cq.enqueue(40)
cq.enqueue(50)        # rear가 wrap around
print(cq.size())      # 4 (가득 참)

4. 연결 리스트 기반 큐 (무제한 크기)

고정 크기 제한이 없는 큐가 필요하면 연결 리스트를 사용한다. tail(rear) 포인터를 유지해 enqueue를 O(1)에 수행한다. 

class Node:
    def __init__(self, value, next_node=None):
        self.value = value
        self.next = next_node


class LinkedQueue:
    def __init__(self):
        self._head = None   # front
        self._tail = None   # rear
        self._size = 0

    def enqueue(self, value):
        new_node = Node(value)
        if self._tail is None:          # 빈 큐
            self._head = self._tail = new_node
        else:
            self._tail.next = new_node
            self._tail = new_node
        self._size += 1

    def dequeue(self):
        if self.is_empty():
            raise IndexError("dequeue from empty queue")
        value = self._head.value
        self._head = self._head.next
        if self._head is None:          # 큐가 비어진 경우
            self._tail = None
        self._size -= 1
        return value

    def peek(self):
        if self.is_empty():
            raise IndexError("peek from empty queue")
        return self._head.value

    def is_empty(self):
        return self._head is None

    def size(self):
        return self._size
enqueue(A), enqueue(B), enqueue(C) 후: head → [A|→] → [B|→] → [C|None] ↑ tail dequeue() → A 반환: head → [B|→] → [C|None] ↑ tail

연결 리스트 큐: head에서 dequeue, tail에서 enqueue — 둘 다 O(1)

5. Python collections.deque — 실무에서 가장 빠른 선택

Python 표준 라이브러리의 collections.deque는 이중 연결 리스트 기반으로 양쪽 끝에서 O(1) 연산을 제공한다. 큐와 덱(deque) 모두 구현할 수 있다. 

from collections import deque

# 큐로 사용
q = deque()
q.append(1)      # enqueue — rear에 추가
q.append(2)
q.append(3)
print(q.popleft())   # dequeue — front에서 제거 → 1
print(q.popleft())   # 2

# 덱(양방향)으로 사용
dq = deque()
dq.appendleft(10)    # front에 추가
dq.append(20)        # rear에 추가
dq.appendleft(5)     # front에 추가
print(dq)            # deque([5, 10, 20])
print(dq.pop())      # rear에서 제거 → 20
print(dq.popleft())  # front에서 제거 → 5

덱(Deque)의 주요 연산

연산설명시간복잡도
append(x)rear에 추가 (큐 enqueue)O(1)
appendleft(x)front에 추가O(1)
pop()rear에서 제거 (스택 pop)O(1)
popleft()front에서 제거 (큐 dequeue)O(1)
rotate(k)k 칸 오른쪽 회전 (음수면 왼쪽)O(k)
maxlen최대 크기 지정 — 넘으면 반대쪽 원소 자동 제거—
큐 구현 선택 가이드
  • Python 실무: collections.deque — 구현 없이 바로 사용, O(1) 보장.
  • 고정 크기 버퍼: 원형 큐 (링 버퍼) — 네트워크 패킷, 오디오 스트림 처리.
  • 학습·면접 구현: 연결 리스트 기반 큐 — 내부 동작을 명확히 보여준다.
  • 절대 사용 금지: list.pop(0) — O(n)이라 큐에 부적합.

6. 응용 — BFS(너비 우선 탐색)

그래프나 트리에서 가까운 노드부터 탐색하는 BFS는 큐를 핵심 자료구조로 사용한다. 방문할 노드를 큐에 enqueue하고, dequeue하며 인접 노드를 확인한다.

그래프 예: 1 / \ 2 3 / \ 4 5 BFS 탐색 순서 (시작: 1): 단계 큐 상태 방문 0 [1] — 1 [2, 3] 1 2 [3, 4, 5] 2 3 [4, 5] 3 4 [5] 4 5 [] 5 방문 순서: 1 → 2 → 3 → 4 → 5

BFS — 레벨(깊이)별로 넓게 퍼져 나가며 탐색한다

from collections import deque

def bfs(graph: dict, start: int) -> list:
    visited = set()
    queue = deque([start])
    visited.add(start)
    order = []

    while queue:
        node = queue.popleft()   # dequeue
        order.append(node)

        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)   # enqueue

    return order


# 사용 예
graph = {
    1: [2, 3],
    2: [4, 5],
    3: [],
    4: [],
    5: [],
}
print(bfs(graph, 1))   # [1, 2, 3, 4, 5]

BFS 시간·공간복잡도

복잡도값이유
시간O(V + E)모든 노드(V)와 간선(E)을 한 번씩 처리
공간O(V)최악의 경우 모든 노드가 큐에 들어감
BFS vs DFS — 어떤 탐색이 더 적합한가?
  • 최단 경로 (가중치 없음): BFS — 레벨별 탐색이 자동으로 최단 거리를 보장.
  • 연결 요소 찾기: BFS 또는 DFS 모두 가능.
  • 깊이 우선, 메모리 절약: DFS (재귀 또는 명시적 스택, Day 12에서 상세 학습).

핵심 정리

이 레슨의 핵심
  • 큐는 FIFO — 먼저 들어온 것이 먼저 나간다.
  • enqueue, dequeue, peek 모두 O(1) (구현에 따라).
  • Python에서는 collections.deque가 최선의 큐 구현이다. list.pop(0)은 절대 사용하지 않는다.
  • 원형 큐는 고정 크기 배열로 공간을 효율적으로 재활용한다.
  • BFS는 큐의 핵심 응용 — 가중치 없는 그래프에서 최단 경로를 보장한다.
