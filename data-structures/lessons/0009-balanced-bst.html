1. BST의 편향 문제

Day 7에서 이진 탐색 트리(BST)를 배웠다. BST의 탐색·삽입·삭제는 평균 O(log n)이지만, 최악의 경우 O(n)으로 떨어진다.

이 최악 케이스는 언제 발생하는가? 데이터가 이미 정렬된 순서로 삽입될 때다:

[1, 2, 3, 4, 5]를 순서대로 BST에 삽입하면: 1 \ 2 \ 3 \ 4 \ 5 → 연결 리스트처럼 편향됨. 탐색 5를 찾으려면 5번 비교 — O(n)

정렬된 데이터 삽입 시 BST는 연결 리스트로 퇴화한다.

핵심 통찰

BST의 성능은 트리 높이(height)에 달려 있다. n개 노드에서 높이가 O(log n)이면 빠르고, O(n)이면 느리다. 균형 트리는 삽입·삭제 후 자동으로 재정렬하여 높이를 O(log n)으로 유지한다.

2. AVL 트리

AVL 트리는 1962년 Adelson-Velsky와 Landis가 발표한 최초의 자가 균형 BST다. 모든 노드에서 왼쪽 서브트리 높이와 오른쪽 서브트리 높이의 차이(균형 인수)가 최대 1이어야 한다는 조건으로 균형을 보장한다.

2-1. 균형 인수 (Balance Factor)

BF(node) = height(left subtree) - height(right subtree)

AVL 조건: 모든 노드에서 BF ∈ {-1, 0, +1}

BF ≤ -2 또는 BF ≥ +2이면 불균형 → 회전 필요.

균형 AVL 트리: 불균형 (BF=+2): 10 (BF=0) 10 (BF=+2) / \ / 5(0) 15(0) 5 (BF=+1) / \ / \ / 3 7 12 20 3 (BF=0) 각 노드의 BF: |BF| ≤ 1 ✓ 10의 BF = height(left)−height(right) = 2−0 = +2 ✗

AVL 조건 위반 시 회전으로 균형을 복원한다.

2-2. 4가지 불균형 케이스와 회전

불균형이 발생하는 위치와 방향에 따라 4가지 케이스로 분류된다:

케이스발생 조건해결 방법
LL (Left-Left)노드의 왼쪽 자식의 왼쪽에 삽입우회전 1번
RR (Right-Right)노드의 오른쪽 자식의 오른쪽에 삽입좌회전 1번
LR (Left-Right)노드의 왼쪽 자식의 오른쪽에 삽입좌회전 후 우회전
RL (Right-Left)노드의 오른쪽 자식의 왼쪽에 삽입우회전 후 좌회전

2-3. LL 케이스 — 우회전 (Right Rotation)

삽입 전 (LL 불균형 — BF=+2 at z): z (BF=+2) / y (BF=+1) / x 우회전(z를 y 기준으로 오른쪽으로 회전): y / \ x z 회전 후 모든 BF ∈ {-1, 0, +1} ✓

LL 케이스: z를 y 기준으로 우회전하면 균형이 복원된다.

2-4. LR 케이스 — 이중 회전

삽입 전 (LR 불균형 — BF=+2 at z): z (BF=+2) / y (BF=-1) \ x Step 1 — y를 x 기준으로 좌회전: z (BF=+2) / x / y Step 2 — z를 x 기준으로 우회전: x / \ y z

LR 케이스: 두 번의 회전(좌회전 → 우회전)으로 균형 복원.

2-5. 회전 연산 코드 (파이썬)

class AVLNode:
    def __init__(self, key):
        self.key = key
        self.left = self.right = None
        self.height = 1

def height(node):
    return node.height if node else 0

def bf(node):                          # Balance Factor
    return height(node.left) - height(node.right) if node else 0

def update_height(node):
    node.height = 1 + max(height(node.left), height(node.right))

def right_rotate(y):                   # y를 피벗으로 우회전
    x = y.left
    T2 = x.right
    x.right = y
    y.left = T2
    update_height(y)
    update_height(x)
    return x                           # 새 서브트리 루트

def left_rotate(x):                    # x를 피벗으로 좌회전
    y = x.right
    T2 = y.left
    y.left = x
    x.right = T2
    update_height(x)
    update_height(y)
    return y

def insert(node, key):
    # 1. 일반 BST 삽입
    if not node:
        return AVLNode(key)
    if key < node.key:
        node.left = insert(node.left, key)
    elif key > node.key:
        node.right = insert(node.right, key)
    else:
        return node                    # 중복 무시

    update_height(node)
    b = bf(node)

    # 2. 불균형이면 회전으로 복원
    if b > 1 and key < node.left.key:    # LL
        return right_rotate(node)
    if b < -1 and key > node.right.key:  # RR
        return left_rotate(node)
    if b > 1 and key > node.left.key:    # LR
        node.left = left_rotate(node.left)
        return right_rotate(node)
    if b < -1 and key < node.right.key:  # RL
        node.right = right_rotate(node.right)
        return left_rotate(node)

    return node
AVL 트리 복잡도 요약
연산시간 복잡도비고
탐색O(log n)높이 ≤ 1.44 log₂(n+2)
삽입O(log n)회전 최대 2번
삭제O(log n)회전 O(log n)번 가능
공간O(n)각 노드에 높이 1 int 추가

3. Red-Black 트리

Red-Black 트리(RB 트리)는 각 노드에 색상(빨강 또는 검정)을 부여하여 균형을 유지하는 BST다. AVL보다 엄격하지 않은 균형 조건을 사용하므로 삽입·삭제가 더 빠르고, 탐색은 AVL보다 약간 느릴 수 있다. 실제로 Linux 커널의 완전 공정 스케줄러(CFS), Java의 TreeMap, C++ STL의 std::map이 RB 트리를 사용한다.

3-1. Red-Black 트리의 5가지 규칙

  1. 색상 규칙: 모든 노드는 빨강(R) 또는 검정(B)이다.
  2. 루트 규칙: 루트는 항상 검정이다.
  3. 리프 규칙: 모든 NIL 리프(빈 노드)는 검정이다.
  4. 빨강 규칙: 빨간 노드의 두 자식은 항상 검정이다. (빨강–빨강 연속 금지)
  5. 흑색 높이 규칙: 임의의 노드에서 모든 리프까지의 경로에 있는 검정 노드의 수가 동일하다.
유효한 Red-Black 트리 (B=검정, R=빨강): 13(B) / \ 8(R) 17(R) / \ / \ 1(B) 11(B) 15(B) 25(B) \ \ \ 6(R) 16(R) 27(R) 루트 → 리프 각 경로의 검정 노드 수: 모두 3 ✓ 빨강 노드(8,17,6,16,27)의 자식은 모두 검정 ✓

흑색 높이(Black-Height)가 동일하면 최장 경로는 최단 경로의 최대 2배 이내다.

3-2. 높이 보장 원리

흑색 높이 규칙과 빨강 규칙의 조합이 높이를 제한한다:

  • 최단 경로: 모두 검정 노드 → 길이 = 흑색 높이(bh)
  • 최장 경로: 빨강–검정–빨강–검정… 교대 → 길이 ≤ 2·bh

따라서 전체 높이 h ≤ 2 log₂(n+1), 즉 O(log n)이 보장된다.

3-3. 삽입 후 복원 — 삼촌 색상에 따른 처리

새 노드는 항상 빨강으로 삽입된다. 부모도 빨강이면(빨강 규칙 위반) 아래 세 케이스로 복원한다:

케이스삼촌 색상처리
Case 1빨강부모·삼촌 → 검정, 조부모 → 빨강으로 색 변경(Recoloring). 조부모를 다시 확인.
Case 2검정 (삼각형 배치)부모 기준으로 회전 → Case 3으로 변환.
Case 3검정 (직선 배치)조부모 기준으로 회전 + 부모·조부모 색 교환.
Recoloring vs Rotation

삼촌이 빨강이면 색 변경(O(1))만으로 해결되고, 문제가 위로 올라갈 수 있다. 삼촌이 검정이면 회전(O(1))으로 즉시 확정된다. 삽입 시 회전은 최대 2번이다.

실무 선택 기준
  • 읽기 중심(탐색 많음): AVL — 더 낮은 트리 높이로 캐시 효율 우수.
  • 쓰기 중심(삽입·삭제 많음): Red-Black — 회전 횟수가 적어 쓰기 비용 낮음.
  • 파이썬에서는? 내장 dict는 해시 테이블이고, 정렬 순서가 필요하면 sortedcontainers.SortedList를 사용한다.

5. 파이썬 실전 — sortedcontainers

파이썬 표준 라이브러리에 AVL이나 RB 트리 구현체는 없다. 실전에서는 sortedcontainers 라이브러리의 SortedList를 사용한다. 내부적으로 정렬된 배열 목록(list-of-lists) 구조로 O(log n) 삽입·탐색을 제공한다. 

from sortedcontainers import SortedList

sl = SortedList()
sl.add(5)
sl.add(3)
sl.add(7)
sl.add(1)

print(sl)              # SortedList([1, 3, 5, 7])
print(sl[0])           # 1  — 최솟값 O(1)
print(sl[-1])          # 7  — 최댓값 O(1)
print(sl.bisect_left(4))   # 2  — 4가 들어갈 인덱스 O(log n)

# k번째 순위 원소 O(log n)
print(sl[2])           # 5

# 범위 조회 O(log n + k)
print(list(sl.irange(3, 6)))  # [3, 5]

sl.remove(3)
print(sl)              # SortedList([1, 5, 7])
코딩 테스트 팁

코딩 테스트에서 "정렬 순서 유지 + 빠른 삽입/삭제/탐색"이 필요하면 SortedList를 바로 사용하라. heapq는 최솟값 추출만 O(log n)이고 임의 삭제는 O(n)이다. SortedList는 임의 삭제도 O(log n)이다.
