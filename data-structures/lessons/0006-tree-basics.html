1. 트리(Tree)란?

트리는 계층적(hierarchical) 관계를 표현하는 비선형 자료구조다. 노드(node)와 엣지(edge)로 이루어지며, 하나의 루트(root)에서 출발해 자식 노드들로 가지가 뻗어 나간다. 사이클이 없는 연결 그래프이기도 하다.

트리 구조 예시: A ← 루트 (root) / \ B C ← 내부 노드 (internal node) / \ \ D E F ← 리프 (leaf) • 루트: A • A의 자식: B, C / B의 자식: D, E / C의 자식: F • D, E, F: 자식이 없는 리프 노드 • 깊이(depth) — 루트에서 해당 노드까지의 엣지 수 → A=0, B=1, D=2 • 높이(height) — 해당 노드에서 가장 먼 리프까지의 엣지 수 → A=2, B=1, D=0

트리의 기본 구조 — 계층적 부모-자식 관계

트리 vs 그래프
  • 트리는 그래프의 특수한 형태 — 사이클 없음(acyclic), 연결(connected).
  • N개 노드의 트리는 항상 N-1개의 엣지를 가진다.
  • 임의의 두 노드 사이에 정확히 하나의 경로(path)가 존재한다.

3. 이진 트리 (Binary Tree)

각 노드가 최대 2개의 자식(왼쪽, 오른쪽)을 가지는 트리. 가장 많이 활용되는 트리 구조로, BST·힙·트라이의 기반이 된다.

이진 트리의 종류

① 포화 이진 트리 (Full/Perfect Binary Tree) 모든 내부 노드가 자식 2개, 모든 리프가 같은 레벨: A / \ B C / \ / \ D E F G ② 완전 이진 트리 (Complete Binary Tree) 마지막 레벨을 제외한 모든 레벨이 꽉 참, 마지막 레벨은 왼쪽부터 채움: A / \ B C / \ / D E F ← G 자리가 비어도 왼쪽부터 채움 ③ 편향 이진 트리 (Skewed Binary Tree) 모든 노드가 한쪽 방향으로만 자식을 가짐: A \ B \ C ← 사실상 연결 리스트 — 탐색 O(n) 저하

이진 트리 세 가지 형태 비교

종류특징높이
포화 이진 트리모든 레벨이 완전히 채워짐. 노드 수 = 2h+1-1O(log n)
완전 이진 트리왼쪽부터 채워짐. 배열로 표현 가능. 힙의 기반.O(log n)
편향 이진 트리최악의 경우. 연결 리스트와 동일한 성능.O(n)

Python으로 이진 트리 노드 정의

class TreeNode:
    def __init__(self, val=0):
        self.val = val
        self.left = None
        self.right = None

# 위 예시 트리 직접 구성
root = TreeNode('A')
root.left  = TreeNode('B')
root.right = TreeNode('C')
root.left.left  = TreeNode('D')
root.left.right = TreeNode('E')
root.right.right = TreeNode('F')

4. 트리 순회 (Tree Traversal)

모든 노드를 빠짐없이 한 번씩 방문하는 방법. 순서에 따라 4가지로 나뉜다.

순회 예시 트리: 1 / \ 2 3 / \ 4 5

4-1. 전위 순회 (Pre-order): 루트 → 왼쪽 → 오른쪽

결과: 1 → 2 → 4 → 5 → 3

활용: 트리 복사, 트리를 직렬화(serialize)할 때.

def preorder(node):
    if node is None:
        return
    print(node.val)      # ① 현재 노드 처리
    preorder(node.left)  # ② 왼쪽 서브트리
    preorder(node.right) # ③ 오른쪽 서브트리

4-2. 중위 순회 (In-order): 왼쪽 → 루트 → 오른쪽

결과: 4 → 2 → 5 → 1 → 3

활용: 이진 탐색 트리(BST)에서 중위 순회를 하면 정렬된 순서로 노드가 출력된다.

def inorder(node):
    if node is None:
        return
    inorder(node.left)   # ① 왼쪽 서브트리
    print(node.val)      # ② 현재 노드 처리
    inorder(node.right)  # ③ 오른쪽 서브트리

4-3. 후위 순회 (Post-order): 왼쪽 → 오른쪽 → 루트

결과: 4 → 5 → 2 → 3 → 1

활용: 자식을 모두 처리한 뒤 부모를 처리해야 할 때 — 디렉터리 용량 계산, 트리 삭제.

def postorder(node):
    if node is None:
        return
    postorder(node.left)  # ① 왼쪽 서브트리
    postorder(node.right) # ② 오른쪽 서브트리
    print(node.val)       # ③ 현재 노드 처리

4-4. 레벨 순서 순회 (Level-order / BFS)

결과: 1 → 2 → 3 → 4 → 5

활용: 같은 깊이(레벨)의 노드를 묶어 처리할 때, 최단 경로 탐색(BFS 기반).

from collections import deque

def level_order(root):
    if root is None:
        return
    queue = deque([root])
    while queue:
        node = queue.popleft()
        print(node.val)
        if node.left:
            queue.append(node.left)
        if node.right:
            queue.append(node.right)
순회 4종 비교 요약
순회순서결과 (위 트리)대표 활용
Pre-order루트→좌→우1 2 4 5 3트리 직렬화, 복사
In-order좌→루트→우4 2 5 1 3BST 정렬 출력
Post-order좌→우→루트4 5 2 3 1자식 먼저 처리(삭제·용량)
Level-order레벨별 좌→우1 2 3 4 5BFS, 레벨별 처리

5. 반복(Iterative) 방식 순회

재귀 대신 스택이나 큐를 명시적으로 사용하면 콜 스택 오버플로를 피할 수 있다. 특히 트리 깊이가 매우 클 때 유용하다.

전위 순회 — 스택 활용

def preorder_iterative(root):
    if not root:
        return
    stack = [root]
    while stack:
        node = stack.pop()
        print(node.val)
        # 오른쪽을 먼저 push해야 왼쪽이 먼저 pop됨
        if node.right:
            stack.append(node.right)
        if node.left:
            stack.append(node.left)

중위 순회 — 스택 활용

def inorder_iterative(root):
    stack = []
    curr = root
    while curr or stack:
        while curr:             # 가장 왼쪽 노드까지 이동
            stack.append(curr)
            curr = curr.left
        curr = stack.pop()
        print(curr.val)
        curr = curr.right       # 오른쪽 서브트리로 전환

6. 시간·공간 복잡도

연산일반 이진 트리완전/포화 이진 트리
순회 (4종)O(n)O(n)
높이 계산O(n)O(log n) — 균형일 때
공간 (재귀 콜 스택)O(h) — 높이만큼O(log n) 균형 / O(n) 편향
공간 (BFS 큐)O(w) — 최대 너비O(n/2) ≈ O(n) 최하위 레벨
핵심 인사이트 — 높이가 성능을 결정한다
  • 균형 이진 트리(h = O(log n)): 순회 외 탐색·삽입·삭제도 O(log n).
  • 편향 이진 트리(h = O(n)): 모든 연산이 O(n)으로 저하. BST가 이 문제를 해결하려 등장.

7. 실전 연습

연습 1: 트리 높이 계산

def height(node):
    if node is None:
        return -1  # 빈 트리의 높이를 -1로 정의 (리프 = 0)
    return 1 + max(height(node.left), height(node.right))

# 위 1-2-3-4-5 트리에서 height(root) == 2

연습 2: 노드 개수 세기

def count_nodes(node):
    if node is None:
        return 0
    return 1 + count_nodes(node.left) + count_nodes(node.right)

연습 3: 두 트리가 동일한지 비교

def is_same_tree(p, q):
    if p is None and q is None:
        return True
    if p is None or q is None:
        return False
    return p.val == q.val and is_same_tree(p.left, q.left) and is_same_tree(p.right, q.right)

연습 4: 중위 순회 결과를 리스트로 반환

def inorder_list(root):
    result = []
    def dfs(node):
        if node:
            dfs(node.left)
            result.append(node.val)
            dfs(node.right)
    dfs(root)
    return result

8. Day 6 핵심 정리

  • 트리는 계층적 관계를 표현하는 비선형 자료구조 — 사이클 없음, N개 노드 = N-1개 엣지.
  • 이진 트리는 자식이 최대 2개. 포화·완전·편향으로 분류.
  • 순회 4종: 전위(루좌우)·중위(좌루우)·후위(좌우루)·레벨(BFS).
  • 중위 순회는 BST에서 정렬 출력, 레벨 순회는 BFS에 직결.
  • 트리 높이 h가 O(log n)이면 효율적, O(n)이면 연결 리스트 수준으로 저하.
  • 재귀는 간결하지만 스택 오버플로 위험 — 깊은 트리에서는 반복 방식 사용.
