배열과 연결 리스트

Day 2 2026-06-23  ·  Arrays & Linked Lists

학습 목표

이 레슨을 마치면 다음을 할 수 있다:

  • 정적 배열과 동적 배열의 차이를 설명하고, 각각의 메모리 구조를 그릴 수 있다.
  • 단일 연결 리스트와 이중 연결 리스트를 코드로 구현하고 순회할 수 있다.
  • 배열 vs 연결 리스트의 연산별 시간복잡도를 표로 비교하고, 상황에 맞게 선택할 수 있다.
  • "어디에 삽입·삭제하느냐"에 따라 시간복잡도가 어떻게 달라지는지 설명할 수 있다.

1. 배열 (Array)

배열은 같은 타입의 데이터를 연속된 메모리 공간에 나란히 저장하는 가장 기본적인 자료구조다. 인덱스(index)를 알면 임의 원소에 O(1)로 접근할 수 있다.

인덱스: 0 1 2 3 4 ┌─────┬─────┬─────┬─────┬─────┐ 값: │ 7 │ 3 │ 9 │ 1 │ 5 │ └─────┴─────┴─────┴─────┴─────┘ 주소: 1000 1004 1008 1012 1016 (int, 4바이트씩) arr[2] 접근 → 주소 = 1000 + 2×4 = 1008 → O(1)

정적 배열의 메모리 레이아웃 — 주소 계산으로 인덱스 접근이 즉각적이다

1-1. 정적 배열 vs 동적 배열

구분정적 배열동적 배열 (예: Python list, Java ArrayList)
크기선언 시 고정런타임에 자동 확장
메모리스택 또는 정적 영역힙(heap)에 할당·재할당
확장 비용불가 (직접 새 배열 생성)크기 초과 시 2배 확장 → Amortized O(1) 추가
대표 언어C int arr[5]Python list, JavaScript Array
동적 배열의 확장 전략

동적 배열은 용량(capacity)이 꽉 차면 보통 2배짜리 새 배열을 할당하고 기존 원소를 복사한다. 복사 비용은 O(n)이지만, n번 append를 합산하면 평균 O(1)이 된다 — 이를 상각 분석(amortized analysis)이라 한다.

1-2. 배열의 삽입·삭제

배열 중간에 원소를 삽입하거나 삭제하면, 그 뒤 원소를 전부 한 칸씩 밀거나 당겨야 한다. 

# 인덱스 2에 삽입 전:  [7, 3, 9, 1, 5]
# 인덱스 2에 42 삽입 후: [7, 3, 42, 9, 1, 5]
#                              ↑ 9, 1, 5를 오른쪽으로 이동 → O(n)

# 인덱스 2 삭제 후:    [7, 3, 1, 5]
#                           ↑ 1, 5를 왼쪽으로 이동 → O(n)

2. 연결 리스트 (Linked List)

연결 리스트는 각 원소(노드)가 다음 노드를 가리키는 포인터를 함께 저장한다. 메모리가 연속적이지 않아도 되며, 삽입·삭제가 빠른 대신 임의 접근이 느리다.

2-1. 단일 연결 리스트 (Singly Linked List)

head │ ▼ ┌────┬──┐ ┌────┬──┐ ┌────┬──┐ ┌────┬──┐ │ 7 │ ─┼──▶│ 3 │ ─┼──▶│ 9 │ ─┼──▶│ 1 │ ✕│ └────┴──┘ └────┴──┘ └────┴──┘ └────┴──┘ data next data next data next data None

단일 연결 리스트 — 각 노드는 (data, next) 쌍

class Node:
    def __init__(self, data):
        self.data = data
        self.next = None   # 다음 노드 포인터

class SinglyLinkedList:
    def __init__(self):
        self.head = None

    def append(self, data):
        new_node = Node(data)
        if self.head is None:
            self.head = new_node
            return
        cur = self.head
        while cur.next:        # 꼬리(tail)까지 순회 O(n)
            cur = cur.next
        cur.next = new_node

    def prepend(self, data):   # 맨 앞 삽입 O(1)
        new_node = Node(data)
        new_node.next = self.head
        self.head = new_node

    def delete(self, data):    # 탐색 O(n), 삭제 포인터 조작 O(1)
        if self.head is None:
            return
        if self.head.data == data:
            self.head = self.head.next
            return
        cur = self.head
        while cur.next and cur.next.data != data:
            cur = cur.next
        if cur.next:
            cur.next = cur.next.next   # 노드 건너뛰기

2-2. 이중 연결 리스트 (Doubly Linked List)

head tail │ │ ▼ ▼ ┌──┬────┬──┐ ┌──┬────┬──┐ ┌──┬────┬──┐ ┌──┬────┬──┐ │✕ │ 7 │ ─┼──▶│ ◀│ 3 │ ─┼──▶│ ◀│ 9 │ ─┼──▶│ ◀│ 1 │✕ │ └──┴────┴──┘ └──┴────┴──┘ └──┴────┴──┘ └──┴────┴──┘ prev data next prev data next prev data next prev data next

이중 연결 리스트 — (prev, data, next) 세 필드. 양방향 순회 가능

이중 연결 리스트가 유리한 상황
  • 꼬리에서 앞으로 역방향 탐색이 필요할 때
  • 노드 참조(reference)를 이미 알고 있을 때 삭제가 O(1) — 앞 노드를 따로 찾지 않아도 된다.
  • LRU(Least Recently Used) 캐시 구현 — 가장 오래된 노드를 O(1)로 제거

3. 시간복잡도 비교

연산 정적/동적 배열 단일 연결 리스트 이중 연결 리스트
임의 접근 (인덱스) O(1) O(n) O(n)
맨 앞 삽입 O(n) O(1) O(1)
맨 끝 삽입 O(1)* O(n) / tail 유지 시 O(1) O(1) (tail 포인터)
중간 삽입 (위치 알 때) O(n) O(1) O(1)
탐색 O(n) O(n) O(n)
이진 탐색 가능? 가능 O(log n) 불가 (연속 메모리 아님) 불가
메모리 오버헤드 낮음 (데이터만) 중간 (next 포인터 1개) 높음 (prev + next 포인터 2개)

* 동적 배열 맨 끝 삽입은 Amortized O(1)

선택 기준 요약

임의 인덱스 접근이 잦다 → 배열 앞/중간 삽입·삭제가 잦다 → 연결 리스트 메모리가 제한적이다 → 배열 (포인터 오버헤드 없음) 캐시 지역성(cache locality) → 배열 (연속 메모리) 순서 유지 + 빠른 삽입 삭제 → 이중 연결 리스트
캐시 지역성(Cache Locality)
배열은 연속 메모리라 CPU 캐시에 올라갈 때 주변 원소도 함께 로드된다. 연결 리스트는 노드가 메모리 곳곳에 흩어져 캐시 미스(cache miss)가 잦다. 실제 벤치마크에서는 이론적 Big O 차이보다 이 효과가 크게 나타날 때가 있다.

4. 실전 패턴 — 두 포인터 (Two Pointers)

연결 리스트 문제에서 두 포인터(slow / fast) 기법은 핵심 패턴이다. 

# 연결 리스트 중간 노드 찾기 (Floyd's 알고리즘)
def find_middle(head):
    slow = head
    fast = head
    while fast and fast.next:
        slow = slow.next        # 한 칸씩
        fast = fast.next.next   # 두 칸씩
    return slow  # fast가 끝에 도달하면 slow가 중간

# 사이클 감지 (Floyd's Cycle Detection)
def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast:   # 같은 노드에서 만남 → 사이클
            return True
    return False
두 포인터가 작동하는 이유
사이클이 있으면 fast가 slow를 추월해 결국 같은 위치에서 만난다 (원형 트랙에서 빠른 선수가 느린 선수를 따라잡는 것과 같다). 사이클이 없으면 fast가 먼저 None에 도달한다.
