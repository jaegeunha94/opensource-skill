1. 자료구조란?

자료구조(Data Structure)는 데이터를 효율적으로 저장하고, 탐색하고, 수정하고, 삭제하기 위한 체계적인 방식이다. 같은 데이터라도 어떻게 구성하느냐에 따라 프로그램 속도가 수백 배 차이날 수 있다.

비유
도서관을 생각해 보자. 책이 아무렇게나 쌓여 있으면 원하는 책을 찾는 데 한참 걸린다. 하지만 분류 번호 순으로 꽂아 두면 즉시 찾을 수 있다. 자료구조는 바로 이 책 배치 방식에 해당한다.

자료구조를 배우는 이유

  • 성능 최적화: 올바른 자료구조 선택만으로 O(n²) 코드가 O(n log n)이 된다.
  • 코딩 테스트: 대부분의 알고리즘 문제는 특정 자료구조를 알면 바로 풀린다.
  • 실무 설계: 캐시, 데이터베이스 인덱스, 네트워크 라우팅 — 모두 자료구조가 핵심이다.

2. 자료구조 분류

자료구조 ├── 선형(Linear) │ ├── 배열 (Array) │ ├── 연결 리스트 (Linked List) │ ├── 스택 (Stack) │ └── 큐 (Queue) └── 비선형(Non-Linear) ├── 트리 (Tree) │ ├── 이진 탐색 트리 (BST) │ ├── 힙 (Heap) │ └── 트라이 (Trie) └── 그래프 (Graph) * 해시 테이블은 별도 범주 (선형 저장 + 해시 함수 매핑)

자료구조 전체 지도 — 이 커리큘럼에서 모두 다룬다

구분특징대표 예
선형데이터가 한 줄로 나열됨배열, 스택, 큐
비선형데이터가 계층·그물 구조트리, 그래프

3. 추상 자료형 (ADT)

추상 자료형(Abstract Data Type, ADT)은 자료구조를 무엇을 할 수 있는가 (인터페이스) 관점에서만 정의하고, 어떻게 구현하는가는 숨긴 개념이다.

핵심 구분
ADT = "뭘 할 수 있냐" (계약)
자료구조 = "어떻게 구현하냐" (구체적 메모리 구조)

예시: 스택 ADT

스택 ADT 정의:
  push(item)   → 맨 위에 데이터 추가
  pop()        → 맨 위 데이터 제거 후 반환
  peek()       → 맨 위 데이터 확인 (제거 안 함)
  is_empty()   → 비어 있으면 True

구현 방법은 여러 가지:
  1. 배열(Array)로 구현
  2. 연결 리스트(Linked List)로 구현
  → 인터페이스는 동일, 내부 구조만 다름

ADT를 먼저 이해하면 "어떤 연산이 필요한가"를 설계하고, 그 다음 "가장 효율적인 구현"을 선택하는 사고 흐름이 생긴다.

4. Big O 표기법 — 성능 언어

자료구조를 비교하려면 공통 언어가 필요하다. Big O 표기법은 입력 크기 n이 커질 때 연산 횟수가 어떻게 늘어나는지를 나타낸다. 상수 계수는 무시하고 지배적인 항만 남긴다.

표기이름예시 연산n=1,000일 때 연산 수
O(1)상수배열 인덱스 접근1
O(log n)로그이진 탐색~10
O(n)선형선형 탐색1,000
O(n log n)선형로그병합 정렬~10,000
O(n²)이차버블 정렬1,000,000
O(2ⁿ)지수브루트포스 부분집합10³⁰⁰ (불가)
주의
Big O는 최악의 경우(worst case)를 나타낼 때가 많다. 평균(average)·최선(best) 케이스와 구분해야 할 때도 있다 — 이는 각 자료구조 레슨에서 다룬다.

직관: 전화번호부에서 이름 찾기

  • 선형 탐색: 처음부터 한 장씩 넘김 → O(n)
  • 이진 탐색: 중간을 펴서 반씩 좁힘 → O(log n)

1,000페이지 전화번호부라면: 선형은 최대 1,000번, 이진은 최대 10번 비교면 충분하다.
