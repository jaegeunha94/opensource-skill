1. 자료구조란 무엇인가?

자료구조(Data Structure)는 데이터를 효율적으로 저장하고 접근하기 위한 조직화 방법입니다. 같은 데이터라도 어떻게 구성하느냐에 따라 작업 속도가 수백만 배 차이날 수 있습니다.

핵심 직관
10만 개의 이름이 담긴 전화번호부가 있다고 하자. 무작위로 나열되어 있으면 특정 이름을 찾으려면 평균 5만 번 확인해야 한다. 알파벳 순으로 정렬되어 있으면 이진 탐색으로 17번 이하의 비교로 찾을 수 있다. 자료구조의 선택이 성능을 결정한다.

자료구조를 선택할 때 묻는 질문

질문이유
탐색이 자주 일어나는가? 해시 테이블, 이진 탐색 트리 고려
삽입/삭제가 자주 일어나는가? 연결 리스트, 힙 고려
순서가 중요한가? 배열, 정렬된 트리 고려
메모리가 제한적인가? 공간복잡도를 함께 고려

2. 추상 자료형 (Abstract Data Type, ADT)

ADT는 자료구조의 "무엇"을 정의하는 계약서입니다 — 어떤 연산이 가능하고, 각 연산의 의미가 무엇인지. 어떻게 구현하는지는 ADT에 속하지 않습니다.

ADT: 리스트(List) ───────────────────────────────────────── 연산: insert(index, value) — index 위치에 값 삽입 remove(index) — index 위치의 값 삭제 get(index) — index 위치의 값 반환 size() — 원소 개수 반환 ↓ 구현 1: 배열(Array) ↓ 구현 2: 연결 리스트(Linked List) ↓ 구현 3: 동적 배열(ArrayList / Python list) 같은 ADT, 다른 성능 트레이드오프

그림 1 — ADT vs 구현체: ADT는 "무엇"을, 구현체는 "어떻게"를 결정한다

흔한 오해
"스택(Stack)"은 ADT입니다. LIFO(Last In, First Out) 동작을 정의하지만, 배열로 구현할 수도 있고 연결 리스트로 구현할 수도 있습니다. 자료구조 공부에서 ADT와 구현을 구분하는 것이 매우 중요합니다.

3. Big-O 표기법 — 효율을 비교하는 언어

알고리즘의 실행 시간은 입력 크기(n)에 따라 달라집니다. Big-O 표기법은 최악의 경우 입력 크기에 따른 성능 증가율을 표현합니다. 상수 계수와 하위 항은 무시하고, 가장 빠르게 증가하는 항만 남깁니다.

자주 만나는 복잡도 클래스

표기이름n=1000일 때 연산 수예시
O(1) 상수 1 배열 인덱스 접근, 해시 테이블 평균 탐색
O(log n) 로그 ~10 이진 탐색, 균형 트리 탐색
O(n) 선형 1,000 순차 탐색, 배열 순회
O(n log n) 선형 로그 ~10,000 병합 정렬, 퀵 정렬 평균
O(n²) 이차 1,000,000 버블 정렬, 중첩 반복문
O(2ⁿ) 지수 ∞ (사실상 불가) 피보나치 순수 재귀, 부분집합 열거

Big-O 구하는 방법

코드를 보고 복잡도를 추정하는 패턴을 익혀두면 빠르게 판단할 수 있습니다:

# 예제 1 — O(1): 입력 크기와 무관한 고정 연산
def get_first(arr):
    return arr[0]          # 항상 1번 연산

# 예제 2 — O(n): 루프 1개가 n번 반복
def linear_search(arr, target):
    for item in arr:       # n번 반복
        if item == target:
            return True
    return False

# 예제 3 — O(n²): 중첩 루프
def bubble_sort(arr):
    for i in range(len(arr)):        # n번
        for j in range(len(arr)-1):  # 각 i마다 n번
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
    # 총 n × n = n² 비교
시간복잡도 vs 공간복잡도
시간복잡도: 연산 횟수가 n에 따라 어떻게 증가하는가
공간복잡도: 메모리 사용량이 n에 따라 어떻게 증가하는가

두 가지 사이에는 트레이드오프가 존재합니다. 캐싱(memoization)은 공간을 더 써서 시간을 줄이는 대표적 예입니다.

4. 자료구조 전체 지도

이 과정에서 다룰 자료구조의 전체 맥락을 미리 파악해 둡니다.

자료구조 ├── 선형(Linear) — 데이터가 순서를 가짐 │ ├── 배열 (Array) → Day 2 │ ├── 연결 리스트 (Linked List) → Day 3-4 │ ├── 스택 (Stack) → Day 5 │ └── 큐 (Queue) → Day 6 │ └── 비선형(Non-linear) — 데이터가 계층/네트워크 구조 ├── 트리 (Tree) → Day 8-10 │ ├── 이진 탐색 트리 (BST) │ ├── 균형 트리 (AVL, RB-Tree) │ ├── 힙 (Heap) → Day 11 │ └── 트라이 (Trie) → Day 17 └── 그래프 (Graph) → Day 14-16 + 해시 테이블 (Hash Table) → Day 12-13 (선형/비선형 어느 쪽에도 속하지 않는 독립적 구조)

오늘의 핵심 요약

  • 자료구조는 데이터를 효율적으로 조직화하는 방법이며, 선택이 성능을 결정한다
  • ADT는 "무엇"(연산과 의미)을 정의하고, 구현은 "어떻게"를 결정한다. 같은 ADT에 여러 구현이 존재한다
  • Big-O는 입력 크기 n에 따른 최악 성능 증가율을 표현한다. 상수 계수는 무시한다
  • 복잡도 클래스 순서: O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ)
  • 자료구조 선택 시 탐색/삽입/삭제 빈도와 메모리 제약을 함께 고려한다
