1. 이진 탐색 트리(BST)란?

이진 탐색 트리(Binary Search Tree, BST)는 이진 트리에 정렬 성질을 추가한 자료구조다. 각 노드 n에 대해 다음 조건이 항상 성립한다:

  • 왼쪽 서브트리의 모든 값 < n.value
  • 오른쪽 서브트리의 모든 값 > n.value

이 단 하나의 불변 조건 덕분에 이진 탐색과 동일한 논리로 O(log n) 탐색이 가능하다. 단, 이 보장은 트리가 균형 잡혀 있을 때만 유효하다.

올바른 BST 예시 (루트 = 8): 8 / \ 3 10 / \ \ 1 6 14 / \ / 4 7 13 • 3의 왼쪽(1) < 3 < 3의 오른쪽(6) ✓ • 10의 오른쪽(14) > 10 ✓ • 14의 왼쪽(13) < 14 ✓ → 모든 노드에서 BST 성질 성립

BST — 왼쪽 < 부모 < 오른쪽 성질이 전체 트리에 재귀적으로 적용된다.

2. 탐색 (Search)

찾고자 하는 값 target과 현재 노드 값을 비교하고, 작으면 왼쪽·크면 오른쪽으로 이동한다. 이는 정렬된 배열에서의 이진 탐색과 정확히 동일한 전략이다.

def search(node, target):
    if node is None:          # 없음
        return None
    if target == node.value:
        return node
    elif target < node.value:
        return search(node.left, target)
    else:
        return search(node.right, target)
시간 복잡도
  • 평균 — O(log n) : 각 비교마다 탐색 범위가 절반으로 줄어든다.
  • 최악 — O(n) : 편향 트리(skewed tree)에서 높이가 n이 될 수 있다.

3. 삽입 (Insert)

항상 리프 위치에 삽입된다. 탐색과 같은 경로를 따라 내려가다 None에 닿으면 그 자리에 새 노드를 붙인다.

def insert(node, value):
    if node is None:
        return Node(value)
    if value < node.value:
        node.left  = insert(node.left,  value)
    elif value > node.value:
        node.right = insert(node.right, value)
    # 중복값은 무시 (구현에 따라 다름)
    return node
BST에 값 5 삽입 (위 예시 트리에 추가): 5 < 8 → 왼쪽으로 5 > 3 → 오른쪽으로 5 < 6 → 왼쪽으로 5 > 4 → 오른쪽으로 None 도달 → 여기에 삽입 8 / \ 3 10 / \ \ 1 6 14 / \ / 4 7 13 \ 5 ← 삽입 위치

값 5 삽입 — 탐색 경로를 따라 내려가 리프 자리에 배치

4. 삭제 (Delete)

삭제는 세 가지 케이스로 나뉜다:

  1. 케이스 1 — 리프 노드: 그냥 제거한다.
  2. 케이스 2 — 자식이 하나: 삭제할 노드를 그 자식으로 대체한다.
  3. 케이스 3 — 자식이 둘: 중위 후계자(in-order successor) — 오른쪽 서브트리의 최솟값 — 로 현재 값을 교체한 뒤, 그 중위 후계자 노드를 삭제한다.
케이스 3 예시: 루트 8 삭제 중위 후계자 = 오른쪽 서브트리의 최솟값 = 10 1) 루트 값을 10으로 교체 2) 오른쪽 서브트리에서 10(리프가 아닌 경우 재귀 처리)을 삭제 삭제 전: 삭제 후: 8 10 / \ / \ 3 10 → 3 14 / \ \ / \ / 1 6 14 1 6 13

자식이 둘인 삭제 — 중위 후계자로 교체

def delete(node, value):
    if node is None:
        return None
    if value < node.value:
        node.left  = delete(node.left,  value)
    elif value > node.value:
        node.right = delete(node.right, value)
    else:
        # 케이스 1·2: 자식이 없거나 하나
        if node.left is None:  return node.right
        if node.right is None: return node.left
        # 케이스 3: 자식이 둘 — 중위 후계자 찾기
        successor = find_min(node.right)
        node.value = successor.value
        node.right = delete(node.right, successor.value)
    return node

def find_min(node):
    while node.left:
        node = node.left
    return node

5. 중위 순회 = 정렬된 출력

BST에서 중위 순회(left → root → right)를 수행하면 저장된 모든 값이 오름차순으로 출력된다. 이는 BST 성질의 직접적인 결과다 — 왼쪽이 항상 작고 오른쪽이 항상 크기 때문이다.

트리: 중위 순회 결과: 8 / \ 3 10 1 → 3 → 4 → 6 → 7 → 8 → 10 → 13 → 14 / \ \ 1 6 14 ← 정렬된 순서 / \ / 4 7 13

BST의 중위 순회는 항상 오름차순 정렬 결과를 반환한다.

7. 균형 문제 (Imbalance Problem)

BST는 삽입 순서에 따라 심하게 편향될 수 있다. 정렬된 데이터를 순서대로 삽입하면 트리가 연결 리스트처럼 되어 O(n) 성능으로 저하된다.

1, 2, 3, 4, 5를 순서대로 삽입하면: 1 \ 2 \ 3 \ 4 \ 5 ← 높이 = n-1, 모든 연산 O(n)

편향 트리 — 정렬된 입력이 최악의 BST를 만든다

핵심 시사점

순수한 BST는 입력 순서에 취약하다. 이 문제를 해결하기 위해 AVL 트리와 Red-Black 트리가 등장했다 (Day 9). 이 두 구조는 삽입·삭제 시 자동으로 균형을 유지하여 최악에서도 O(log n)을 보장한다.

8. 파이썬 전체 구현

class Node:
    def __init__(self, value):
        self.value = value
        self.left  = None
        self.right = None

class BST:
    def __init__(self):
        self.root = None

    def insert(self, value):
        self.root = self._insert(self.root, value)

    def _insert(self, node, value):
        if node is None:
            return Node(value)
        if value < node.value:
            node.left  = self._insert(node.left,  value)
        elif value > node.value:
            node.right = self._insert(node.right, value)
        return node

    def search(self, value):
        return self._search(self.root, value)

    def _search(self, node, value):
        if node is None or node.value == value:
            return node
        if value < node.value:
            return self._search(node.left, value)
        return self._search(node.right, value)

    def inorder(self):
        result = []
        self._inorder(self.root, result)
        return result

    def _inorder(self, node, result):
        if node:
            self._inorder(node.left,  result)
            result.append(node.value)
            self._inorder(node.right, result)

# 사용 예시
bst = BST()
for v in [8, 3, 10, 1, 6, 14, 4, 7, 13]:
    bst.insert(v)

print(bst.inorder())   # [1, 3, 4, 6, 7, 8, 10, 13, 14]
print(bst.search(6))   # <Node value=6>
print(bst.search(99))  # None
