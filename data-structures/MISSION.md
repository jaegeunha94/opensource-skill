# Mission: 자료구조

## Why

소프트웨어를 개발하면서 "왜 이 코드는 느릴까?", "메모리를 더 효율적으로 쓸 수 없을까?" 라는 질문에 정확히 답할 수 있으려면 자료구조의 내부 동작 원리를 알아야 한다. 자료구조를 익히면 주어진 문제에 맞는 최적의 데이터 저장·탐색·수정 방법을 선택할 수 있고, 코딩 테스트와 실무 설계 모두에서 근거 있는 선택을 할 수 있다.

## Success looks like

- 배열, 연결 리스트, 스택, 큐, 트리, 그래프, 해시 테이블을 각각 언제 쓸지 설명할 수 있다.
- 주어진 자료구조 연산의 시간·공간 복잡도를 Big O 표기법으로 분석할 수 있다.
- 코딩 테스트 문제를 읽고 어떤 자료구조가 적합한지 30초 안에 판단할 수 있다.
- 직접 연결 리스트, 이진 탐색 트리, 해시 테이블을 밑바닥부터 구현할 수 있다.

## Constraints

- 하루 30~60분 학습 가능 (Day 단위 분할)
- 언어는 Python 또는 JavaScript 기준 (의사코드 병행)
- 수학적 증명보다 직관적 이해와 코드 구현 중심

## Out of scope

- 고급 그래프 알고리즘 (네트워크 플로우, 매칭 등)
- 암호화 특화 자료구조
- 병렬/분산 자료구조 (기초 이해 후 별도 과목으로)
