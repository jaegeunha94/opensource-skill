# Mission: 자료구조 (Data Structures)

## Why
CS 핵심 과목인 자료구조를 체계적으로 학습하여, 코딩 인터뷰에서 문제를 정확히 분석하고 최적의 자료구조를 선택해 구현할 수 있게 된다. 또한 실무에서 성능 병목을 진단할 때 기저에 깔린 자료구조의 특성을 근거로 판단할 수 있게 된다.

## Success looks like
- 배열, 연결 리스트, 스택, 큐, 트리, 힙, 해시 테이블, 그래프 각각의 시간/공간 복잡도를 암기 없이 유도할 수 있다
- 주어진 문제에서 어떤 자료구조가 최적인지 근거와 함께 선택할 수 있다
- 코딩 테스트에서 각 자료구조를 직접 구현하거나 표준 라이브러리로 활용할 수 있다
- 자료구조의 트레이드오프(탐색 속도 vs 삽입 속도, 메모리 vs 시간)를 설명할 수 있다
- 기술 면접에서 "왜 이 자료구조를 선택했는가?"라는 질문에 자신 있게 답할 수 있다

## Constraints
- 매일 단일 세션으로 학습 (1회 레슨 기준)
- 이론 + 코드 예제(의사코드 또는 Python)를 균형 있게 다룸
- 이전 개념을 전제로 점진적으로 심화

## Out of scope
- 고급 자료구조 구현 세부사항 (B-Tree 내부 구현, Skip List 등)은 개요 수준으로만 다룸
- 알고리즘 심화 (동적 프로그래밍, NP-완전 이론)는 별도 과목
- 특정 프로그래밍 언어의 표준 라이브러리 API 암기
