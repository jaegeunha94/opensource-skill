자료구조 용어집

레슨에서 사용하는 핵심 용어를 정의합니다. 레슨들이 이 용어집을 공통 기준으로 사용합니다.

A

ADT (Abstract Data Type, 추상 자료형)
연산의 의미와 집합을 정의하는 수학적 모델. 구현 방법을 명시하지 않는다. 예: 스택 ADT는 push, pop, peek 연산을 정의하지만 배열/연결 리스트 중 무엇으로 구현할지는 ADT 밖의 문제다.
Array (배열)
메모리의 연속된 공간에 같은 타입의 원소를 저장하는 자료구조. 인덱스로 O(1) 접근 가능. 크기가 고정된 정적 배열과 크기가 가변인 동적 배열로 나뉜다.

B

Big-O 표기법 (Big-O Notation)
알고리즘의 시간 또는 공간 복잡도를 입력 크기 n의 함수로 표현하는 점근적 표기법. 최악의 경우 성능 상한을 나타낸다. 상수 계수와 하위 항을 무시한다. 예: 3n² + 5n + 2 → O(n²).
BST (Binary Search Tree, 이진 탐색 트리)
이진 트리의 일종. 각 노드에서 왼쪽 서브트리의 모든 값 < 노드 값 < 오른쪽 서브트리의 모든 값이라는 BST 속성을 만족한다. 균형 잡힌 경우 탐색/삽입/삭제 O(log n).

D

Data Structure (자료구조)
데이터를 효율적으로 저장, 조직화, 접근하기 위한 구체적인 방법 또는 구현. ADT를 실제로 메모리에 구현한 것.

G

Graph (그래프)
정점(Vertex)과 간선(Edge)으로 구성된 비선형 자료구조. 방향 그래프(Directed)와 무방향 그래프(Undirected)로 나뉜다. 표현 방법: 인접 행렬(Adjacency Matrix), 인접 리스트(Adjacency List).

H

Hash Table (해시 테이블)
키(Key)를 해시 함수로 변환하여 배열의 인덱스로 사용하는 자료구조. 평균 O(1) 탐색/삽입/삭제. 충돌(Collision) 처리가 핵심 과제.
Heap (힙)
완전 이진 트리 구조를 가지는 자료구조. 최대 힙(Max-Heap)은 부모 ≥ 자식, 최소 힙(Min-Heap)은 부모 ≤ 자식을 항상 만족한다. 최솟값/최댓값을 O(1)에 조회, O(log n)에 삽입/삭제.

L

Linked List (연결 리스트)
각 노드가 데이터와 다음 노드의 포인터를 가지는 선형 자료구조. 삽입/삭제 O(1) (위치를 이미 알 때), 탐색 O(n). 단순/이중/원형 연결 리스트로 나뉜다.

Q

Queue (큐)
FIFO(First In, First Out) 원칙의 ADT. 삽입은 뒤(rear)에서, 삭제는 앞(front)에서 일어난다. 배열 또는 연결 리스트로 구현 가능. BFS 구현에 사용된다.

S

Space Complexity (공간복잡도)
알고리즘이 사용하는 메모리 양의 입력 크기 n에 대한 점근적 표현. 입력 데이터 크기 자체는 제외하고 추가로 필요한 메모리(보조 공간)만을 표현하기도 한다.
Stack (스택)
LIFO(Last In, First Out) 원칙의 ADT. 삽입(push)과 삭제(pop)가 모두 한 쪽(top)에서 일어난다. 함수 호출 스택, 괄호 검사, 수식 계산에 사용된다.

T

Time Complexity (시간복잡도)
알고리즘이 수행하는 연산 횟수의 입력 크기 n에 대한 점근적 표현. 최악(Worst), 평균(Average), 최선(Best) 케이스로 구분할 수 있으며, 일반적으로 최악 케이스를 기준으로 한다.
Tree (트리)
사이클이 없는 연결 그래프. 루트(Root), 내부 노드(Internal Node), 리프(Leaf) 노드로 구성된다. 이진 트리는 각 노드의 자식이 최대 2개인 트리.
Trie (트라이)
문자열 집합을 저장하고 검색하는 트리 자료구조. 각 노드가 문자 하나를 나타내며, 루트에서 리프까지의 경로가 하나의 문자열을 형성한다. 자동완성, 사전 구현에 사용.
