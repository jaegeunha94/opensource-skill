# MISSION — Backend 구성을 위한 Python 실무자 기준 학습

## 왜 이 과목을 배우는가

Python은 백엔드 시스템 구축에 가장 널리 쓰이는 언어 중 하나다. FastAPI, Django, SQLAlchemy, Celery, Docker 조합은 스타트업부터 대기업까지 실제 서비스에서 반복적으로 등장한다. 이 과목의 목표는 Python 문법 학습이 아니라 **실무 백엔드 시스템을 설계하고 운영할 수 있는 역량**을 갖추는 것이다.

## 핵심 목표

- API를 설계하고, 서비스 경계를 결정하고, 데이터 저장소를 선택하는 의사결정 능력
- 트랜잭션과 동시성 문제를 이해하고 안전하게 다루는 역량
- JWT/OAuth2 기반 인증 시스템을 직접 구현할 수 있는 실력
- Celery + Redis 기반 백그라운드 작업 파이프라인 구성 능력
- 로깅·메트릭·트레이싱으로 운영 중인 시스템을 관측하는 능력
- Docker와 CI/CD 파이프라인으로 서비스를 배포·운영하는 능력

## 학습 원칙

1. **Python 기본 문법은 이미 안다고 가정한다.** 변수, 함수, 클래스, async/await은 별도 설명 없이 사용한다.
2. **모든 개념은 코드와 함께 제시한다.** 추상적인 설명보다 실행 가능한 예제를 우선한다.
3. **설계 의사결정의 맥락을 가르친다.** "어떻게"보다 "왜 이렇게 하는가"를 더 중요하게 다룬다.
4. **실무에서 자주 마주치는 장애 시나리오를 포함한다.** N+1 쿼리, 데드락, 토큰 탈취 등.

## 수료 기준

다음을 혼자 할 수 있으면 이 과목을 마친 것이다:

- FastAPI로 CRUD + 인증이 포함된 API 서버를 설계하고 구현한다
- PostgreSQL 마이그레이션을 안전하게 적용한다
- Celery 태스크를 작성하고 장애 상황에서 재시도 전략을 결정한다
- 운영 중인 서비스의 로그·메트릭·트레이스를 읽고 문제를 진단한다
- Docker Compose로 로컬 개발 환경을 구성하고, 프로덕션 배포 체크리스트를 완성한다
